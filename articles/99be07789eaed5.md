## 理論編
### はじめに(Introduction)
データセットにアクセスして中身を確認する方法がわかったので、いよいよ最初のSQLクエリを書いてみましょう！これからわかるように、SQLクエリを使えば巨大なデータセットの中から、必要な情報だけを取り出すことができます。

まずは、`SELECT`、`FROM`、`WHERE` というキーワードを使って、指定した条件に基づいて特定の列からデータを取得する方法を学びます。

説明をわかりやすくするために、`pet_records` という架空の小さなデータセットを使います。このデータセットには `pets` という1つのテーブルだけが含まれています。

![](https://storage.googleapis.com/zenn-user-upload/c8fba5a7600c-20260212.png)

### SELECT ... FROM

最も基本的なSQLクエリは、1つのテーブルから1つの列を選択するものです。やり方は次のとおりです。

* `SELECT` の後に、取得したい列名を書く
* `FROM` の後に、テーブル名を書く

たとえば、`pets` テーブル（bigquery-public-data プロジェクト内の pet_records データベース）から `Name` 列を取得する場合、クエリは次のようになります。

![](https://storage.googleapis.com/zenn-user-upload/b722d57ac14b-20260212.png)

注意点として、SQLクエリを書くときに `FROM` に渡す引数は、シングルクォーテーション（'）やダブルクォーテーション（"）ではなく、バッククォート（`）で囲みます。

### WHERE ...

BigQuery のデータセットは非常に大きいため、通常は特定の条件を満たす行だけを取得したい場合がほとんどです。そのときに使うのが `WHERE` 句です。

次のクエリは、`Animal` 列が `'Cat'` である行に対応する `Name` 列の値を返します。

![](https://storage.googleapis.com/zenn-user-upload/e83abe8fc1f8-20260212.png)

### 複数列の取得

複数の列を取得したい場合は、列名をカンマで区切ります。

```python
query = """
        SELECT city, country
        FROM `bigquery-public-data.openaq.global_air_quality`
        WHERE country = 'US'
        """
```

すべての列を取得したい場合は、`*` を使います。

```python
query = """
        SELECT *
        FROM `bigquery-public-data.openaq.global_air_quality`
        WHERE country = 'US'
        """
```

### Q&A：書式について

**質問：三重引用符（"""）は何のため？**
→ Pythonに対して、中の内容をすべて1つの文字列として扱うよう指示しています。改行は必須ではありませんが、読みやすくなります。

**質問：SELECT や FROM は大文字で書く必要がある？**
→ 必須ではありません。SQLは大文字・小文字を区別しません。ただし、慣習として大文字で書くことが多く、その方が読みやすくなります。

### 大規模データセットの扱い

BigQuery のデータセットは非常に大きいことがあります。無料でかなりの計算が可能ですが、上限があります。

**Kaggleユーザーは30日間で5TBまで無料でスキャンできます。上限に達すると、リセットまで待つ必要があります。**

現在Kaggleで最大のデータセットは3TBなので、注意しないと数回のクエリで上限に達してしまう可能性があります。

ただし心配はいりません。データを一度にスキャンしすぎない方法を学びます。

### クエリサイズの見積もり（dry run）

実行前に、どれくらいのデータをスキャンするか見積もることができます。`QueryJobConfig` を作成し、`dry_run=True` に設定します。

```python
query = """
        SELECT score, title
        FROM `bigquery-public-data.hacker_news.full`
        WHERE type = "job" 
        """

dry_run_config = bigquery.QueryJobConfig(dry_run=True)
dry_run_query_job = client.query(query, job_config=dry_run_config)

print("This query will process {} bytes.".format(dry_run_query_job.total_bytes_processed))
```

→ このクエリは 553,320,240 バイトを処理します。

### 2) High air quality（高い空気品質）
汚染レベルがちょうど 0 と報告されたのはどれですか？

```python
# 汚染レベルがちょうど0の行をすべて選択するクエリ

zero_pollution_query = """
        SELECT *
        FROM `bigquery-public-data.openaq.global_air_quality`
        WHERE value = 0
        """

# クエリを設定
safe_config = bigquery.QueryJobConfig(maximum_bytes_billed=10**10)
query_job = client.query(zero_pollution_query, job_config=safe_config)
```
