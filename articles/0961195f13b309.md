# Abstract
Git Bashで、makeが出来るという。

というか何十年ってコマンドプロンプト使ってても、すっごい使いにくいって嫌だったんだけど、Git Bashでwindows環境でshellが使えるやんか。
知らんかった。完全にスルーしてた。
とはいえ、そのままだとmakeコマンドは使えない。

# makeコマンドインストール
[gitBash_windows.md](https://gist.github.com/evanwill/0207876c3243bbb6863e65ec5dc3f058)に書いてくれてるまんまだけど、インストールというより、ダウンロード->解凍してgit配下にコピーするだけ。
以下、手順。
1. [ezwinports](https://sourceforge.net/projects/ezwinports/files/)からmake-4.4.1-without-guile-w32-bin.zipをダウンロード。
2. 解凍
3. gitのインストールフォルダ、今回は→C:\Program Files\Git\mingw64配下にコピー。
4. 管理者権限を求められるけど、続行で。
![](https://storage.googleapis.com/zenn-user-upload/544cd3d3c44e-20240108.png =250x)
完了.
```shell:Git Bash
$ make --version
GNU Make 4.4.1
Built for Windows32
Copyright (C) 1988-2023 Free Software Foundation, Inc.
License GPLv3+: GNU GPL version 3 or later <https://gnu.org/licenses/gpl.html>
This is free software: you are free to change and redistribute it.
There is NO WARRANTY, to the extent permitted by law.
```
出来た!!すばらしい。

# ついでにwgetコマンドもインストール
1. [GNU Wget](https://eternallybored.org/misc/wget/)から、x64のEXEをダウンロード。
2. ダウンロードしたwget.exeを、C:\Program Files\Git\mingw64\bin配下にコピー
完了.
```shell:Git Bash
$ wget --version
GNU Wget 1.21.4 built on mingw32.
```
出来た!!
