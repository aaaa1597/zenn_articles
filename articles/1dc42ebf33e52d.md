## XGBoost
構造化データに対する最も正確なモデリング手法。

このチュートリアルでは、勾配ブースティングを用いたモデルの構築と最適化の方法を学びます。この手法は多くのKaggleコンペティションで優位に立っており、様々なデータセットで最先端の結果を達成しています。

### 初めに
このコースの大部分では、ランダム フォレスト メソッドを使用して予測を行ってきました。このメソッドは、多数の決定木の予測を平均化するだけで、単一の決定木よりも優れたパフォーマンスを実現します。

ランダムフォレスト法は「アンサンブル法」と呼ばれます。定義上、アンサンブル法は複数のモデル（例えばランダムフォレストの場合は複数のツリー）の予測値を組み合わせます。

次に、勾配ブースティングと呼ばれる別のアンサンブル手法について学習します。

### 勾配ブースティング (Gradient Boosting)
**勾配ブースティング**は、サイクルを繰り返しながらモデルを順番にアンサンブル（集合体）に加えていく手法です。

まず、非常に単純な単一のモデルでアンサンブルを初期化することから始まります。（その予測がどれほど不正確であっても、後から追加されるモデルがその誤差を修正していきます。）

その後、以下のサイクルを開始します：
- **予測の生成**: 現在のアンサンブルを使って、データセットの各観測値に対する予測値を計算します。予測を行う際は、アンサンブルに含まれるすべてのモデルの予測値を足し合わせます。
- **損失関数の計算**: これらの予測値を使用して、損失関数（例：[平均二乗誤差](https://en.wikipedia.org/wiki/Mean_squared_error)など）を計算します。
- **新モデルの学習**: 損失関数を最小化するように、アンサンブルに追加するための新しいモデルを学習させます。具体的には、この新モデルを加えることで全体の損失が減少するようにモデルのパラメータを決定します。（補足：「勾配ブースティング」の「勾配」とは、新モデルのパラメータを決定する際に、損失関数に対して**勾配降下法（Gradient Descent）** を使用することに由来します。）
- **追加と反復**: 新しいモデルをアンサンブルに追加し、このプロセスを繰り返します。

### 実装例
まず、訓練データと検証データを X_train, X_valid, y_train, y_valid に読み込みます。
```python
import pandas as pd
from sklearn.model_selection import train_test_split

# Read the data
data = pd.read_csv('../input/melbourne-housing-snapshot/melb_data.csv')

# Select subset of predictors
cols_to_use = ['Rooms', 'Distance', 'Landsize', 'BuildingArea', 'YearBuilt']
X = data[cols_to_use]

# Select target
y = data.Price

# Separate data into training and validation sets
X_train, X_valid, y_train, y_valid = train_test_split(X, y)
```

この例では **XGBoost** ライブラリを使用します。XGBoostは **「Extreme Gradient Boosting」** の略で、パフォーマンスと速度に焦点を当てた、いくつかの追加機能を備えた勾配ブースティングの実装です。（Scikit-learnにも別の勾配ブースティングの実装がありますが、XGBoostには技術的な利点が多くあります。）

次のコードでは、XGBoostのScikit-learn API（xgboost.XGBRegressor）をインポートします。これにより、Scikit-learnと同じようにモデルを構築し、学習させることができます。出力を見るとわかるように、XGBRegressorクラスには多くの調整可能なパラメータがあります。それらについては後ほど説明します。

```python
from xgboost import XGBRegressor

# モデルの定義
my_model = XGBRegressor()
# 学習
my_model.fit(X_train, y_train)
```

### パラメータのチューニング (Parameter Tuning)
XGBoostには、精度と学習速度に劇的な影響を与えるパラメータがいくつかあります。まず理解すべき重要なパラメータは以下の通りです。

1. n_estimators
上記で説明したモデリングのサイクルを何回繰り返すかを指定します。これはアンサンブルに含まれるモデルの数に等しくなります。
- 低すぎる値: 「未学習（Underfitting）」を引き起こし、訓練データ・テストデータの両方で予測が不正確になります。
- 高すぎる値: 「過学習（Overfitting）」を引き起こし、訓練データには正確ですが、肝心なテストデータでの予測が不正確になります。

一般的な値の範囲は 100〜1000 ですが、これは後述する learning_rate に大きく依存します。

2. early_stopping_rounds
n_estimators の理想的な値を自動的に見つける方法を提供します。早期停止（Early stopping）を使用すると、たとえ n_estimators で設定した上限に達していなくても、検証スコアの改善が止まった時点で反復を停止します。 通常は n_estimators を高めに設定しておき、early_stopping_rounds を使って最適な停止タイミングを見つけるのが賢明です。

時折、偶然の偏りによって1回だけスコアが改善しないことがあるため、停止するまでに「何回連続で悪化を許容するか」を指定する必要があります。一般的には early_stopping_rounds=5 程度が妥当な選択です。この場合、検証スコアが 5 回連続して低下したら停止します。

early_stopping_rounds を使用する場合は、検証スコアを計算するためのデータも確保する必要があります。これは、eval_set パラメータを設定することによって行われます。
上記の例を修正して、早期停止を含めることができます。

```python
my_model = XGBRegressor(n_estimators=500)
my_model.fit(X_train, y_train, 
             early_stopping_rounds=5, 
             eval_set=[(X_valid, y_valid)],
             verbose=False)
```

3. learning_rate（学習率）
各モデルの予測値をそのまま足すのではなく、各モデルの予測値に小さな数（**学習率**）を掛けてから足し合わせます。

これにより、アンサンブルに追加される各樹木の貢献度が小さくなるため、過学習することなく n_estimators の値をより大きく設定できるようになります。一般的に、**「小さな学習率」と「大きな推定実行回数（n_estimators）」**　の組み合わせが最も正確なモデルを生みますが、反復回数が増えるため学習時間は長くなります。デフォルトでは learning_rate=0.1 です。

4. n_jobs
実行時間が問題になる大きなデータセットでは、並列処理を使用してモデルをより速く構築できます。通常、マシンのCPUコア数に合わせて n_jobs を設定します。小規模なデータセットでは効果はありませんが、学習に時間がかかる大規模データでは非常に有用です。

例は次のとおりです。
```python
my_model = XGBRegressor(n_estimators=1000, learning_rate=0.05, n_jobs=4)
my_model.fit(X_train, y_train, 
             early_stopping_rounds=5, 
             eval_set=[(X_valid, y_valid)], 
             verbose=False)
```

### 結論
XGBoostは、標準的な表形式データ（画像や動画のような特殊なデータではなく、PandasのDataFrameに保存するようなデータ）を扱うための主要なライブラリです。パラメータを丁寧にチューニングすることで、非常に精度の高いモデルを構築することができます。
