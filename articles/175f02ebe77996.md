このコースでは、コンピュータビジョンの基本的な考え方を学びます。

今回扱う問題は**画像分類**です。

### 畳み込み分類器(The Convolutional Classifier)

画像分類用の convnet は、2つの部分から構成されます：

1. **畳み込みベース（convolutional base）**
2. **Dense ヘッド（dense head）**

![](https://storage.googleapis.com/zenn-user-upload/f1d84ea0329d-20260221.png)

---

#### ベース（base）

ベースの役割は、画像から特徴を抽出することです。主に畳み込み層で構成されますが、他の層に含まれることもあります。(それについては次のレッスンで学習します。)

#### ヘッド（head）

ヘッドの役割は、抽出された特徴から**クラスを決定**することです。主に Dense 層で構成され、Dropout などが含まれることもあります。

---

#### 視覚的特徴（visual feature）とは？
特徴とは、線、色、質感、形状、パターン、あるいはそれらの複雑な組み合わせを指します。

### 分類器のトレーニング

全体のプロセスは次のようになります。
![](https://storage.googleapis.com/zenn-user-upload/5e177fdb3250-20260221.png)*実際に抽出された特徴は少し異なって見えますが、アイデアは伝わります。*

トレーニング中のネットワークの目標は、次の 2 つのことを学習することです。
1. ベースが特徴を抽出
2. ヘッドがそれを使って分類

という仕組みです。

---

#### 分類器の学習(Training the Classifier)

学習中、ネットワークは2つのことを学びます：

1. 画像から抽出する特徴(base)
2. どのクラスがどの特徴(head)に対応するか。

現在では、畳み込みニューラルネットワーク を最初から学習させることはほとんどありません。事前学習済みモデルのベースを再利用し、その上に新しいヘッドを取り付けるます。

つまり、1. 特徴を抽出し、それにいくつかの新しいレイヤーを追加して 2. 分類することをすでに学習したネットワークの一部を再利用します。
これは**転移学習**と呼ばれます。

特徴(head)は通常、少数の密な層で構成されているため、比較的少量のデータから非常に正確な分類器を作成できます。
![](https://storage.googleapis.com/zenn-user-upload/967a387318b6-20260221.png)
現在の画像分類では、ほぼ必須のテクニックです。

#### Example - Convnet分類器 を訓練する

このコースでは、`「この画像は車か？トラックか？」`という問題を解きます。

データセットには、約10,000枚、うち半分が車、もう半分がトラックの構成となっています。

---

##### Step 1 - データ読み込み

* 学習用：ds_train
* 検証用：ds_valid

以下のコードは、いくつかのライブラリをインポートし、データパイプラインをセットアップします。トレーニング用のスプリットデータとしてds_train、検証用のスプリットデータとしてds_validを使用します。
```python
# Imports
import os, warnings
import matplotlib.pyplot as plt
from matplotlib import gridspec

import numpy as np
import tensorflow as tf
from tensorflow.keras.preprocessing import image_dataset_from_directory

# Reproducability
def set_seed(seed=31415):
    np.random.seed(seed)
    tf.random.set_seed(seed)
    os.environ['PYTHONHASHSEED'] = str(seed)
    os.environ['TF_DETERMINISTIC_OPS'] = '1'
set_seed(31415)

# Set Matplotlib defaults
plt.rc('figure', autolayout=True)
plt.rc('axes', labelweight='bold', labelsize='large',
       titleweight='bold', titlesize=18, titlepad=10)
plt.rc('image', cmap='magma')
warnings.filterwarnings("ignore") # to clean up output cells


# Load training and validation sets
ds_train_ = image_dataset_from_directory(
    '../input/car-or-truck/train',
    labels='inferred',
    label_mode='binary',
    image_size=[128, 128],
    interpolation='nearest',
    batch_size=64,
    shuffle=True,
)
ds_valid_ = image_dataset_from_directory(
    '../input/car-or-truck/valid',
    labels='inferred',
    label_mode='binary',
    image_size=[128, 128],
    interpolation='nearest',
    batch_size=64,
    shuffle=False,
)

# Data Pipeline
def convert_to_float(image, label):
    image = tf.image.convert_image_dtype(image, dtype=tf.float32)
    return image, label

AUTOTUNE = tf.data.experimental.AUTOTUNE
ds_train = (
    ds_train_
    .map(convert_to_float)
    .cache()
    .prefetch(buffer_size=AUTOTUNE)
)
ds_valid = (
    ds_valid_
    .map(convert_to_float)
    .cache()
    .prefetch(buffer_size=AUTOTUNE)
)
```

##### Step 2 - 事前学習済みベースの定義

最もよく使われる事前学習データセットは **ImageNet** です。Keras の[アプリケーション モジュール](https://www.tensorflow.org/api_docs/python/tf/keras/applications)には ImageNet で事前学習済みのモデルが含まれています。
今回使うのは **VGG16** です。

```python
pretrained_base = tf.keras.models.load_model(
    '../input/cv-course-models/cv-course-models/vgg16-pretrained-base',
)
pretrained_base.trainable = False
```

`trainable = False` にすることで、ベース部分は学習しません。

---

##### Step 3 - 特徴(ヘッド)を取り付ける
次に、特徴(ヘッド)分類器を接続します。この例では、隠れユニット層（最初のDense層）を使用し、その後に出力をクラス1（トラック）の確率スコアに変換する層を追加します。Flatten レイヤーは、特徴(ベース)の 2 次元出力を特徴(ヘッド)に必要な 1 次元入力に変換します。

```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    pretrained_base,
    layers.Flatten(),
    layers.Dense(6, activation='relu'),
    layers.Dense(1, activation='sigmoid'),
])
```

* Flatten → ベースの2次元出力を1次元に変換
* Dense(6) → 中間層
* Dense(1, sigmoid) → クラス1（Truck）の確率

---

##### Step 4 - 学習
最後に、モデルを学習させましょう。これは2クラス問題なので、交差エントロピーと精度のバイナリバージョンを使用します。Adamオプティマイザーは一般的に良好なパフォーマンスを示すため、これも選択します。
```python
model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['binary_accuracy'],
)
history = model.fit(
    ds_train,
    validation_data=ds_valid,
    epochs=30,
    verbose=0,
)
```
30 エポック学習してます。

ニューラルネットワークをトレーニングする際には、損失とメトリックのプロットを常に調べることをお勧めします。履歴オブジェクトには、辞書 history.history にこの情報が含まれています。Pandas を使用してこの辞書をデータフレームに変換し、組み込みメソッドでプロットすることができます。

```python
import pandas as pd

history_frame = pd.DataFrame(history.history)
history_frame.loc[:, ['loss', 'val_loss']].plot()
history_frame.loc[:, ['binary_accuracy', 'val_binary_accuracy']].plot();
```

#### まとめ(Conclusion)

このレッスンでは、特徴抽出を実行する**ベース**の上に分類器として機能する**ヘッド**がある、ConvNet 分類器の構造について学習しました。

ヘッドは、基本的には入門コースで学んだような通常の分類器です。特徴量については、ベースによって抽出された特徴量を使用します。これが畳み込み分類器の基本的な考え方です。つまり、分類器自体に特徴エンジニアリングを実行するユニットを接続できるということです。
