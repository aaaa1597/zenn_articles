![](https://source.android.com/static/docs/automotive/images/dev_avd_01.png?hl=ja)

AAOS(Android Automotive OS)とは、車載用のAndroidです。
自動車の操作パネルの部分がAndroidなんですね。今回はAVD(Android Virtual Device：Android仮想デバイス)で動かしてみました。

# Abstract
- AAOS15をフルビルドして、Androidエミュレータで動かしてみた。

# 概要
諸事情あって、Android Automotive OS の ScalableUIを理解する必要が出てきたんだけど、さっぱりわかんない。[Android公式サイト](https://source.android.com/docs/automotive/scalableui?hl=ja)にはいろいろ説明書いてくれてるけど、いつもに増してさっぱり。なので、ひとまずフルビルドから手を動かせば何かしら分かるようになるんじゃなかろうか。
ってことで、フルビルドでAAOSのAVDエミュレータを作成してみた。

# マシンスペック
[Androidの公式](https://source.android.com/docs/setup/start/requirements?hl=ja)だと64GBメモリ, 72コアのマシンでビルドして40分。72コアだってよ。すごー!そんなモンスターマシンは準備できないので、ひとまず、以下の通り。
- メモリ: 32GB
- 外付けSSD: 960GB
- NVMe: 250GB
- CPU: 24コア

SATAのSSD(960GB)が未使用で放置してたのでそれを使いたいのだけど、今のPCにはSATAに空きがない。なのでやむなくUSB接続の外付けSSDにした。
そうすると、せっかくのSSDの速度が残念になるので、CドライブのNVMeからも空き分を少し使う構成にした。
さいわい、AAOSビルド用のストレージは書込み頻繁領域と読込み中心領域に分けて設定することができるみたいだから、書込み頻繁領域にNVMeを割り当てて、読込み中心領域に外付けSSDの領域を割り当てることにする。

# ビルドにかかった時間の参考
- Repo初期化 ... 10分。だけど1時間かかった時もあった。
- AAOS15 ソースコードのダウンロード ... 2.5時間(160Mbps)。だけど丸1日で30%って時もあった(2.5Mbps)。
- AAOS15 ソースコードのビルド実行 ... 2.5時間。

## 2. リソース抑制の設定
wsl側に全リソースを割くと、自分側のPC作業が全くできなくなるから、抑制するための設定を実行しときます。
```batch: コマンドプロンプト
(echo [wsl2] && echo memory=28GB && echo swap=32GB && echo processors=20) > "%USERPROFILE%\.wslconfig"
```
実行すると、`c:\Users\Aaa\.wslconfig` ファイルができててその中身が上記になってるはず。
ホントに効いてるのかを確認
```shell: Ubuntu
wsl
$ free -h
               total        used        free      shared  buff/cache   available
Mem:            27Gi       417Mi        26Gi       3.0Mi       403Mi        26Gi
Swap:           32Gi          0B        32Gi
$ nproc
20
```
期待通り、メモリ26GB、コア数20になってる。

```tree: 構成イメージ

 ├ NVMe(250GB)
 │  ├─ Ubuntu本体
 │  └─ out
 └ USB SSD (960GB)
    └─ AOSP Source
```

### 8.2.1 ビルドターゲットのファイルを探す。


- `find . -name "*.mk` ... カレントフォルダ配下の.mkファイル
- `-exec grep ... {} +` ... 見つかったファイル全部をまとめて grep に渡して実行。
- `{}` ... 置き換え位置。
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;例えば`grep -n "COMMON_LUNCH_CHOICES" file1.mk file2.mk file3.mk ...`みたいになる。
