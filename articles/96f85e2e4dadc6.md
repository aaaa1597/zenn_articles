### 特徴量エンジニアリングの目的

特徴量エンジニアリングの目的は、とてもシンプルです。
データを、今扱っている問題により適したものにすることです。

たとえば、「体感温度」を表す指標として、**暑さ指数（ヒートインデックス）や風冷指数（ウィンドチル）** があります。これらは、気温・湿度・風速といった直接測定できる量をもとに、「人間が実際にどう感じるか」を表そうとしたものです。体感温度は、観測データを「本当に知りたいこと（外でどう感じるか）」により関連づけるための、一種の特徴量エンジニアリングの結果だと考えられます。

特徴量エンジニアリングを行う理由には、次のようなものがあります。
- モデルの予測性能を向上させるため
- 計算コストや必要なデータ量を削減するため
- 結果の解釈性を向上させるため

### 特徴量エンジニアリングの指針となる考え方

ある特徴量が有用であるためには、モデルが学習可能な形でターゲットと関係していなければなりません。
たとえば線形モデルは、線形な関係しか学習できません。したがって、線形モデルを使う場合の目標は、「特徴量とターゲットの関係が線形になるように特徴量を変換すること」です。

ここで重要なのは、特徴量に施す変換は、実質的にモデルの一部になるということです。
たとえば、正方形の土地の価格を、一辺の長さから予測したいとします。長さをそのまま使って線形モデルを当てはめると、うまくいきません。というのも、この関係は線形ではないからです。
（長さを横軸、価格を縦軸に取ると、点は曲線状に増加し、そこに引いた直線はうまく当てはまりません）
→ 長さだけを特徴量にした線形モデルは、当てはまりが悪い。

![](https://storage.googleapis.com/zenn-user-upload/652e2ad51fd3-20260207.png)
*線形モデルは、長さのみを特徴とするとうまく適合しません。*

しかし、長さを二乗して「面積」という特徴量を作ると、線形な関係が生まれます。特徴量セットに面積を加えることで、この線形モデルは放物線を表現できるようになります。
つまり、特徴量を二乗することで、線形モデルに「二乗の関係」を学習させる力を与えたのです。

![](https://storage.googleapis.com/zenn-user-upload/73aab0bb6670-20260207.png)*左：面積への適合性が大幅に向上しました。右：長さへの適合性も向上しました。*

（左：横軸が面積。点は直線状に並び、直線がよくフィットしている。
右：横軸が長さ。点は曲線状だが、適切な曲線がうまくフィットしている。）

この例から分かるように、特徴量エンジニアリングは、投入した時間に対するリターンが非常に大きいことがあります。
モデルが自力では学習できない関係を、変換という形でこちらが与えてあげるのです。
特徴量を作る際には、モデルが最高の性能を出すために、どんな情報が使えそうかを考えてみてください。

### 例：コンクリート配合データ

これらの考え方を示すために、合成特徴量をいくつか追加することで、ランダムフォレストモデルの予測性能がどのように改善されるかを見てみましょう。

[Concrete](https://www.kaggle.com/datasets/sinamhd9/concrete-comprehensive-strength) データセットには、さまざまなコンクリートの配合と、その結果得られる**圧縮強度（Compressive Strength）** が含まれています。圧縮強度は、そのコンクリートがどれだけの荷重に耐えられるかを示す指標です。このタスクでは、配合内容から圧縮強度を予測します。

```python
import pandas as pd
from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import cross_val_score

df = pd.read_csv("../input/fe-course-data/concrete.csv")
df.head()
```

|0|540.0|0.0|0.0|162.0|2.5|1040.0|676.0|28|79.99|

ここには、各種コンクリートに含まれる材料が示されています。次に、これらから派生させた合成特徴量を追加することで、モデルが重要な関係性をよりよく学習できることを見ていきます。

まずは、特徴量を追加しない状態でモデルを学習させ、ベースラインを作ります。
このようにベースラインを最初に確立するのは、特徴量エンジニアリングの良い習慣です。新しい特徴量が本当に有用かどうか、あるいは捨てて別のアイデアを試すべきかを判断する基準になります。

```python
X = df.copy()
y = X.pop("CompressiveStrength")

# Train and score baseline model
baseline = RandomForestRegressor(criterion="absolute_error", random_state=0)
baseline_score = cross_val_score(
    baseline, X, y, cv=5, scoring="neg_mean_absolute_error"
)
baseline_score = -1 * baseline_score.mean()

print(f"MAE Baseline Score: {baseline_score:.4}")
```
MAE Baseline Score: 8.232

家庭で料理をする人なら、レシピの出来は材料の絶対量よりも、材料同士の比率のほうが重要だと感じることがあるかもしれません。同じように、上の特徴量でも「比率」が圧縮強度の良い予測指標になると考えられます。

次のセルでは、3つの比率特徴量をデータセットに追加しています。

- 細骨材 / 粗骨材
- （粗骨材 + 細骨材）/ セメント
- 水 / セメント

```python
X = df.copy()
y = X.pop("CompressiveStrength")

# Create synthetic features
X["FCRatio"] = X["FineAggregate"] / X["CoarseAggregate"]
X["AggCmtRatio"] = (X["CoarseAggregate"] + X["FineAggregate"]) / X["Cement"]
X["WtrCmtRatio"] = X["Water"] / X["Cement"]

# Train and score model on dataset with additional ratio features
model = RandomForestRegressor(criterion="absolute_error", random_state=0)
score = cross_val_score(
    model, X, y, cv=5, scoring="neg_mean_absolute_error"
)
score = -1 * score.mean()

print(f"MAE Score with Ratio Features: {score:.4}")
```
MAE Score with Ratio Features: 7.948

予想どおり、性能が向上しました！
これは、新しく作った比率特徴量が、モデルにとってこれまで捉えられていなかった重要な情報を明らかにした証拠です。
