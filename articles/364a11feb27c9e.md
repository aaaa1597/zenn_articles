## Abstract
Kaggle「ディープラーニング入門の[確率的勾配降下法](https://www.kaggle.com/code/ryanholbrook/stochastic-gradient-descent)」の翻訳と実行方法の解説

## 確率的勾配降下法
Keras と Tensorflow を使用して、最初のニューラル ネットワークをトレーニングします。

## 理論編
### はじめに(Introduction)
最初の2つのレッスンでは、Denseレイヤーを積み重ねて全結合ネットワークを構築する方法を学びました。ネットワークを最初に作成したとき、その重みはすべてランダムに設定されています。つまり、ネットワークはまだ何も「知らない」状態です。このレッスンでは、ニューラルネットワークをどのように訓練するのか、そしてどのように学習するのかを見ていきます。

あらゆる機械学習タスクと同様に、まずは訓練データから始めます。訓練データの各サンプルは、いくつかの特徴量（入力）と、それに対応する正解のターゲット（出力）で構成されています。ネットワークを訓練するとは、特徴量をターゲットに変換できるように重みを調整することを意味します。

たとえば、80 Cereals データセットでは、各シリアルの「sugar（糖分）」「fiber（食物繊維）」「protein（タンパク質）」の値から、「calories（カロリー）」を予測できるネットワークを作りたいと考えます。もしうまく訓練できれば、その重みは、訓練データに表現されている特徴量とターゲットの関係を何らかの形で表しているはずです。

### 訓練に必要なもの

訓練データに加えて、さらに次の2つが必要です。

* 予測の良し悪しを測る「損失関数（loss function）」
* 重みをどのように変えるかを決める「最適化アルゴリズム（optimizer）」

### 損失関数（Loss Function）

これまでにネットワークのアーキテクチャ設計は見てきましたが、「どんな問題を解くのか」をネットワークにどう伝えるかはまだ扱っていませんでした。それを担うのが**損失関数**です。

損失関数は、ターゲットの真の値とモデルの予測値とのずれ（差）を測ります。

問題の種類によって適切な損失関数は異なります。これまで扱ってきたのは**回帰問題**で、数値を予測するタスクでした（例：80 Cereals のカロリー、Red Wine Quality の評価など）。他にも、住宅価格や車の燃費予測などがあります。

回帰問題でよく使われる損失関数に **平均絶対誤差（MAE: Mean Absolute Error）** があります。
各予測値 `y_pred` に対して、真の値 `y_true` との差の絶対値 `abs(y_true - y_pred)` を計算します。

データセット全体の MAE は、これらの絶対誤差の平均です。
![](https://storage.googleapis.com/zenn-user-upload/0772d8a29360-20260218.png)*平均絶対誤差は、近似曲線とデータ ポイント間の平均長さです。*

つまり、MAE は「回帰曲線とデータ点との距離の平均」を表します。

回帰問題で使われる他の損失関数としては、平均二乗誤差（MSE）や Huber loss（どちらも Keras で利用可能）があります。

訓練中、モデルは損失関数を手がかりにして重みの正しい値を探します（損失が小さいほど良い）。言い換えると、損失関数はネットワークにとっての「目的」を示しています。

### 最適化アルゴリズム ― 確率的勾配降下法（SGD）

解くべき問題は定義できましたが、次は「どう解くか」です。それを担当するのが**最適化アルゴリズム**です。最適化アルゴリズムは、損失を最小にするように重みを調整します。

ディープラーニングで使われる最適化アルゴリズムのほとんどは、**確率的勾配降下法（Stochastic Gradient Descent, SGD）** と呼ばれる手法の一種です。

これは反復的（イテレーティブ）なアルゴリズムで、次のようなステップを繰り返します。

1. 訓練データの一部をサンプリングし、ネットワークに通して予測を行う
2. 予測値と真の値との損失を測る
3. 損失が小さくなる方向へ重みを調整する

これを損失が十分小さくなるまで（あるいはそれ以上減らなくなるまで）繰り返します。

![](https://storage.googleapis.com/kaggle-media/learn/images/rFI1tIk.gif)*確率的勾配降下法を使用してニューラル ネットワークをトレーニングします。*

各ステップで使う訓練データのサンプルを **ミニバッチ（minibatch、または batch）** と呼びます。訓練データ全体を1周することを **エポック（epoch）** と呼びます。エポック数とは、各訓練サンプルを何回ネットワークに見せるかを表します。

このアニメーションは、レッスン 1 の線形モデルが SGD でトレーニングされている様子を示しています。淡い赤の点はトレーニング セット全体を表し、実線の赤の点はミニバッチを表します。SGD は新しいミニバッチを検出するたびに、重み (w は傾き、b は y 切片) をそのバッチの正しい値にシフトします。バッチを重ねるごとに、直線は最終的に最適な値に収束します。重みが真の値に近づくにつれて、損失が小さくなっていることがわかります。


### 学習率とバッチサイズ

重みは毎回少しずつしか動きません。この移動量を決めるのが **学習率（learning rate）** です。

学習率が小さいほど、重みが最適値に収束するまでにより多くのミニバッチが必要になります。

学習率とバッチサイズは、SGDの学習の進み方に大きな影響を与える2つの重要なパラメータです。その相互作用は微妙で、最適な値を見つけるのは簡単ではありません。

幸い、多くの場合は大規模なハイパーパラメータ探索をしなくても、十分な結果を得られます。**Adam** は適応的学習率を持つSGD系アルゴリズムで、特別な調整をしなくても多くの問題でうまく機能します（ある意味「自己調整型」）。Adam は汎用性の高い優れたオプティマイザです。

### 損失関数とオプティマイザの追加

モデルを定義した後、`compile` メソッドで損失関数とオプティマイザを追加します。

```python
model.compile(
    optimizer="adam",
    loss="mae",
)
```

文字列で指定できるのが分かります。もちろん、Keras API を通じて直接設定することも可能ですが、今回はデフォルト設定で十分です。
:::message
**勾配（gradient）** とは、重みをどの方向に動かせばよいかを示すベクトルです。より正確には、損失を最も速く変化させる方向を示します。

勾配を使って損失曲線を最小値へ向かって「下っていく」ので、**勾配降下法（gradient descent）** と呼ばれます。

また、「確率的（stochastic）」とは「偶然によって決まる」という意味です。ミニバッチがデータセットからランダムに抽出されるため、学習は確率的になります。これが SGD と呼ばれる理由です。
:::

現時点で注目すべき点として、各特徴量を区間 [ 0 , 1 ] に収まるように再スケール設定していることが挙げられます。レッスン5で詳しく説明しますが、ニューラルネットワークは入力が共通のスケールにある場合に最も高いパフォーマンスを発揮する傾向があります。

### 4) 学習率とバッチサイズ(Learning Rate and Batch Size)

これらのパラメータを変更すると、どのような影響がありましたか？
よく考えたあと、次のセルを実行して解説を確認してください。
![](https://storage.googleapis.com/zenn-user-upload/b8a4784280f0-20260219.png)
バッチサイズが小さいほど、重みの更新と損失曲線のノイズが多くなることがお分かりでしょう。これは、各バッチが少量のデータサンプルであり、サンプル数が少なければ推定値にノイズが多くなる傾向があるためです。ただし、バッチを小さくすると「平均化」効果が得られ、それが有益な場合もあります。

学習率が小さいほど更新が小さくなり、トレーニングの収束に時間がかかります。学習率を大きくするとトレーニング速度は向上しますが、最小値に落ち着くことはありません。学習率が大きすぎると、トレーニングが完全に失敗する可能性があります。（学習率を0.99などの大きな値に設定して、この現象を確認してみてください。）
