### Dates（日付）
次に、日付について説明します。日付は実際のデータベースで非常によく使われます。BigQuery では、日付は **DATE** 型または **DATETIME** 型のどちらかで保存されます。

**DATE** 形式では、最初に年、その次に月、そして日が続きます。形式は次の通りです。

```
YYYY-[M]M-[D]D
```
- YYYY：4桁の年
- [M]M：1桁または2桁の月
- [D]D：1桁または2桁の日

したがって、2019-01-10 は 2019年1月10日として解釈されます。

**DATETIME** 形式は DATE 形式に似ていますが、末尾に時刻が追加されます。

accident_2015 テーブルを調べてみましょう。これは最初の数行の表示です。

このテーブルを使って、曜日ごとに事故件数がどのように変化するかを調べます。

* consecutive_number 列には、各事故に対する一意のIDが含まれており、
* timestamp_of_crash 列には、事故発生日時が DATETIME 形式で保存されています。

そこで、次のようにします。

* timestamp_of_crash 列から曜日を（下のクエリでは day_of_week として）**EXTRACT** で取り出し、
* 曜日ごとに **GROUP BY** でグループ化し、
* consecutive_number 列を **COUNT** して、曜日ごとの事故件数を求めます。

その後、**ORDER BY** 句を使って並び替え、事故件数が多い曜日から順に表示します。
```python
# Query to find out the number of accidents for each day of the week
query = """
        SELECT COUNT(consecutive_number) AS num_accidents, 
               EXTRACT(DAYOFWEEK FROM timestamp_of_crash) AS day_of_week
        FROM `bigquery-public-data.nhtsa_traffic_fatalities.accident_2015`
        GROUP BY day_of_week
        ORDER BY num_accidents DESC
        """
```

### 2）調べる価値のある指標コードを見つける

前の問題では、SE.XPD.TOTL.GD.ZS という特定のコードに注目するように指示がありました。しかし、他にも興味深い指標コードをどのように見つければよいでしょうか？

このデータセットには数千ものコードが含まれているため、すべてを確認するのは大変です。ただし、多くのコードはごく一部の国でしか報告されていません。指標コードを探す際には、多くの国で報告されているコードに絞るとよいでしょう。

2016年において少なくとも175行ある指標コードについて、indicator_code と indicator_name を選択するクエリを書いてください。

要件：

* 各指標コードにつき1行とすること。
* 結果の列名は indicator_code、indicator_name、num_rows とすること。
* 元のデータベースで175行以上あるコードのみを選択すること（ちょうど175行も含む）。
* indicator_code と indicator_name の両方を結果に含めるため、SELECT と GROUP BY の両方にそれぞれを含めること。
* 出現回数が多い順に並べること。

#### あなたのコードを書いてください

code_count_query = 
"""
SELECT indicator_code, indicator_name, count(1) as num_rows
FROM `bigquery-public-data.world_bank_intl_education.international_education`
WHERE year = 2016
GROUP BY indicator_code, indicator_name
HAVING count(1) >= 175
ORDER BY count(1) desc
"""
