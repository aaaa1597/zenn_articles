## 機械学習の説明可能性
あなたのモデルは、どのような特徴を重要だと考えていますか？

## 理論編
### はじめに(Introduction)
最も基本的な質問の一つは、「モデルの予測に最も大きな影響を与える特徴量は何か？」というものです。

この概念は **特徴量重要度(feature importance)** と呼ばれます。

特徴量重要度を測定する方法はいくつもあります。ある方法は、上の質問と少しだけ異なる意味の問いに答えるものです。また、欠点が指摘されている方法もあります。

このレッスンでは **順列重要度(permutation importance)** に焦点を当てます。多くの他の方法と比べて、置換重要度 は次の特徴があります。

* 計算が速い
* 広く使われており理解されている
* 特徴量重要度の指標として望ましい性質と整合している

### 仕組み(How It Works)

置換重要度 は、これまで見てきた方法とは異なる形でモデルを使います。そのため、最初は混乱する人も多いです。そこで、まず具体的な例から説明します。

次のような形式のデータを考えてみましょう。

![](https://storage.googleapis.com/zenn-user-upload/d14e0dc5526d-20260314.png)

私たちは 10歳時点のデータを使って、その人が20歳になったときの身長を予測したいとします。

データには次のような特徴量が含まれています。

* 予測に役立つ特徴量（例：10歳時点の身長）
* 予測力がほとんどない特徴量（例：持っている靴下の数）
* ここでは詳しく扱わないその他の特徴量

置換重要度 は **モデルを学習した後に計算**されます。したがって、モデルそのものや、特定の値（身長・靴下の数など）に対する予測結果を変更することはありません。

代わりに、次のような質問をします。
「検証データの1つの列だけをランダムにシャッフルし、目的変数と他の列はそのままにしたら、予測の精度はどう変化するだろうか？」

![](https://storage.googleapis.com/zenn-user-upload/1f8ec9bb9e27-20260314.png)

単一の列をランダムに並べ替えると、結果として得られるデータが現実世界で観測されるものとは一致しなくなるため、予測精度が低下するはずです。モデルの精度は、予測に大きく依存していた列をシャッフルした場合に特に低下します。この場合、10歳時の身長をシャッフルすると、予測精度が著しく低下します。一方、所有している靴下をシャッフルして比較しても、予測結果の精度はそれほど低下しないと思われます。

この考え方を使うと、手順は次のようになります。

1. 学習済みモデルを用意する
2. データセットの1つの列をシャッフルし、そのデータで予測を行う。予測値と正解値から、損失関数がどれだけ悪化したかを計算する。この性能低下の大きさが、その特徴量の重要度となる。
3. データを元の順序に戻す（手順2のシャッフルを元に戻す）。そして次の列に対して同じ手順を繰り返し、すべての列の重要度を計算する。

### コード例
この例では、サッカー/フットボールのチーム統計をもとに、そのチームに 「Man of the Game」受賞者がいるかどうかを予測するモデルを使用します。「Man of the Game」は、その試合で最も活躍した選手に与えられる賞です。ここではモデル構築自体は主目的ではないため、以下のコードでデータの読み込みと簡単なモデル作成を行います。

```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier

data = pd.read_csv('../input/fifa-2018-match-statistics/FIFA 2018 Statistics.csv')
y = (data['Man of the Match'] == "Yes")  # Convert from string "Yes"/"No" to binary
feature_names = [i for i in data.columns if data[i].dtype in [np.int64]]
X = data[feature_names]
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state=1)
my_model = RandomForestClassifier(n_estimators=100,
                                  random_state=0).fit(train_X, train_y)
```

次に、[`eli5`](https://eli5.readthedocs.io/en/latest/) ライブラリを使って 置換重要度 を計算して表示します。

```python
import eli5
from eli5.sklearn import PermutationImportance

perm = PermutationImportance(my_model, random_state=1).fit(val_X, val_y)
eli5.show_weights(perm, feature_names = val_X.columns.tolist())
```

### 順列の重要性の解釈(Interpreting Permutation Importances)
上記表は、上の方にある値ほど 重要な特徴量であり、下に行くほど 重要度が低い特徴量です。

各行の最初の数値は、列をランダムにシャッフルしたときに モデルの性能がどれだけ低下したかを示しています(この例では評価指標として accuracy を使用しています)。

データサイエンスの多くの手法と同様に、シャッフルによる性能変化には多少のランダム性があります。そのため 置換重要度 では、シャッフルを複数回繰り返します。`±` の後ろの値は、シャッフルを繰り返したときに 性能低下の値がどれだけばらついたかを表しています。

たまに、負の値の重要度が出ることがあります。これは、シャッフルされた(ノイズの入った)データの方が、偶然にも元データより正確に予測できてしまった場合です。これは、その特徴量が実際には重要でない(本来は0に近い)にも関わらず、ランダムな偶然によってシャッフルされたデータに対する予測精度が向上した場合に発生します。特に、この例のように データセットが小さい場合は、偶然の影響が大きくなるため起こりやすくなります。

この例では、最も重要な特徴量は **得点数(Goal Scored)**でした。これは直感的にも納得できる結果です。サッカーファンであれば、他の変数の重要度の順序が意外かどうかについても、ある程度の直感を持てるかもしれません。
