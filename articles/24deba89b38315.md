## 理論編
### はじめに(Introduction)
これまでこのコースでは、ニューラルネットワークが**回帰問題**をどのように解けるかを学んできました。
ここからは、機械学習でよくあるもう一つの問題、**分類（classification）** にニューラルネットワークを適用していきます。

これまで学んできたことのほとんどはそのまま使えます。
主な違いは、**使用する損失関数**と、**最終層で出力させる出力の種類**です。

### Binary Classification（二値分類）

2つのクラスのいずれかに分類する問題は、非常に一般的な機械学習タスクです。

たとえば：
* 顧客が購入するかどうか
* クレジットカード取引が不正かどうか
* 深宇宙信号が新しい惑星の証拠を示しているかどうか
* 医療検査が病気を示しているかどうか

これらはすべて**二値分類問題**です。

元のデータでは、クラスは `"Yes"` と `"No"`、あるいは `"Dog"` と `"Cat"` のような文字列で表されているかもしれません。
ニューラルネットワークで使う前に、これらに**数値ラベル**を割り当てます：

* 片方のクラスを **0**
* もう片方を **1**

このように数値にすることで、ニューラルネットワークが扱える形式になります。

### Accuracy と Cross-Entropy

#### Accuracy（正解率）

**Accuracy** は分類問題の成功度を測る代表的な指標の一つです。

$$ \text{Accuracy} = \frac{\text{正しく予測した数}}{\text{全予測数}} $$

常に正しく予測するモデルなら、accuracy は **1.0** になります。

クラスの出現頻度がほぼ同じ場合、accuracy は妥当な評価指標です。

#### Accuracy(とその他のほとんどの分類指標)の問題点
Accuracy（および多くの分類指標）は、**損失関数として使えません**。
理由は：
* SGD（確率的勾配降下法）は、**滑らかに変化する損失関数**を必要とする
* Accuracy は「正解数の比率」なので、値が**飛び飛びに変化する**
つまり、連続的に微分できないため学習に使えないのです。

そこで代わりに使うのが **クロスエントロピー（cross-entropy）** です。

さて、損失関数は、学習中にネットワークが最小化しようとする目標を定義します。
回帰では：
* 予測値と正解値の「距離」を最小化
* MAE などを使用

という流れでした。

そして、分類では：
* **確率同士の距離**を最小化したい
* そのためにクロスエントロピーを使用

とします。
クロスエントロピーは、ある確率分布と別の確率分布との「距離」のようなものを測る指標です。

![](https://storage.googleapis.com/zenn-user-upload/6c7e3d97cd56-20260221.png)*クロスエントロピーは不正確な確率予測にペナルティを課します。*

#### 直感的なイメージ

私たちは、モデルに正しいクラスを **確率1.0で予測してほしい**と考えます。

* 予測確率が 1.0 に近い → 損失は小さい
* 1.0 から遠い → 損失は大きい

つまり、**間違った確率予測を強く罰する**のがクロスエントロピーです。

技術的な理由はやや複雑ですが、重要なのは：

> - 分類問題ではクロスエントロピーを損失関数として使う
> - Accuracy などの指標は、それに伴って改善していく

ということです。

### シグモイド関数で確率を作る

クロスエントロピーや accuracy は、**0〜1 の確率値**を入力として必要とします。

しかし Dense 層の出力は実数（−∞〜∞）です。

そこで使うのが **シグモイド（sigmoid）活性化関数**です。
![](https://storage.googleapis.com/zenn-user-upload/76899dd68b4b-20260221.png)*シグモイド関数は、実数を区間 [0,1] にマッピングします。*

最終的なクラス予測を得るために、閾値確率を定義します。通常、この値は 0.5 になるため、四捨五入によって正しいクラスが得られます。0.5 未満はラベル 0 のクラスを意味し、0.5 以上はラベル 1 のクラスを意味します。0.5 のしきい値は、Keras が[精度メトリック](https://www.tensorflow.org/api_docs/python/tf/keras/metrics/BinaryAccuracy)でデフォルトで使用するものです。

つまり、
シグモイドは下記の役割を持っています。：
* S字カーブ
* 左側で 0 に近づく
* 右側で 1 に近づく
* 実数を **[0,1] の範囲**に変換する

#### そして最終的な所属クラスを決定します。
確率からクラスを決めるには、**閾値（threshold）** を定めます。
通常は **0.5** を使います。
* 0.5 未満 → クラス 0
* 0.5 以上 → クラス 1
Keras の accuracy 指標も、デフォルトで 0.5 を使用します。

### 例：Binary Classification

#### Ionosphere データセット

[Ionosphere データセット](https://archive.ics.uci.edu/dataset/52/ionosphere)は、地球大気の電離層に向けたレーダー信号から得られた特徴量を含んでいます。
目的は、信号が何らかの物体の存在を示しているのか、それともただの空気を示しているのかを判断することです。

```python
import pandas as pd
from IPython.display import display

ion = pd.read_csv('../input/dl-course-data/ion.csv', index_col=0)
display(ion.head())

df = ion.copy()
df['Class'] = df['Class'].map({'good': 0, 'bad': 1})

df_train = df.sample(frac=0.7, random_state=0)
df_valid = df.drop(df_train.index)

max_ = df_train.max(axis=0)
min_ = df_train.min(axis=0)

df_train = (df_train - min_) / (max_ - min_)
df_valid = (df_valid - min_) / (max_ - min_)
df_train.dropna(axis=1, inplace=True) # drop the empty feature in column 2
df_valid.dropna(axis=1, inplace=True)

X_train = df_train.drop('Class', axis=1)
X_valid = df_valid.drop('Class', axis=1)
y_train = df_train['Class']
y_valid = df_valid['Class']
```

### モデル定義

回帰のときとほぼ同じ構造ですが、最後の層に違いがあります。

```python
layers.Dense(1, activation='sigmoid')
```

最終層に **sigmoid** を入れることで、出力が確率になります。

```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.Dense(4, activation='relu', input_shape=[33]),
    layers.Dense(4, activation='relu'),    
    layers.Dense(1, activation='sigmoid'),
])
```

### compile の設定
コンパイル メソッドを使用して、クロスエントロピー損失と精度メトリックをモデルに追加します。2クラスの問題の場合は、必ず「バイナリ」バージョンを使用してください。（分類数が多い問題の場合は、若干異なります。）Adamは、オプティマイザーとして、分類にも非常に効果的なので、これを使い続けることにします。

```python
model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['binary_accuracy'],
)
```

* optimizer：adam
* loss：binary_crossentropy
* metric：binary_accuracy

### Early Stopping の設定

この問題では学習に多くのエポックが必要(学習にデータを何周も回す必要がある)なので、**EarlyStopping**(便宜上早期停止コールバック)を使います(を設定します)。
※ エポック（epoch）... 学習データを「全部」1回まわし切ること

```python
early_stopping = keras.callbacks.EarlyStopping(
    patience=10,
    min_delta=0.001,
    restore_best_weights=True,
)

history = model.fit(
    X_train, y_train,
    validation_data=(X_valid, y_valid),
    batch_size=512,
    epochs=1000,
    callbacks=[early_stopping],
    verbose=0, # hide the output because we have so many epochs
)
```

* patience=10
* min_delta=0.001
* restore_best_weights=True

これを定義することで、下記が出来るようになります：
* 改善が止まったら学習終了
* 最良の重みに戻す

### 結果
いつものように学習曲線を確認し、検証セットで得られた損失と精度の最適値も検証します。（早期終了により、重みはこれらの値を取得した時点のものに戻されることに注意してください。）
```python
history_df = pd.DataFrame(history.history)
# Start the plot at epoch 5
history_df.loc[5:, ['loss', 'val_loss']].plot()
history_df.loc[5:, ['binary_accuracy', 'val_binary_accuracy']].plot()

print(("Best Validation Loss: {:0.4f}" +\
      "\nBest Validation Accuracy: {:0.4f}")\
      .format(history_df['val_loss'].min(), 
              history_df['val_binary_accuracy'].max()))
```

**結果：**
* **Best Validation Loss: 0.3534**
* **Best Validation Accuracy: 0.8857**

つまり：

* 約 88.6% の正解率
* クロスエントロピー損失は 0.3534

となりました。

## 実践編
### 導入(Introduction)

この演習では、**ホテル予約のキャンセルを予測する二値分類モデル**を構築します。
![](https://storage.googleapis.com/zenn-user-upload/6dc949b65afc-20260221.png)

まず、Hotel Cancellations データセットを読み込みます。
```python
import pandas as pd

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.impute import SimpleImputer
from sklearn.pipeline import make_pipeline
from sklearn.compose import make_column_transformer

hotel = pd.read_csv('../input/dl-course-data/hotel.csv')

X = hotel.copy()
y = X.pop('is_canceled')

X['arrival_date_month'] = \
    X['arrival_date_month'].map(
        {'January':1, 'February': 2, 'March':3,
         'April':4, 'May':5, 'June':6, 'July':7,
         'August':8, 'September':9, 'October':10,
         'November':11, 'December':12}
    )

# 数値特徴量とカテゴリ特徴量を分ける
features_num = [
    "lead_time", "arrival_date_week_number",
    "arrival_date_day_of_month", "stays_in_weekend_nights",
    "stays_in_week_nights", "adults", "children", "babies",
    "is_repeated_guest", "previous_cancellations",
    "previous_bookings_not_canceled", "required_car_parking_spaces",
    "total_of_special_requests", "adr",
]
features_cat = [
    "hotel", "arrival_date_month", "meal",
    "market_segment", "distribution_channel",
    "reserved_room_type", "deposit_type", "customer_type",
]

#下記の変換ルールを定義 
#- 数値：欠損値補完 → 標準化
#- カテゴリ：欠損値補完 → One-Hot エンコーディング
transformer_num = make_pipeline(
    SimpleImputer(strategy="constant"), # there are a few missing values
    StandardScaler(),
)
transformer_cat = make_pipeline(
    SimpleImputer(strategy="constant", fill_value="NA"),
    OneHotEncoder(handle_unknown='ignore'),
)

# 変換実行
preprocessor = make_column_transformer(
    (transformer_num, features_num),
    (transformer_cat, features_cat),
)

# `train_test_split` では `stratify=y` を指定して
#     学習用と検証用のデータで、キャンセルあり／なしの比率が同じになるようにする
# stratify - make sure classes are evenlly represented across splits
X_train, X_valid, y_train, y_valid = \
    train_test_split(X, y, stratify=y, train_size=0.75)

X_train = preprocessor.fit_transform(X_train)
X_valid = preprocessor.transform(X_valid)

# 入力層のサイズを決める
input_shape = [X_train.shape[1]]
```

### 1) モデル定義(Define Model)

今回のモデルは、バッチ正規化層とドロップアウト層の両方が含まれます。読みやすくするために図をブロックに分割していますが、通常どおりレイヤーごとに定義できます。

**BatchNorm → Dense → BatchNorm → Dropout → Dense → BatchNorm → Dropout → Dense** という構成です。

![](https://storage.googleapis.com/zenn-user-upload/884a2026fe53-20260221.png)*バイナリ分類器の図*

ポイント：

* BatchNormalization → 学習を安定・高速化
* Dropout → 過学習を防ぐ
* 最終層は1ユニット＋sigmoid → 確率出力


```python
model = keras.Sequential([
    layers.BatchNormalization(input_shape=input_shape),
    layers.Dense(256, activation='relu'),
    layers.BatchNormalization(),
    layers.Dropout(0.3),
    layers.Dense(256, activation='relu'),
    layers.BatchNormalization(),
    layers.Dropout(0.3),
    layers.Dense(1, activation='sigmoid'),
])
```
最後は二値分類なので **sigmoid** を使います。

![](https://storage.googleapis.com/zenn-user-upload/b79117026028-20260221.png)


### 2) オプティマイザー、損失、メトリックの追加(Add Optimizer, Loss, and Metric)

ここで、Adam オプティマイザーと、クロスエントロピー損失および精度メトリックのバイナリ バージョンを使用してモデルをコンパイルします。

```python
model.compile(
    optimizer='adam',
    loss='binary_crossentropy',
    metrics=['binary_accuracy'],
)
```

* 損失関数：binary_crossentropy
* 評価指標：binary_accuracy

### 3) 学習と評価(Train and Evaluate)

学習曲線を見て判断します。
- モデルの適合度は低すぎますか？
  もし、両方とも高いままなら → **アンダーフィット**です。
- それとも高すぎますか？
  もし、訓練損失 ↓、検証損失 ↑、なら → **オーバーフィット**です。
- クロスエントロピー損失は精度の指標として適切でしたか？
  もし、訓練損失 ↓、検証損失 も ↓、両者が近い　なら → **良いフィット**です。


![](https://storage.googleapis.com/zenn-user-upload/fb0fe65db8ef-20260221.png)
トレーニング損失が減少し続けていることがわかりますが、早期停止コールバックにより過剰適合が防止されました。さらに、クロスエントロピーの減少と同率で精度が向上したため、クロスエントロピーを最小化することが良い代替策となったようです。全体として、このトレーニングは成功したようです！

通常、下記の対応関係が見られるはずです。
* loss が下がる
* accuracy が上がる

つまり：
> クロスエントロピーは accuracy の良い代理指標になっている
と言えます。

もし両方の曲線の動きが似ていれば、それは「良い stand-in」だったということです。
