## 理論編
### はじめに(Introduction)
これで畳み込み分類器の基礎を学び終えたので、いよいよ、より高度なトピックへ進む準備が整いました。

このレッスンでは、画像分類器の性能を向上させるためのテクニックを学びます。それは「**データ拡張（data augmentation）**」と呼ばれるものです。

### 偽データの有用性(The Usefulness of Fake Data)
機械学習モデルの性能を改善する最も良い方法は、より多くのデータで学習させることです。
モデルが学習できる例が多ければ多いほど、画像の違いのうち「重要な違い」と「無視すべき違い」をより適切に見分けられるようになります。データが多いほど、モデルの汎化性能は向上します。

データを増やす簡単な方法の一つは、すでに持っているデータを活用することです。もしデータセット内の画像を「クラスを保ったまま」変換できるなら、分類器にそのような変換を無視するよう学習させることができます。たとえば、写真の中で車が左向きか右向きかは、それが「Car」であって「Truck」ではないという事実を変えません。したがって、左右反転した画像を学習データに追加すれば、分類器は「左か右か」という違いを無視すべきであると学習します。

これがデータ拡張の基本的な考え方です。
本物のデータに十分似た“追加の疑似データ”を加えることで、分類器の性能を向上させます。

### データ拡張の使用(Using Data Augmentation)
通常、データセットを拡張する際にはさまざまな変換が用いられます。画像の回転、色やコントラストの調整、画像の歪み変形など、多くの場合これらを組み合わせて適用します。以下は、1枚の車の画像に対して施されるさまざまな変換の例です。

![](https://storage.googleapis.com/zenn-user-upload/c30be42cf245-20260225.png)

データ拡張は通常オンラインで行われます。つまり、画像が学習のためにネットワークへ入力されるその場で変換が行われます。学習は通常ミニバッチ単位で行われることを思い出してください。データ拡張を使用した場合、16枚の画像からなるバッチは次のようになります。

![](https://storage.googleapis.com/zenn-user-upload/88ab55a3e0df-20260225.png)

学習中に画像が使われるたびに、新しいランダム変換が適用されます。こうすることで、モデルは毎回、これまでに見たものと少しずつ異なる画像を見ることになります。この「学習データの追加的なばらつき」が、新しいデータに対する性能向上に役立ちます。

ただし、すべての変換があらゆる問題に有効というわけではない点に注意が必要です。最も重要なのは、使用する変換がクラスを混同させないことです。たとえば、[数字認識モデル](https://www.kaggle.com/c/digit-recognizer)を学習している場合、画像を回転させると「9」と「6」が混同されてしまいます。最終的に有効なデータ拡張を見つける最善の方法は、他の多くの機械学習問題と同様、「試してみること」です。

### 例 
### Step 1 - データ拡張によるトレーニング(Training with Data Augmentation)

Kerasでは、データ拡張を行う方法が2つあります。
1つ目は、[ImageDataGenerator](https://www.tensorflow.org/api_docs/python/tf/keras/preprocessing/image/ImageDataGenerator)のような関数を使ってデータパイプラインに組み込む方法です。
2つ目は、Kerasの前処理レイヤー（**preprocessing layers**）を使ってモデル定義の中に組み込む方法です。ここでは2つ目の方法を採用します。
この方法の主な利点は、画像変換がCPUではなくGPU上で実行されるため、学習が高速化する可能性があることです。

この演習では、レッスン1の分類器をデータ拡張によって改善する方法を学びます。
下記コードでデータパイプラインを設定します。
```python
# Imports
import os, warnings
import matplotlib.pyplot as plt
from matplotlib import gridspec

import numpy as np
import tensorflow as tf
from tensorflow.keras.preprocessing import image_dataset_from_directory

# Reproducability
def set_seed(seed=31415):
    np.random.seed(seed)
    tf.random.set_seed(seed)
    os.environ['PYTHONHASHSEED'] = str(seed)
    #os.environ['TF_DETERMINISTIC_OPS'] = '1'
set_seed()

# Set Matplotlib defaults
plt.rc('figure', autolayout=True)
plt.rc('axes', labelweight='bold', labelsize='large',
       titleweight='bold', titlesize=18, titlepad=10)
plt.rc('image', cmap='magma')
warnings.filterwarnings("ignore") # to clean up output cells


# Load training and validation sets
ds_train_ = image_dataset_from_directory(
    '../input/car-or-truck/train',
    labels='inferred',
    label_mode='binary',
    image_size=[128, 128],
    interpolation='nearest',
    batch_size=64,
    shuffle=True,
)
ds_valid_ = image_dataset_from_directory(
    '../input/car-or-truck/valid',
    labels='inferred',
    label_mode='binary',
    image_size=[128, 128],
    interpolation='nearest',
    batch_size=64,
    shuffle=False,
)

# Data Pipeline
def convert_to_float(image, label):
    image = tf.image.convert_image_dtype(image, dtype=tf.float32)
    return image, label

AUTOTUNE = tf.data.experimental.AUTOTUNE
ds_train = (
    ds_train_
    .map(convert_to_float)
    .cache()
    .prefetch(buffer_size=AUTOTUNE)
)
ds_valid = (
    ds_valid_
    .map(convert_to_float)
    .cache()
    .prefetch(buffer_size=AUTOTUNE)
)
```

### Step 2 - モデルの定義(Define Model)
データ拡張の効果を示すため、Tutorial 1のモデルにいくつかの簡単な変換を追加します。

```python
from tensorflow import keras
from tensorflow.keras import layers
# these are a new feature in TF 2.2
from tensorflow.keras.layers.experimental import preprocessing


pretrained_base = tf.keras.models.load_model(
    '../input/cv-course-models/cv-course-models/vgg16-pretrained-base',
)
pretrained_base.trainable = False

model = keras.Sequential([
    # Preprocessing
    preprocessing.RandomFlip('horizontal'), # 左右反転
    preprocessing.RandomContrast(0.5), # 最大50%のコントラスト変化
    # Base
    pretrained_base,
    # Head
    layers.Flatten(),
    layers.Dense(6, activation='relu'),
    layers.Dense(1, activation='sigmoid'),
])
```

### Step 3 - トレーニングと評価(Train and Evaluate)
それでは学習を開始します。

```python
model.compile(

)

history = model.fit(
    ds_train,
    validation_data=ds_valid,
    epochs=30,
    verbose=0,
)
```

### 3）前処理レイヤーを追加する
次に、演習5で作成したものに似たカスタム畳み込みネットワークでデータ拡張を使用します。
データ拡張は実質的にデータセットのサイズを増やすため、過学習のリスクをそれほど高めることなく、モデルの容量（表現力）を大きくすることができます。

以下の前処理レイヤーをモデルに追加してください。

* preprocessing.RandomContrast(factor=0.10)
* preprocessing.RandomFlip(mode='horizontal')
* preprocessing.RandomRotation(factor=0.10)
```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.InputLayer(input_shape=[128, 128, 3]),
    
    # Data Augmentation
    preprocessing.RandomContrast(factor=0.10),   # ←追加
    preprocessing.RandomFlip(mode='horizontal'), # ←追加
    preprocessing.RandomRotation(factor=0.10),   # ←追加
    
    # Block One
    layers.BatchNormalization(renorm=True),
    layers.Conv2D(filters=64, kernel_size=3, activation='relu', padding='same'),
    layers.MaxPool2D(),

    # Block Two
    layers.BatchNormalization(renorm=True),
    layers.Conv2D(filters=128, kernel_size=3, activation='relu', padding='same'),
    layers.MaxPool2D(),

    # Block Three
    layers.BatchNormalization(renorm=True),
    layers.Conv2D(filters=256, kernel_size=3, activation='relu', padding='same'),
    layers.Conv2D(filters=256, kernel_size=3, activation='relu', padding='same'),
    layers.MaxPool2D(),

    # Head
    layers.BatchNormalization(renorm=True),
    layers.Flatten(),
    layers.Dense(8, activation='relu'),
    layers.Dense(1, activation='sigmoid'),
])
```

### まとめ(Conclusion)
データ拡張は、畳み込みネットワークに限らず、さまざまな種類のニューラルネットワークモデルの学習を改善するための、強力で広く使われている手法です。どのような問題であっても原則は同じです。データが十分でない場合には、「疑似データ」を追加することでその不足を補うことができます。データ拡張を試行錯誤することは、自分のデータがどこまで性能を伸ばせるかを探る優れた方法です。
