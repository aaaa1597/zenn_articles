## AIの公平性
4つの異なる公平性について学びましょう。クレジットカードの申し込みを判断するように訓練されたおもちゃのモデルを評価しましょう。

## 理論編
### はじめに(Introduction)
機械学習（ML）モデルにおいて「公平さ」をどのように定義するかには、さまざまな方法があります。例えば、クレジットカードの申請を承認（または拒否）するモデルを扱っているとしましょう。このとき、次のどちらが公平と言えるでしょうか。

* 男女間で承認率が同じであれば公平なのか
* それとも、データセットから性別を削除してモデルから見えないようにする方がよいのか

このチュートリアルでは、モデルが公平かどうかを判断するための 4つの基準 を紹介します。その後、実際にコードを実行して複数のモデルを訓練し、それぞれの公平性を分析する演習に取り組みます。（このコースの他のレッスンと同様に、事前のコーディング経験は必要ありません。）

### 4つの公平性基準
ここで紹介する4つの基準は出発点として有用ですが、公平性を定式化する方法は他にも存在することに注意してください。興味があれば、[さらに調べてみる](https://arxiv.org/pdf/1710.03184)ことをおすすめします。

ここでは、ある結果を受け取る人を選択するモデルを想定します。例えば、ローンを承認する人を選ぶ、大学への合格者を選ぶ、就職機会を提供する人を選ぶといったケースです。
(つまり、顔認識やテキスト生成のようなタスクを行うモデルはここでは扱いません。)

#### 1. 人口統計的パリティ/統計的パリティ(Demographic parity/statistical parity)

**人口統計的パリティ**(**Demographic parity**) とは、モデルによって選ばれた人の構成が、応募者全体のグループ割合と一致している場合に公平とする考え方です。

> 例：
ある非営利団体が国際会議を開催し、20,000人が参加登録しました。
主催者は、その中から 興味深い講演をしてくれそうな100人 を選ぶ機械学習モデルを作成しました。参加者の 50% が女性（20,000人中10,000人） であるため、選ばれる講演候補者も 50%を女性 にするようにモデルを設計しました。

#### 2. 機会均等(Equal opportunity)

機会均等(**Equal opportunity**) の公平性は、「本来選ばれるべき人（positives）」のうち、モデルが正しく選んだ割合が すべてのグループで同じ になることを求めます。この割合は True Positive Rate（TPR） または 感度（sensitivity） と呼ばれます。

> 例：
医師が、重い病気を発症するリスクのある患者を特定するツールを使用するとします。(これは医師の診断を補助する セカンドオピニオン として使われます。)
このツールは、すべての人口統計グループに対して 同じTPRを持つように設計 されています。

#### 3. 正確度の平等(Equal accuracy)

別の考え方として、各グループで **同じ精度**(**equal accuracy**) を持つことを確認する方法があります。つまり、拒否されるべき人が拒否される、承認されるべき人が承認されるという 正しい分類の割合 が、すべてのグループで同じになるべきという考え方です。例えば、あるグループで 98%の精度 なら、他のグループでも 98%の精度 であるべきという考え方です。
> 例：
銀行がローン承認のためにモデルを使用しているとします。モデルは すべての人口統計グループで同じ精度になるよう設計 されています。これにより、本来拒否すべき人を承認してしまう(銀行にも申請者にも損失)、本来承認すべき人を拒否してしまう(申請者の機会損失、銀行の収益減)、といった問題を防ぐことができます。

---

#### 4. グループ非認識/無知による公平性(Group unaware/"Fairness through unawareness")
この方法では、グループ属性の情報をデータセットから完全に削除 します。例えば、性別データを削除することで、モデルを異なる性別グループに対して公平なものにすることができます。同様に、人種や年齢に関する情報も削除します。

> 例：
このアプローチを実際に適用する際の難しさの 1 つは、グループ メンバーシップ データのプロキシを識別して削除する際に注意する必要があることです。例えば、人種ごとに居住地域が分かれている都市では、郵便番号(zip code) が人種の強い代理変数(proxy)になります。つまり、人種データを削除する場合は、郵便番号も削除する必要があります。そうしないと、機械学習モデルがデータから個人の人種を推測できてしまう可能性があります。さらに、グループに気づかない公平性は、歴史的偏見に対する良い解決策にはならない可能性が高いです。

### 例(Example)

例として、精度80%(10人中8人を正しく分類してたため)、真陽性率(TPR) 83%(6人のpositiveのうち5人を正しく検出してたため)のモデルを考えます。

##### Group unaware

これは 混同行列からは判断できません。
データセットからグループ情報を削除しているかどうかに関係します。

これらの例をよく観察して、それぞれの公平性の違いを直感的に理解してください。
例えば、Group A の応募者が Group B の2倍だったらどうなるでしょうか？

また、重要なポイントとして、どの例も複数の公平性基準を同時に満たしていません。
たとえば、人口統計上の平等の例は、正確性の平等や機会均等の要件を満たしていません。
さあ、今すぐこのことを検証してみましょう。実際には、モデルを複数の種類の公平性に合わせて最適化することはできません。詳しくは、「[機械公平性の不可能性定理](https://arxiv.org/abs/2007.06024)」をご覧ください。では、1 つしか満たせない場合、どの公平性基準を選択すればよいのでしょうか。ほとんどの倫理的な質問と同様に、正しい答えは通常は単純ではなく、基準の選択はチーム全員が参加する長い会話になるはずです。

実際のプロジェクトでは、データははるかに大きくなります。その場合でも、混同行列はモデルのパフォーマンスを分析するための有用なツールとなります。ただし、注目すべき重要な点は、現実世界のモデルは通常、公平性の定義を完全に満たすことは期待できないということです。たとえば、公平性の指標として「人口統計上の平等」が選択され、モデルが 50% の男性を選択することを目標としている場合、最終的なモデルは最終的に 50% に近いものの正確に 50% ではない割合 (48% や 53% など) を選択する可能性があります。

## 実践編
上述の理論編では、機械学習モデルの**公平性（fairness）** を測定するさまざまな方法について学びました。この実践編では、クレジットカード申請を承認（approve）または拒否（deny） するいくつかのモデルを訓練し、その公平性を分析します。コーディングに慣れていなくても心配はいりません。この演習ではプログラミングの知識がないことを前提 にしています。

ここでは、クレジットカード申請者が提出した情報をまとめた**人工的（synthetic）データセット**を使用します。

次のコードセルを変更せずに実行すると、クレジットカードの承認・拒否を決めるシンプルなモデルが学習されます。出力では、そのモデルの性能が表示されます。
```python
from sklearn import tree
from sklearn.metrics import confusion_matrix, ConfusionMatrixDisplay
import matplotlib.pyplot as plt

# Train a model and make predictions
model_baseline = tree.DecisionTreeClassifier(random_state=0, max_depth=3)
model_baseline.fit(X_train, y_train)
preds_baseline = model_baseline.predict(X_test)

# Function to plot confusion matrix
def plot_confusion_matrix(estimator, X, y_true, y_pred, display_labels=["Deny", "Approve"],
                          include_values=True, xticks_rotation='horizontal', values_format='',
                          normalize=None, cmap=plt.cm.Blues):
    cm = confusion_matrix(y_true, y_pred, normalize=normalize)
    disp = ConfusionMatrixDisplay(confusion_matrix=cm, display_labels=display_labels)
    return cm, disp.plot(include_values=include_values, cmap=cmap, xticks_rotation=xticks_rotation,
                     values_format=values_format)

# Function to evaluate the fairness of the model
def get_stats(X, y, model, group_one, preds):
        
    y_zero, preds_zero, X_zero = y[group_one==False], preds[group_one==False], X[group_one==False]
    y_one, preds_one, X_one = y[group_one], preds[group_one], X[group_one]
    
    print("Total approvals:", preds.sum())
    print("Group A:", preds_zero.sum(), "({}% of approvals)".format(round(preds_zero.sum()/sum(preds)*100, 2)))
    print("Group B:", preds_one.sum(), "({}% of approvals)".format(round(preds_one.sum()/sum(preds)*100, 2)))
    
    print("\nOverall accuracy: {}%".format(round((preds==y).sum()/len(y)*100, 2)))
    print("Group A: {}%".format(round((preds_zero==y_zero).sum()/len(y_zero)*100, 2)))
    print("Group B: {}%".format(round((preds_one==y_one).sum()/len(y_one)*100, 2)))
    
    cm_zero, disp_zero = plot_confusion_matrix(model, X_zero, y_zero, preds_zero)
    disp_zero.ax_.set_title("Group A")
    cm_one, disp_one = plot_confusion_matrix(model, X_one, y_one, preds_one)
    disp_one.ax_.set_title("Group B")
    
    print("\nSensitivity / True positive rate:")
    print("Group A: {}%".format(round(cm_zero[1,1] / cm_zero[1].sum()*100, 2)))
    print("Group B: {}%".format(round(cm_one[1,1] / cm_one[1].sum()*100, 2)))
    
# Evaluate the model    
get_stats(X_test, y_test, model_baseline, X_test["Group"]==1, preds_baseline)
```

### 1) 公平性の種類
チュートリアルでは、次の3種類の公平性について学びました。
- 人口統計的パリティ(Demographic parity): 承認された申請者の中で、どちらのグループがより多く含まれているかという観点の公平性です。(このデータでは申請者の約50%がGroup A、約50%がGroup Bです)
- 正解率の平等(Equal accuracy): どちらのグループの申請者がより正確に分類されているかという公平性です。
- 機会の平等(Equal opportunity): 真陽性率(True Positive Rate)が高いグループはどちらか、という観点の公平性です。

![](https://storage.googleapis.com/zenn-user-upload/a697d59aa69d-20260310.png)
解答: すべての公平性基準において、Group B に有利な偏りが見られます。申請者の79.01% が Group Bです。もし 人口統計的パリティ(demographic parity)を公平性の基準とするならば、公平であるためには Group B が申請者の50%である必要があります。また、このモデルは Group B の申請者に対してわずかに高い精度を示しています(正解率は Group B が95.02%、Group A が94.56%)。さらに、真陽性率(True Positive Rate)は Group B で非常に高く、Group B(98.03%) に対して Group A(77.23%) となっています。言い換えると、Group B の場合、本来承認されるべき人のほとんどが実際に承認されています。一方で Group A の場合、本来承認されるべきであっても、実際に承認される確率はかなり低くなっています。

モデルを視覚化するには、次のコード セルを変更せずに実行します。
```python
def visualize_model(model, feature_names, class_names=["Deny", "Approve"], impurity=False):
    plot_list = tree.plot_tree(model, feature_names=feature_names, class_names=class_names, impurity=impurity)
    [process_plot_item(item) for item in plot_list]

def process_plot_item(item):
    split_string = item.get_text().split("\n")
    if split_string[0].startswith("samples"):
        item.set_text(split_string[-1])
    else:
        item.set_text(split_string[0])

plt.figure(figsize=(20, 6))
plot_list = visualize_model(model_baseline, feature_names=X_train.columns)
```

このフローチャートは、モデルがどのように意思決定をしているかを示しています。
- Group <= 0.5 は、申請者がどのグループに属しているかをチェックします。申請者がグループ A に属している場合、Group <= 0.5 は真になります。
- 「Income <= 80210.5」のようなエントリは、申請者の収入を確認します。

フローチャートに沿って、上から始めて、申請者の詳細に応じて経路を辿ります。分岐点で条件が真であれば、下へ進み、左の分岐に進みます。偽であれば、右の分岐に進みます。

例えば、グループBの75,000ドルの収入がある申請者を考えてみましょう。
この場合の流れは次のようになります。

- フローチャートの上から始めます。申請者の収入は 75,000 ドルなので、Income <= 80210.5 が真となり、左に移動します。
- 次に、所得を再度確認します。所得 <= 71909.5 は偽なので、右に進みます。

- 最後に確認するのは、申請者がどのグループに属しているかです。申請者はグループBに属しているため、Group <= 0.5はFalseとなり、右側のモデルで申請者を承認する決定が下されます。

### 2) ベースラインモデルの理解
この可視化をもとに考えると、このモデルにおける不公平性の原因の一つは何でしょうか？

ヒント: 先ほどの例の申請者について考えてみてください。ただし、Group B → Group A
に変更し、それ以外の特徴はすべて同じにします。このわずかに異なる申請者は、モデルによって承認されるでしょうか？それとも拒否されるでしょうか？

### 3) 公平性の種類（パート2）
この新しいモデルは、最初のモデルと比べて、**人口統計の同一性(demographic parity)**、**精度の同等性(equal accuracy)**、**機会の平等性(equal opportunity)** の観点でどのように違うでしょうか？答えを考えた後、次のコードセルを実行してください。
![](https://storage.googleapis.com/zenn-user-upload/9307a9bdae5a-20260310.png)
人口統計的パリティ(demographic parity)を考えると、新しいモデルは依然としてグループBに有利なバイアスを持っていますが、元のモデルよりは少し公平になっています。
しかし、精度の平等(equal accuracy)や機会の平等(equal opportunity) のいずれかを考えると、今度はモデルがグループAに有利なバイアスを持つことになります。また、モデル全体の精度が低下していることにも注意が必要です。つまり、どちらのグループに対しても、モデルはわずかに精度の低い判断をするようになっています。

あなたは3つ目のモデルを試すことにしました。
今回の目標は、承認された申請者の中で、各グループが同じ割合になるようにすることです。(これはgroup thresholdsという方法の一例です)

# Change the value of zero_threshold to hit the objective

### 4) 公平性の種類（パート3）

この最後のモデルは、最初のモデルと比べて、**人口統計の同一性(demographic parity)**、**精度の同等性(equal accuracy)**、**機会の平等性(equal opportunity)** の観点でどのように違うでしょうか？
![](https://storage.googleapis.com/zenn-user-upload/f07aef328ffe-20260310.png)
答え：このモデルは、承認された申請者の集団において、各グループからほぼ同じ割合の人が含まれるようになっています。もし人口統計的パリティ(demographic parity)を重視するのであれば、このモデルは最初の2つのモデルよりもはるかに公平です。精度は各グループでほぼ同じですが、各グループにおける全体的な精度は大きく低下しています。

機会の平等（equal opportunity）の観点からこのモデルを調べると、モデルはグループAに有利なバイアスを持っています。つまり、本来承認されるべきグループAの個人は全員承認されている一方で、本来承認されるべきグループBの個人のうち承認されているのは63%にとどまっています。（これは最初のモデルと似た状況ですが、優遇されるグループが入れ替わっています。つまり、最初のモデルでは、本来承認されるべきグループBのほぼすべての個人がモデルによって承認されていました。）

この演習は、公平性の異なる概念を探索し、"ある公平性を最適化すると、別の公平性とのトレードオフが生じる"ことを示すための短い例です。ここでは主にモデルの学習に焦点を当てましたが、実際には

- データ収集
- モデル作成
- 最終製品の公開

までの**すべての段階**を注意深く検討する必要があります。

例えばデータをよく見ると、Group B の人々は平均的に 収入が高い、家や車を所有している可能性が高い ことが分かります。この事実を理解することは、どの公平性基準を採用すべきか、公平性をどう実現するか を考えるうえで非常に重要です。(例えば、データに含まれる歴史的バイアスを取り除かずに、各グループの正解率を同じにしようとするという方法は、望ましくないアプローチである可能性があります。)
