### WITH ... AS
**AS** 単体でも、クエリの結果を整理する便利な方法ですが、**WITH** と組み合わせるとさらに強力になります。これを「共通テーブル式（common table expression）」と呼びます。

共通テーブル式（**common table expression** = CTE）とは、クエリの中で一時的に作成されるテーブルのことです。CTE を使うと、クエリを読みやすいまとまりに分割でき、その CTE に対してさらにクエリを書くことができます。

例えば、pets テーブルを使って、特に高齢の動物について調べたいとします。その場合、まず5歳より上の動物だけを含む CTE を次のように作成できます。
![](https://storage.googleapis.com/zenn-user-upload/3fc116e0d9d9-20260214.png)

上の未完成のクエリは何も返しませんが、Seniors という名前の CTE を作成しており、その後のクエリ内で参照することができます。

その後、CTE から必要な情報を取得してクエリを完成させます。以下の完全なクエリでは、まず CTE を作成し、その中からすべての ID を返しています。

![](https://storage.googleapis.com/zenn-user-upload/1ee80ad60c48-20260214.png)

CTE を使わずに同じことを行うことも可能ですが、これが非常に長いクエリの最初の部分だった場合、CTE を使わないと流れを追うのがかなり難しくなります。

また重要なのは、CTE は作成したクエリの中でしか存在しないという点です。後続の別のクエリから参照することはできません。つまり、CTE を使うクエリは必ず次の2つの部分に分かれます：
(1) まず CTE を作成する
(2) 次に、その CTE を利用するクエリを書く

### 例：ビットコインの取引は月ごとに何件あるか？

今回は CTE を使って、ビットコインのトランザクションデータセット全期間における、各日の取引件数を求めます。

transactions テーブルを調べます。以下は最初の数行の表示です。

```python

client.list_rows(table, max_results=5).to_dataframe()
```

block_timestamp 列には各取引の日付が DATETIME 形式で含まれているため、**DATE()** コマンドを使って DATE 形式に変換します。

その処理を CTE で行い、続くクエリで各日付ごとの取引件数をカウントし、日付の古い順に並べ替えます。

以下がそのクエリです：

```sql
WITH time AS 
(
    SELECT DATE(block_timestamp) AS trans_date
    FROM `bigquery-public-data.crypto_bitcoin.transactions`
)
SELECT COUNT(1) AS transactions,
       trans_date
FROM time
GROUP BY trans_date
ORDER BY trans_date
```

クエリを実行すると、次のような結果が得られます。

transactions  &nbsp;&nbsp;  trans_date
1             &nbsp;&nbsp;  2009-01-03
14            &nbsp;&nbsp;  2009-01-09
61            &nbsp;&nbsp;  2009-01-10
93            &nbsp;&nbsp;  2009-01-11
101           &nbsp;&nbsp;  2009-01-12

日付順に並んでいるので、結果をそのままプロットすることで、このデータセット全期間にわたる1日あたりのビットコイン取引数を簡単に可視化できます。

```python
transactions_by_date.set_index('trans_date').plot()
```

![](https://storage.googleapis.com/zenn-user-upload/272dcc5349f9-20260214.png)

このように、共通テーブル式（CTE）を使うことで、データの前処理の多くを SQL 側で行うことができます。特に BigQuery の場合、Pandas で処理するよりもはるかに高速なので、大きなメリットがあります。

#### 5) クエリを書いてみよう

いよいよ、より高度なクエリに挑戦します。データセット内の各時間帯（1日の各時刻）について、乗車回数と平均速度を示すクエリを書いてください。

結果には次の3列を含めてください：

* hour_of_day — trip_start_timestamp から抽出した「時」。この列でソートすること。
* num_trips — 各時間帯に開始された乗車の総数（例：6時〜7時の間に開始された全ての乗車数）
* avg_mph — その時間帯に開始された乗車の平均速度（マイル毎時）

平均速度（mph）は次の式で計算します：

```
3600 * SUM(trip_miles) / SUM(trip_seconds)
```
（3600 は秒を時間に変換するために使います。）

以下の条件を満たすデータのみに限定してください：

* trip_start_timestamp > 2016-01-01 かつ < 2016-04-01
* trip_seconds > 0 かつ trip_miles > 0

共通テーブル式（CTE）を使って、条件を満たす乗車のみを先に抽出してください。このデータセットは非常に大きいため、CTE では最終出力に必要な列だけを選択してください（最終的な集計は、CTEの下の SELECT で行います）。
