## Group By, Having & Count
SQLクエリから直接、より興味深い洞察を得ることができます。

## 理論編
### はじめに(Introduction)
生データを選択できるようになったので、次はデータをグループ化し、そのグループ内で件数を数える方法を学びましょう。これにより、次のような質問に答えられるようになります。

* 私たちの店では、果物は種類ごとにいくつ売れましたか？
* その動物病院では、何種類の動物を診察しましたか？

そのために、3つの新しいテクニックを学びます：GROUP BY、HAVING、そして COUNT() です。今回も、ペットに関する架空のテーブルを使います。

### COUNT()

COUNT() は、その名前のとおり「数」を返します。列名を渡すと、その列に含まれるエントリの数を返します。

たとえば、pets テーブルの ID 列に対して COUNT() を SELECT すると、テーブルには ID が4つあるので、4が返されます。

![](https://storage.googleapis.com/zenn-user-upload/b1a1e56ef7d0-20260212.png)

COUNT() は集約関数（aggregate function）の一例で、複数の値を受け取り、1つの値を返します。（他の例としては、SUM()、AVG()、MIN()、MAX() などがあります。）
上の図からわかるように、集約関数を使うと f0__ のような少し分かりにくい列名が付くことがあります。このチュートリアルの後半で、より分かりやすい名前に変更する方法を学びます。

### GROUP BY

GROUP BY は、1つ以上の列名を指定し、その列で同じ値を持つ行を1つのグループとして扱います。そして、COUNT() のような集約関数をそのグループごとに適用します。

たとえば、pets テーブルにそれぞれの動物が何匹いるかを知りたいとします。このとき、Animal 列で GROUP BY し、各グループの ID を COUNT() すればよいです。

![](https://storage.googleapis.com/zenn-user-upload/c27479f3ebcb-20260212.png)

このクエリは、3行（動物の種類ごとに1行）を返します。
pets テーブルには、ウサギが1匹、犬が1匹、猫が2匹いることが分かります。

---

GROUP BY ... HAVING

HAVING は GROUP BY と組み合わせて使い、特定の条件を満たさないグループを除外するために使います。
たとえば、次のクエリは、ID が1つより多いグループだけを含みます。

![](https://storage.googleapis.com/zenn-user-upload/8f5cc04b3677-20260212.png)

指定された条件を満たすグループは1つだけなので、このクエリは1行だけのテーブルを返します。

```sql
# 10件より多く返信を受けたコメントを選択するクエリ
query_popular = """
                SELECT parent, COUNT(id)
                FROM `bigquery-public-data.hacker_news.full`
                GROUP BY parent
                HAVING COUNT(id) > 10
                """
```

クエリを実行して、結果を pandas DataFrame に保存します。

### エイリアス（別名）とその他の改善
クエリをより良くするためのヒントをいくつか紹介します。
* COUNT(id) によってできた列名は f0__ でした。これはあまり分かりやすい名前ではありません。
  集約のあとに AS NumPosts を追加すれば、列名を変更できます。これをエイリアス（aliasing）と呼びます。詳しくは後のレッスンで扱います。

* COUNT() の中に何を書けばよいか迷ったら、COUNT(1) と書くこともできます。これは各グループの行数を数えるという意味です。
  他の列を参照しないことが明確なので読みやすく、またスキャンするデータ量も少なくて済むため、高速でデータ使用量も少なくなります。

これらを使って、クエリを書き直すと次のようになります。

```sql
# エイリアスと可読性を改善したバージョン
query_improved = """
                 SELECT parent, COUNT(1) AS NumPosts
                 FROM `bigquery-public-data.hacker_news.full`
                 GROUP BY parent
                 HAVING COUNT(1) > 10
                 """

safe_config = bigquery.QueryJobConfig(maximum_bytes_billed=10**10)
query_job = client.query(query_improved, job_config=safe_config)

# API request - run the query, and convert the results to a pandas DataFrame
improved_df = query_job.to_dataframe()

# Print the first five rows of the DataFrame
improved_df.head()
```

### GROUP BY を使う際の注意

GROUP BY は、集約関数（COUNT() など）をどのように適用するかをSQLに伝えるものなので、集約関数なしで使うのは意味がありません。

また、GROUP BY 句がある場合、すべての変数は次のどちらかに含まれていなければなりません：

1. GROUP BY に指定されている
2. 集約関数の中に渡されている

次のクエリを見てください。動作するクエリのサンプルです。

```sql
query_good = """
             SELECT parent, COUNT(id)
             FROM `bigquery-public-data.hacker_news.full`
             GROUP BY parent
             """
```

ここには2つの変数があります：parent と id。

* parent は GROUP BY parent でグループ化されています。
* id は COUNT(id) という集約関数に渡されています。

一方、下記のクエリは動きません。author 列が集約関数にも GROUP BY にも渡されていないからです。

```sql
query_bad = """
            SELECT `by` AS author, parent, COUNT(id)
            FROM `bigquery-public-data.hacker_news.full`
            GROUP BY parent
            """
```

このような誤りをすると、
「SELECT list expression references column (列名) which is neither grouped nor aggregated at」
というエラーメッセージが表示されます。

なお、このクエリでは `by` 列がバッククォートで囲まれています。これは BY が GROUP BY などで使われる予約語だからです。
BigQuery では、予約語を識別子として使う場合、バッククォートで囲む必要があります。
また、AS author としてエイリアスを付けることで、以降の参照が読みやすくなっています。
