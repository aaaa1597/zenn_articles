## 時系列を特徴とする
遅延埋め込みを使用して過去から未来を予測します。

## 理論編
### 時系列(Serial Dependence)依存関係とは？
これまでのレッスンでは、時間インデックスから直接作れる特徴量、つまり「時間依存的な性質」として最も簡単にモデル化できる時系列の特性について扱ってきました。しかし、時系列の中には「系列依存的な性質」として、つまり目的変数の過去の値を特徴量として用いることでしかモデル化できない特性もあります。このような時系列の構造は、時間に対してプロットしただけでは明確に見えないことがありますが、過去の値に対してプロットすると、その構造がはっきりと現れます（下図参照）。

![](https://storage.googleapis.com/zenn-user-upload/5ca534f35b69-20260228.png)*これら2つの系列は、時間依存ではなく系列依存を持っています。右側の点は (時点 t-1 の値, 時点 t の値) を座標としています。*

トレンドや季節性の場合、上図左側のようなプロットに曲線を当てはめるモデルを学習しました。つまりモデルは時間依存を学習していました。このレッスンの目的は、上図右側のようなプロットに曲線を当てはめるモデルを学習させること、つまり系列依存を学習させることです。

### 循環(Cycles)
系列依存が現れる特に一般的な形の一つが「**循環（サイクル）**」です。循環とは、ある時点の値が過去の値にどのように依存しているかに関連する、増加と減少のパターンのことです。ただし、必ずしも時間そのもの（特定の日付など）に依存しているわけではありません。循環的な振る舞いは、自らに影響を及ぼすシステムや、その反応が時間を通じて持続するシステムに特徴的です。経済、感染症、動物の個体数、火山噴火などの自然現象は、しばしば循環的な振る舞いを示します。

![](https://storage.googleapis.com/zenn-user-upload/b489640dd166-20260228.png)*循環的な挙動を示す4つの時系列。*

循環が季節性と異なる点は、季節のように必ずしも時間に依存していないことです。循環で重要なのは特定の日付そのものではなく、直近で何が起こったかです。時間から（少なくとも相対的に）独立しているため、循環的な挙動は季節性よりもはるかに不規則になることがあります。

### 遅延系列と遅延プロット

時系列における系列依存（循環など）の可能性を調べるには、その系列の「遅延付き」コピーを作る必要があります。時系列にラグを取るとは、値を1ステップ以上前方(あるいは後方)にずらすことを意味します。いずれの場合も、ラグ系列の観測値は、より後の時点に起こったかのように見える効果があります。

これはアメリカの月次失業率 (y) と、その1期ラグおよび2期ラグ系列（それぞれ y_lag_1 と y_lag_2）を示しています。ラグ系列の値が時間的に前方へずれていることに注目してください。
```python
import pandas as pd

# Federal Reserve dataset: https://www.kaggle.com/federalreserve/interest-rates
reserve = pd.read_csv(
    "../input/ts-course-data/reserve.csv",
    parse_dates={'Date': ['Year', 'Month', 'Day']},
    index_col='Date',
)

y = reserve.loc[:, 'Unemployment Rate'].dropna().to_period('M')
df = pd.DataFrame({
    'y': y,
    'y_lag_1': y.shift(1),
    'y_lag_2': y.shift(2),    
})

df.head()
```

| 1954-09 | 6.1 | 6.0 | 5.8 |

| 1954-11 | 5.3 | 5.7 | 6.1 |


時系列にラグを取ることで、予測しようとしている値と過去の値を同じ行（同時点）に並べることができます。そのため、ラグ系列は系列依存をモデル化するための有用な特徴量になります。アメリカの失業率を予測するには、y_lag_1 や y_lag_2 を特徴量として目的変数 y を予測できます。つまり、直近2か月の失業率に基づいて将来の失業率を予測することになります。

### 遅延プロット(Lag plots)
**遅延プロット**とは、時系列の値をそのラグ値に対してプロットしたものです。時系列の系列依存は、遅延プロットを見ることで明らかになることがよくあります。アメリカの失業率の遅延プロットから、現在の失業率と過去の失業率の間に強く、しかもほぼ線形な関係があることが分かります。

![](https://storage.googleapis.com/zenn-user-upload/e78a42b98be2-20260301.png)*自己相関を示した米国失業率の遅延プロット。*

系列依存の最も一般的な指標は「**自己相関（autocorrelation）**」で、これは時系列とそのラグとの相関のことです。アメリカの失業率は、ラグ1で0.99、ラグ2で0.98といった自己相関を持っています。

### ラグ値の選択
特徴量としてラグを選ぶ際、自己相関が大きいすべてのラグを含めるのが有効とは限りません。例えばアメリカの失業率では、ラグ2の自己相関はラグ1から引き継がれた「減衰した」情報にすぎない可能性があります。ラグ2が新しい情報を含んでいないなら、すでにラグ1を使っている場合、それを含める理由はありません。

**偏自己相関**は、それ以前のすべてのラグの影響を考慮したうえでの相関、いわばそのラグがどれだけ「新しい」相関を追加しているかを示します。偏自己相関をプロットすることで、どのラグを特徴量として使うべきか判断できます。

In the figure below, lag 1 through lag 6 fall outside the intervals of "no correlation" (in blue), so we might choose lags 1 through lag 6 as features for US Unemployment. (Lag 11 is likely a false positive.)
下図では、ラグ1からラグ6が「無相関」の区間（青）を外れているため、アメリカの失業率にはラグ1から6を特徴量として選ぶことが考えられます。（ラグ11はおそらく偽陽性です。）

![](https://storage.googleapis.com/zenn-user-upload/20da99f6b5ad-20260301.png)*相関がない 95% 信頼区間を持つ、ラグ 12 までの米国失業率の部分自己相関。*

このようなプロットはコレログラムと呼ばれます。コレログラムは、ラグ特徴量にとって、フーリエ特徴量に対するピリオドグラムのような役割を果たします。

最後に注意すべき点は、自己相関や偏自己相関は線形依存を測る指標だということです。

なぜなら、実世界の時系列には非線形依存が多く含まれることがあるため、ラグ特徴量を選ぶ際は遅延プロットを確認したり、[相互情報量](https://www.kaggle.com/code/ryanholbrook/mutual-information)のようなより一般的な依存度指標を使うのが望ましいです。サンスポット系列には非線形な依存を持つラグがあり、自己相関だけでは見落としてしまう可能性があります。

![](https://storage.googleapis.com/zenn-user-upload/3a526e76aa94-20260301.png)

このような非線形関係は、線形に変換するか、あるいは適切なアルゴリズムによって学習させることができます。

### 例 ― Flu Trends
Flu Trends データセットには、2009年から2016年までの週ごとのインフルエンザによる医師受診件数が記録されています。私たちの目標は、今後数週間のインフルエンザ患者数を予測することです。

ここでは2つのアプローチを取ります。
1つ目は、ラグ特徴量を使って受診件数を予測します。
2つ目は、Google Trends によって取得されたインフル関連検索語の時系列のラグを用いて受診件数を予測します。

```python
def lagplot(x, y=None, lag=1, standardize=False, ax=None, **kwargs):
    from matplotlib.offsetbox import AnchoredText
    x_ = x.shift(lag)
    if standardize:
        x_ = (x_ - x_.mean()) / x_.std()
    if y is not None:
        y_ = (y - y.mean()) / y.std() if standardize else y
    else:
        y_ = x
    corr = y_.corr(x_)
    if ax is None:
        fig, ax = plt.subplots()
    scatter_kws = dict(
        alpha=0.75,
        s=3,
    )
    line_kws = dict(color='C3', )
    ax = sns.regplot(x=x_,
                     y=y_,
                     scatter_kws=scatter_kws,
                     line_kws=line_kws,
                     lowess=True,
                     ax=ax,
                     **kwargs)
    at = AnchoredText(
        f"{corr:.2f}",
        prop=dict(size="large"),
        frameon=True,
        loc="upper left",
    )
    at.patch.set_boxstyle("square, pad=0.0")
    ax.add_artist(at)
    ax.set(title=f"Lag {lag}", xlabel=x_.name, ylabel=y_.name)
    return ax


def plot_lags(x, y=None, lags=6, nrows=1, lagplot_kwargs={}, **kwargs):
    import math
    kwargs.setdefault('nrows', nrows)
    kwargs.setdefault('ncols', math.ceil(lags / nrows))
    kwargs.setdefault('figsize', (kwargs['ncols'] * 2, nrows * 2 + 0.5))
    fig, axs = plt.subplots(sharex=True, sharey=True, squeeze=False, **kwargs)
    for ax, k in zip(fig.get_axes(), range(kwargs['nrows'] * kwargs['ncols'])):
        if k + 1 <= lags:
            ax = lagplot(x, y, lag=k + 1, ax=ax, **lagplot_kwargs)
            ax.set_title(f"Lag {k + 1}", fontdict=dict(fontsize=14))
            ax.set(xlabel="", ylabel="")
        else:
            ax.axis('off')
    plt.setp(axs[-1, :], xlabel=x.name)
    plt.setp(axs[:, 0], ylabel=y.name if y is not None else x.name)
    fig.tight_layout(w_pad=0.1, h_pad=0.1)
    return fig
```

```python
def make_lags(ts, lags):
    return pd.concat(
        {
            f'y_lag_{i}': ts.shift(i)
            for i in range(1, lags + 1)
        },
        axis=1)


X = make_lags(flu_trends.FluVisits, lags=4)
X = X.fillna(0.0)
```

これまでのレッスンでは、学習データより先の将来を好きなだけ予測することができました。しかしラグ特徴量を使う場合、ラグ値が利用可能な時点までしか予測できません。例えば月曜日にラグ1を使う場合、水曜日を予測することはできません。必要なラグ1の値（火曜日）がまだ存在しないからです。

## 実践編
### はじめに(Introduction)
すべての設定を行うために、このセルを実行してください。

---
すべての商品ファミリーの売上が循環的な挙動を示すわけではなく、平均売上の系列も同様です。しかし、学用品および事務用品の売上は、トレンドや季節性だけではうまく説明できない増減パターンを示しています。この問題および次の問題では、ラグ特徴量を用いて学用品・事務用品の売上に見られる循環をモデル化します。

トレンドと季節性はいずれも系列相関（時系列依存）を生み出し、それはコレログラムやラグプロットに現れます。純粋な循環的挙動を切り分けるため、まず系列から季節性を取り除きます。

次コードで使って、学用品・事務用品の売上から季節性を除去してください。結果は `y_deseason` という変数に保存します。

```python
supply_sales = family_sales.loc(axis=1)[:, 'SCHOOL AND OFFICE SUPPLIES']
y = supply_sales.loc[:, 'sales'].squeeze()

fourier = CalendarFourier(freq='M', order=4)
dp = DeterministicProcess(
    constant=True,
    index=y.index,
    order=1,
    seasonal=True,
    drop=True,
    additional_terms=[fourier],
)
X_time = dp.in_sample()
X_time['NewYearsDay'] = (X_time.index.dayofyear == 1)

model = LinearRegression(fit_intercept=False)
model.fit(X_time, y)
y_deseason = y - model.predict(X_time)
y_deseason.name = 'sales_deseasoned'

ax = y_deseason.plot()
ax.set_title("Sales of School and Office Supplies (deseasonalized)");
```
