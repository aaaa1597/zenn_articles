## 4.モデルの検証
### 理論編
すでにモデルを作成しましたが、そのモデルはどれくらい「良い」のでしょうか？

このレッスンでは、**モデルの品質を測るための「モデルバリデーション（検証）」**について学びます。モデルの品質を測ることは、モデルを改善していくうえで最も重要な鍵となります。

#### モデルバリデーションとは？

あなたが今後つくるほぼすべてのモデルで、評価が必要です。多くの（すべてではないですが）ケースで重要となるのは、予測精度です。つまり、「そのモデルの予測は実際に起きる値にどれくらい近いのか？」ということです。

多くの人が予測精度の測り方で大きな間違いを犯します。それは、訓練データに対して予測を行い、その予測値と訓練データの正解を比較して評価するという方法です。この方法がなぜ問題なのか、そしてどう解決するのかを後で説明しますが、まずはどうやって評価値をまとめるかを考えてみましょう。

10,000件の家の予測値と実際の価格を比べると、当然良い予測も悪い予測も混じるでしょう。10,000件分の結果をすべて目で見るのは意味がありません。そこでまとめるための指標が必要です。

モデルの品質を要約する指標はいくつもありますが、まずは 平均絶対誤差（MAE: Mean Absolute Error） と呼ばれるものから始めます。まずは「誤差」という言葉を分解していきましょう。

#### 予測誤差（Error）とは

1件の家に対する誤差はこう計算します：
```
誤差 = 実際値 − 予測値
```

例えば、実際の価格が150,000ドルで、あなたの予測が100,000ドルだった場合、誤差は50,000ドルです。

MAEでは、まず誤差の「絶対値」を取ります。つまり、すべての誤差を正の値に変換します。そしてその絶対誤差の平均を取ります。これがモデル品質の指標になります。

わかりやすく言うと、
「平均すると、予測は実際の値からだいたい X だけズレている」
ということです。

#### MAE の計算方法
まずモデルを作る必要があります（下記にコードが書かれています）。

```python
# Data Loading Code Hidden Here
import pandas as pd

# Load data
melbourne_file_path = '../input/melbourne-housing-snapshot/melb_data.csv'
melbourne_data = pd.read_csv(melbourne_file_path) 
# Filter rows with missing price values
filtered_melbourne_data = melbourne_data.dropna(axis=0)
# Choose target and features
y = filtered_melbourne_data.Price
melbourne_features = ['Rooms', 'Bathroom', 'Landsize', 'BuildingArea', 
                        'YearBuilt', 'Lattitude', 'Longtitude']
X = filtered_melbourne_data[melbourne_features]

from sklearn.tree import DecisionTreeRegressor
# Define model
melbourne_model = DecisionTreeRegressor()
# Fit model
melbourne_model.fit(X, y)
```

モデルができたら、MAE(平均絶対誤差)は以下のように計算します。
```
from sklearn.metrics import mean_absolute_error

predicted_home_prices = melbourne_model.predict(X)
mean_absolute_error(y, predicted_home_prices)
```

結果（例）：
434.71594577146544

##### 「In-Sample（訓練データ内）」スコアの問題点

今計算したものは「In-Sample」スコアと呼ばれます。
つまり、モデルの作成にも評価にも同じデータを使っている状態です。
これがなぜダメなのかを説明します。

##### データ内にしか存在しない「偶然のパターン」を学習してしまう
たとえば実際の不動産市場では「ドアの色」は価格に関係していないとします。
しかし、たまたまあなたが使った訓練データでは、緑色のドアの家がすべて高額だったとします。
モデルは「緑のドア → 高い家」と学習してしまいます。

訓練データでは正しいように見えても、実際の市場ではそんなパターンは存在しないため、
新しいデータに対してはまったく使い物にならないモデルになります。

##### だから「未知のデータ」で性能を測る必要がある
実用的なモデルは未知のデータに対して予測を行います。
そのため、モデル作成に使わなかったデータで性能を測る必要があります。

その方法として一般的なのが、データの一部を訓練に使う残りを検証用（validation）として使う方法です。バリデーションデータ といいます。

#### コードで実践
scikit-learn の train_test_split を使うと簡単に分割できます。
```python
from sklearn.model_selection import train_test_split
# split data into training and validation data, for both features and target
# The split is based on a random number generator. Supplying a numeric value to
# the random_state argument guarantees we get the same split every time we
# run this script.
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state = 0)
# Define model
melbourne_model = DecisionTreeRegressor()
# Fit model
melbourne_model.fit(train_X, train_y)

# get predicted prices on validation data
val_predictions = melbourne_model.predict(val_X)
print(mean_absolute_error(val_y, val_predictions))
```
出力： 265806.91478373145

Wow!

インサンプル（訓練データ内）のMAEは 約 500ドル でしたが、バリデーションデータ（未知のデータ）では 25万ドル以上 でした。

これは、「ほぼ完璧なモデル」と「実用不可能なモデル」ほどの差があります。

参考までに、検証データ内の平均住宅価格は 110万ドル です。
つまり誤差は平均価格の約25%に相当します。

##### モデル改善はいくらでも可能
指標は低いですが、ここから改善できます：
特徴量を工夫する、別のモデルを使う、ハイパーパラメータを調整する、などなど。
