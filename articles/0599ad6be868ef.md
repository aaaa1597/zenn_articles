### Deep Learning とは？

近年の人工知能における最も目覚ましい進歩のいくつかは、ディープラーニングの分野で起こっています。
自然言語翻訳、画像認識、ゲームプレイなどは、ディープラーニングモデルが**人間レベルに迫る、あるいはそれを超える性能**を示している分野です。

では、ディープラーニングとは何でしょうか？

ディープラーニングとは、「深い（deep）」計算の積み重ねによって特徴づけられる機械学習のアプローチです。

この「計算の深さ」こそが、現実世界の難しいデータセットに含まれる複雑で階層的なパターンを分解・理解することを可能にしています。

ニューラルネットワークは、その強力さとスケーラビリティによって、ディープラーニングを代表するモデルとなっています。

**ニューラルネットワーク**は「ニューロン（neuron）」で構成されます。
それぞれのニューロンは、単独では非常に単純な計算しか行いません。

しかし、ニューロン同士が作る**接続の複雑さ**こそが、ニューラルネットワークの力の源です。

### 線形ユニット（The Linear Unit）

では、ニューラルネットワークの基本構成要素である「ニューロン」から始めましょう。
入力が1つの場合のニューロン（ユニット）は、次のように表されます。
![](https://storage.googleapis.com/zenn-user-upload/2744e34db11f-20260216.png)*線形ユニット：[y = wx + b]*

#### 重み（weight）
入力は ( x ) です。
この入力とニューロンを結ぶ接続には重み ( w ) があります。

値が接続を通るとき、その値は重みと掛け合わされます。
つまり、ニューロンに届く値は ( w * x ) になります。

ニューラルネットワークは、この**重みを調整することによって学習**します。

#### バイアス（bias）

( b ) は「バイアス」と呼ばれる特別な重みです。

バイアスには入力データがありません。図では 1 を入力として描かれます。
そのため、ニューロンに届く値は単に ( b ) になります（1 × b = b）。

バイアスは、入力とは独立に出力を調整する役割を持っています。

#### 出力

( y ) はニューロンの最終的な出力です。

ニューロンは、接続から受け取った値をすべて足し合わせます。
このニューロンの活性化（activation）は次の式で表されます：
y = wx + b

:::message
この式、「直線の方程式」ですね。
[y = wx + b]
傾き切片形式（slope-intercept form）で、
* ( w ) は傾き（slope）
* ( b ) は y 切片（y-intercept）

を表します。
:::

### 例：線形ユニットをモデルとして使う

通常、ニューロンは大きなネットワークの一部として機能しますが、ベースラインとして**単一ニューロンモデル**から始めることはよくあります。単一ニューロンモデルは**線形モデル**です。

#### 例：[80 Cereals](https://www.kaggle.com/datasets/crawford/80-cereals) データセット

「sugars（1食あたりの糖分グラム）」を入力、「calories（1食あたりのカロリー）」を出力とするモデルを考えます。学習の結果、
* バイアス ( b = 90 )
* 重み ( w = 2.5 )

だったとします。

糖分が5グラムのシリアルのカロリーは：[calories = 2.5 \times 5 + 90 = 102.5] となり、
期待どおりの計算結果になります。

![](https://storage.googleapis.com/zenn-user-upload/7fb96a078f57-20260216.png)*線形単位を使用して計算する*

### 複数入力（Multiple Inputs）

80 Cereals データセットには「sugars」以外にも多くの特徴量があります。
例えば「fiber（食物繊維）」や「protein（タンパク質）」も使いたい場合はどうでしょうか？
簡単です。入力を増やせばよいだけです。特徴量ごとに入力接続を1本ずつ追加します。
![](https://storage.googleapis.com/zenn-user-upload/dd33fc1c0563-20260216.png)*3 つの入力を持つ線形ユニット*

式は次のようになります：
[ y = w_0 x_0 + w_1 x_1 + w_2 x_2 + b ]

* 入力が2つなら「平面」を表現
* それ以上なら「超平面（hyperplane）」を表現

### Keras での線形ユニット

Keras でモデルを作る最も簡単な方法は `keras.Sequential` を使うことです。これはレイヤーを積み重ねたニューラルネットワークを作成します。上記のようなモデルは、denseレイヤー（次のレッスンで詳しく説明します）を使って作成できます。

次のコードで、3 つの入力特徴 (「糖類」、「繊維」、「タンパク質」) を受け入れ、単一の出力 (「カロリー」) を生成する線形モデルを定義できます。

```python
from tensorflow import keras
from tensorflow.keras import layers

# 1つの線形ユニットを持つネットワーク
model = keras.Sequential([
    layers.Dense(units=1, input_shape=[3])
])
```

- 第1引数: units
`units=1` は出力の数を指定します。
ここでは「calories」を1つ予測するので1です。

- 第2引数: input_shape
`input_shape=[3]` は入力の次元を指定します。
3つの特徴量（sugars, fiber, protein）を受け取ることを意味します。

このモデルは、すぐに学習データに対して学習（fit）できる状態です！

:::message
### なぜ input_shape は Python のリストなのか？

このコースでは、Pandas のデータフレームのような **表形式データ（tabular data）** を扱います。

特徴量は列ごとに並ぶため、常に
```
input_shape = [列数]
```
という形になります。

Keras がリストを使う理由は、より複雑なデータにも対応できるようにするためです。
例えば画像データなら、
```
[height, width, channels]
```
のように3次元になります。
:::

#### 補足説明

ちなみに、Kerasは重みだけでなく**データもテンソルとして扱います**。

`input_shape` を設定するということは、「各学習データがどのような形の配列なのか」をKerasに伝えているということになります。

たとえば：

```python
input_shape=[3]
```

とすると、

```
[0.2, 0.4, 0.6]
```

のような**長さ3のベクトル**を入力として受け取るネットワークが作られます。


#### （任意）未学習の線形モデルの出力をプロットする

レッスン5までに扱う問題の多くは**回帰問題（regression）** です。
回帰問題は「カーブフィッティング」のようなものです。データに最もよく合う曲線を探します。

では、線形モデルが作る「曲線」を見てみましょう。（予想どおり、これは**直線**になります。）

モデルの重みは、学習前はランダムに初期化されています。
下のセルを何度か実行して、ランダム初期化によって異なる直線が生成される様子を確認してみてください。
```python
import tensorflow as tf
import matplotlib.pyplot as plt

model = keras.Sequential([
    layers.Dense(1, input_shape=[1]),
])

x = tf.linspace(-1.0, 1.0, 100)
y = model.predict(x)

plt.figure(dpi=100)
plt.plot(x, y, 'k')
plt.xlim(-1, 1)
plt.ylim(-1, 1)
plt.xlabel("Input: x")
plt.ylabel("Target y")
w, b = model.weights # you could also use model.get_weights() here
plt.title("Weight: {:0.2f}\nBias: {:0.2f}".format(w[0][0], b[0]))
plt.show()
```
