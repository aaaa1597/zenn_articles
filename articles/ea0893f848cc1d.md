## 理論編
### はじめに(Introduction)
前回のレッスンでは、特徴量エンジニアリングにおける最初の「モデルベース」の手法として、クラスタリングを見ました。今回のレッスンでは、次の手法である主成分分析（PCA: Principal Component Analysis）を扱います。クラスタリングがデータ点の近さに基づいてデータセットを分割するものだとすれば、PCAはデータの「ばらつき（変動）」を分割するものと考えることができます。PCAは、データの中にある重要な関係性を発見するのに役立つ優れたツールであり、より情報量の多い特徴量を作成するためにも使えます。

（技術的な補足：PCAは通常、標準化されたデータに適用されます。標準化されたデータにおける「ばらつき」とは「相関」を意味し、標準化されていないデータでは「共分散」を意味します。このコースでは、PCAを適用する前にすべてのデータを標準化します。）

### 主成分分析（Principal Component Analysis）

アワビ（[Abalone](https://www.kaggle.com/datasets/rodolfomendes/abalone-dataset)）データセットには、タスマニア産の数千個のアワビについて測定された物理的な特徴量が含まれています（アワビは、ハマグリやカキに似た海の生き物です）。ここではまず、殻の「高さ（Height）」と「直径（Diameter）」という2つの特徴量だけを見てみましょう。

このデータの中には、アワビ同士がどのように異なっているかを表す「変動の軸（axes of variation）」が存在すると考えられます。図で表すと、これらの軸はデータの自然な次元に沿って走る互いに直交する直線として現れ、元の特徴量1つにつき1本の軸があります。
![](https://storage.googleapis.com/zenn-user-upload/09a7fbe57bb6-20260211.png)

多くの場合、これらの変動の軸には名前を付けることができます。長い方の軸は「サイズ（Size）」成分と呼べるでしょう。これは、高さも直径も小さい（左下）ものと、高さも直径も大きい（右上）ものの対比を表します。
一方、短い方の軸は「形状（Shape）」成分と呼べます。これは、高さが小さく直径が大きい（平たい形）ものと、高さが大きく直径が小さい（丸い形）ものの対比を表します。

ここで注目してほしいのは、アワビを「Height」と「Diameter」で記述する代わりに、「Size」と「Shape」で記述することもできる、という点です。これこそがPCAの基本的な考え方です。つまり、元の特徴量でデータを表す代わりに、データの変動の軸で表すのです。この変動の軸が、新しい特徴量になります。

![](https://storage.googleapis.com/zenn-user-upload/b68085118135-20260211.png)


主成分は、特徴空間においてデータセットを回転させることで、新しい特徴量として得られます。
PCAによって作られる新しい特徴量は、実際には元の特徴量の線形結合（重み付き和）です。

```
df["Size"]  = 0.707 * X["Height"] + 0.707 * X["Diameter"]
df["Shape"] = 0.707 * X["Height"] - 0.707 * X["Diameter"]
```

これらの新しい特徴量を、データの **主成分（principal components）** と呼びます。また、このときの重みは **負荷量（loadings）** と呼ばれます。
主成分の数は、元のデータセットの特徴量の数と同じになります。もし2つではなく10個の特徴量を使っていれば、10個の主成分が得られることになります。

---

各主成分の負荷量は、その成分がどのような変動を表しているかを、符号と大きさによって教えてくれます。

| 特徴量 \ 成分 | Size (PC1) | Shape (PC2) |
| -------- | ---------- | ----------- |
| Height   | 0.707      | 0.707       |
| Diameter | 0.707      | -0.707      |

この負荷量の表から、Size成分ではHeightとDiameterが同じ方向に変動している（符号が同じ）のに対し、Shape成分では逆方向に変動している（符号が逆）ことが分かります。また、どちらの成分でも負荷量の大きさは同じなので、両方の特徴量が等しく寄与しています。

---

PCAは、各成分がどれだけの変動を説明しているかも教えてくれます。図を見ると、Shape成分よりもSize成分の方が、データの変動を多く含んでいることが分かります。PCAでは、これを各成分の **説明分散比（explained variance）** として定量的に示します。

![](https://storage.googleapis.com/zenn-user-upload/0156e251aa43-20260211.png)*Size accounts for about 96% and the Shape for about 4% of the variance between Height and Diameter.<br/>高さと直径の差異のうち、サイズは約 96% を占め、形状は約 4% を占めます。*

Size成分が、HeightとDiameterの間の変動の大部分を捉えているわけです。ただし重要なのは、分散の大きさがそのまま予測性能の良さを意味するとは限らない、という点です。何を予測したいかによって重要性は変わります。

### 特徴量エンジニアリングとしてのPCA

PCAを特徴量エンジニアリングに使う方法は、大きく2つあります。

1つ目は、**記述的な手法**として使う方法です。主成分はデータの変動を表しているので、各主成分についてMI（相互情報量）スコアを計算し、どの種類の変動がターゲットに対して最も予測力が高いかを見ることができます。
例えば、Sizeが重要であれば「Height × Diameter」のような特徴量を考えられますし、Shapeが重要であれば「Height / Diameter」のような比率を考えられます。スコアの高い主成分に対してクラスタリングを行うこともできます。

2つ目は、**主成分そのものを特徴量として使う方法**です。主成分はデータの変動構造を直接表しているため、元の特徴量よりも情報量が多い場合があります。代表的なユースケースは次の通りです。

* **次元削減**：特徴量同士が強く冗長（特に多重共線性がある）な場合、PCAはその冗長性を分散のほぼゼロの成分として切り出します。これらは情報をほとんど含まないため、削除できます。
* **異常検知**：元の特徴量では目立たないような異常な変動が、低分散の成分として現れることがあります。これは異常値検出に有用です。
* **ノイズ低減**：センサーの測定値などでは、共通の背景ノイズが含まれることがよくあります。PCAは、情報のある信号を少数の成分に集約し、ノイズを分離することで、S/N比を改善できる場合があります。
* **非相関化**：強く相関した特徴量を苦手とするMLアルゴリズムもあります。PCAは、相関した特徴量を非相関な成分に変換します。

PCAは、データの相関構造に直接アクセスする手段を与えてくれます。きっと他にも応用例を思いつくはずです。

---

### PCAを使う際の注意点（PCA Best Practices）
PCAを適用する際には、次の点に注意してください。
* PCAは、連続量やカウントなどの数値特徴量にしか使えません。
* PCAはスケールに敏感です。特別な理由がない限り、事前にデータを標準化するのが良い習慣です。
* 外れ値は結果に過度な影響を与えることがあるため、除去や制約を検討してください。

---

### 例：1985年 自動車データ（Example - 1985 Automobiles）

以下のセルでは、データの読み込みと、`plot_variance` および `make_mi_scores` 関数の定義を行っています。
```python
import matplotlib.pyplot as plt
import numpy as np
import pandas as pd
import seaborn as sns
from IPython.display import display
from sklearn.feature_selection import mutual_info_regression


plt.style.use("seaborn-whitegrid")
plt.rc("figure", autolayout=True)
plt.rc(
    "axes",
    labelweight="bold",
    labelsize="large",
    titleweight="bold",
    titlesize=14,
    titlepad=10,
)


def plot_variance(pca, width=8, dpi=100):
    # Create figure
    fig, axs = plt.subplots(1, 2)
    n = pca.n_components_
    grid = np.arange(1, n + 1)
    # Explained variance
    evr = pca.explained_variance_ratio_
    axs[0].bar(grid, evr)
    axs[0].set(
        xlabel="Component", title="% Explained Variance", ylim=(0.0, 1.0)
    )
    # Cumulative Variance
    cv = np.cumsum(evr)
    axs[1].plot(np.r_[0, grid], np.r_[0, cv], "o-")
    axs[1].set(
        xlabel="Component", title="% Cumulative Variance", ylim=(0.0, 1.0)
    )
    # Set up figure
    fig.set(figwidth=8, dpi=100)
    return axs

def make_mi_scores(X, y, discrete_features):
    mi_scores = mutual_info_regression(X, y, discrete_features=discrete_features)
    mi_scores = pd.Series(mi_scores, name="MI Scores", index=X.columns)
    mi_scores = mi_scores.sort_values(ascending=False)
    return mi_scores


df = pd.read_csv("../input/fe-course-data/autos.csv")
```

### 3) 外れ値検出
極端な値に何かパターンが見られますか？外れ値はデータの特定のサブセットから発生しているように見えますか？ 答えを考え終わったら、次のセルを実行して解答と考察を確認してください。

![](https://storage.googleapis.com/zenn-user-upload/3073d56924c2-20260211.png)

Edwards 地区で「Partial（部分的）」な売却として記録されている住宅が、いくつか際立っていることに気づくでしょう。Partial sale（部分売却）とは、ある不動産に複数の所有者がいて、そのうちの一人または複数人が自分の「持ち分」だけを売却する場合に起こるものです。

この種の売却は、家族の遺産整理や事業解散の際に行われることが多く、一般には公開されません。もしあなたが公開市場における住宅価格を予測しようとしているのであれば、こうした売却データをデータセットから除外するのは妥当だと言えるでしょう――これらはまさに真の外れ値だからです。
