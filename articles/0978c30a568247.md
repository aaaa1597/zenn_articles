## 理論編
### はじめに(Introduction)
これまでのエージェントは、ゲームのプレイ方法に関する詳細な情報に依存してきました。ヒューリスティック(Heuristic)は、どの手を選ぶべきかについて多くの指針を与えてくれます。

このチュートリアルでは、ヒューリスティック(Heuristic)を使わずに、強化学習(reinforcement learning)を用いて知的なエージェントを構築する方法を学びます。その代わりに、ゲームをプレイしながら勝率を最大化することで、時間をかけて戦略を徐々に改善していきます。

このノートブックでは、この複雑な分野を詳細に扱うことはできませんが、全体像を学び、自分自身のエージェントを訓練するために使えるコードを確認します。

### ニューラルネットワーク(Neural Networks)
完璧なヒューリスティック(Heuristic)を作るのは難しいものです。ヒューリスティック(Heuristic)を改善するには、通常、多くの対戦を行い、エージェントがより良い選択をできたはずの具体的なケースを見つける必要があります。また、何が問題なのかを正確に理解し、既存のミスを修正しつつ新たなミスを生まないようにするのは困難です。

もし、プレイ経験を通じてエージェントを体系的に改善できる方法があれば、もっと簡単だと思いませんか？

このチュートリアルでは、そのためにヒューリスティック(Heuristic)の代わりにニューラルネットワーク(NN)を使います。

このネットワークは現在の盤面を入力として受け取り、各可能な手に対する確率を出力します。
![](https://storage.googleapis.com/zenn-user-upload/e3bd11883f77-20260404.png)

そして、エージェントはこれらの確率に基づいて手をサンプリングして選択します。例えば、上の画像の盤面では、エージェントは50%の確率で列4を選択します。

このようにして、良い戦略を表現するためには、あらゆる盤面に対してより良い手に高い確率を割り当てるように、ネットワークの重みを調整すればよいことになります。

少なくとも理論的にはそれが目標です。実際にはそれを厳密に検証することはありません――なぜなら、コネクトフォーには4兆以上の盤面が存在するからです！

### Setup
では、実際にネットワークの重みをどのように調整すればよいのでしょうか？このレッスンでは次の方法を採用します。

- 各手の後で、エージェントに報酬を与え、その行動の良し悪しを評価します。
    * その手で勝利した場合：+1
    * 無効な手を打ってゲームが終了した場合：-10
    * 次の相手の手で負ける場合（防げなかった場合）：-1
    * それ以外：1/42

- 各ゲームの終了時に、エージェントは報酬を合計します。この合計を「累積報酬」と呼びます。例えば、以下の様に計算します。
    * 8手で終了し（各プレイヤー4手）、エージェントが勝った場合
      　→ 累積報酬 = 3×(1/42) + 1
    
    * 11手で終了し（相手先手でエージェントは5手）、最後に相手が勝った場合
      　→ 累積報酬 = 4×(1/42) - 1
    
    * 引き分けの場合（エージェントは21手）
      　→ 累積報酬 = 21×(1/42)
    
    * 7手で無効な手により終了した場合
      　→ 累積報酬 = 3×(1/42) - 10

私たちの目標は、この累積報酬を平均的に最大化するようなニューラルネットワーク(NN)の重みを見つけることです。

このように報酬を用いてエージェントの性能を評価する考え方は、強化学習の基本概念です。この形式に問題を定義すれば、さまざまな強化学習アルゴリズムを使ってエージェントを作ることができます。

### Reinforcement Learning
強化学習には、DQN、A2C、PPOなど多くのアルゴリズムがあります。これらはいずれも似たような流れでエージェントを作ります。
* 最初は重みをランダムに設定する。
* エージェントがプレイする中で、重みを変えながら累積報酬への影響を試す。
* 多くのゲームを通じて、どの重みが良いかを学び、より良い方向へ収束していく。

ここでは詳細は省略していますが、実際にはかなり複雑な処理が含まれています。今は全体像に注目しましょう。

こうして、エージェントは勝利（+1）を目指し、負け（-1）や反則（-10）を避けるようになります。

また、ゲームを長く続けることで1/42の報酬も多く得ようとします。「ゲームは早く勝つ方がいいのでは？」と思うかもしれませんが、その通りです。この設計だと、勝てる場面でも遅らせる傾向が出る可能性があります。それでも1/42の報酬を入れるのは、学習を安定させるためです。詳しくは「temporal credit assignment problem」や「reward shaping」を調べてみてください。

次のセクションでは、[PPOアルゴリズム](https://openai.com/index/openai-baselines-ppo/)を使ってエージェントを作成します。

### Code
強化学習アルゴリズムには多くの実装がありますが、この講座では[Stable-Baselines3](https://stable-baselines3.readthedocs.io/en/master/)を使用します。

このライブラリに対応させるため、ConnectFourGymクラスを定義します。このクラスはConnectXをOpenAI Gym環境として実装します。

主なメソッドは以下の通り：
* reset()：ゲーム開始時に呼ばれ、初期盤面を返す
* change_reward()：報酬の調整
* step()：エージェントの行動と相手の応答を実行し、結果を返す
返り値：
    * 次の盤面
    * 直前の行動の報酬
    * ゲーム終了フラグ

環境の定義方法について詳しく知りたい場合は、[こちらのドキュメント](https://stable-baselines.readthedocs.io/en/master/guide/custom_env.html)をご覧ください。

```python
import random
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
%matplotlib inline

import gym
from kaggle_environments import make, evaluate
from gym import spaces

class ConnectFourGym(gym.Env):
    def __init__(self, agent2="random"):
        ks_env = make("connectx", debug=True)
        self.env = ks_env.train([None, agent2])
        self.rows = ks_env.configuration.rows
        self.columns = ks_env.configuration.columns
        # Learn about spaces here: http://gym.openai.com/docs/#spaces
        self.action_space = spaces.Discrete(self.columns)
        self.observation_space = spaces.Box(low=0, high=2, 
                                            shape=(1,self.rows,self.columns), dtype=int)
        # Tuple corresponding to the min and max possible rewards
        self.reward_range = (-10, 1)
        # StableBaselines throws error if these are not defined
        self.spec = None
        self.metadata = None
    def reset(self):
        self.obs = self.env.reset()
        return np.array(self.obs['board']).reshape(1,self.rows,self.columns)
    def change_reward(self, old_reward, done):
        if old_reward == 1: # The agent won the game
            return 1
        elif done: # The opponent won the game
            return -1
        else: # Reward 1/42
            return 1/(self.rows*self.columns)
    def step(self, action):
        # Check if agent's move is valid
        is_valid = (self.obs['board'][int(action)] == 0)
        if is_valid: # Play the move
            self.obs, old_reward, done, _ = self.env.step(int(action))
            reward = self.change_reward(old_reward, done)
        else: # End the game and penalize agent
            reward, done, _ = -10, True, {}
        return np.array(self.obs['board']).reshape(1,self.rows,self.columns), reward, done, _
```

```python
def get_win_percentages(agent1, agent2, n_rounds=100):
    # Use default Connect Four setup
    config = {'rows': 6, 'columns': 7, 'inarow': 4}
    # Agent 1 goes first (roughly) half the time          
    outcomes = evaluate("connectx", [agent1, agent2], config, [], n_rounds//2)
    # Agent 2 goes first (roughly) half the time      
    outcomes += [[b,a] for [a,b] in evaluate("connectx", [agent2, agent1], config, [], n_rounds-n_rounds//2)]
    print("Agent 1 Win Percentage:", np.round(outcomes.count([1,-1])/len(outcomes), 2))
    print("Agent 2 Win Percentage:", np.round(outcomes.count([-1,1])/len(outcomes), 2))
    print("Number of Invalid Plays by Agent 1:", outcomes.count([None, 0]))
    print("Number of Invalid Plays by Agent 2:", outcomes.count([0, None]))
```

### 2) 報酬を決める
チュートリアルでは、コネクトフォーで勝つようにエージェントを促す報酬の与え方を学びました。ここでは、[マインスイーパー](https://en.wikipedia.org/wiki/Minesweeper_(video_game))で勝つエージェントを訓練することを考えます。このゲームの目的は、爆弾を爆発させずに盤面をすべて開くことです。

このゲームはGoogle検索でプレイできます（[リンク](https://www.google.com/search?q=minesweeper)の[Play]ボタンをクリック）。
![](https://storage.googleapis.com/zenn-user-upload/3c64bc8600d0-20260404.png)

各手において、以下のいずれかが起こります：
* エージェントが無効な手を選んだ（すでに開かれているマスを選択した）。この場合、ゲームは終了し、エージェントは負けとします。
* 地雷がないマスを開いた。このとき、すべての安全なマスが開かれたため、エージェントの勝利となります。
* 地雷がないマスを開いたが、まだ勝敗は決まっていない。
* 地雷を踏んでしまい、負けとなる。

これら4つのケースそれぞれに対して、どのように報酬を設定すれば、累積報酬を最大化することでエージェントが勝とうとするようになるでしょうか？

答えを決めたら、下のコードセルを実行してこの問題の完了として記録してください。

答え:
![](https://storage.googleapis.com/zenn-user-upload/e971e52595a1-20260404.png)
考えられる解決策は、エージェントが各動作を行うたびに、その動作の良し悪しを示す報酬を与えるというものです。
- エージェントがその手でゲームに勝利した場合、+1の報酬を得ます。
- そうでなく、エージェントが無効な手を選択した場合、報酬は-10になります。
- そうでなく、地雷を爆発させた場合は、-1の報酬を得る。
- そうでなければ、エージェントが隠された地雷のないマスをクリアした場合、+1/100の報酬を得ます。

解答の妥当性を確認するには、無効な手を選んだ場合と地雷を爆発させた場合の報酬がどちらもマイナスになることを確認してください。ゲームに勝利した場合の報酬はプラスであるべきです。また、地雷が隠されていないマスをクリアした場合の報酬は、ゼロかわずかにプラスであるべきです。

```python
import pandas as pd
import gym
import matplotlib.pyplot as plt

from kaggle_environments import make, evaluate
from gym import spaces

class ConnectFourGym(gym.Env):
    def __init__(self, agent2="random"):
        ks_env = make("connectx", debug=True)
        self.env = ks_env.train([None, agent2])
        self.rows = ks_env.configuration.rows
        self.columns = ks_env.configuration.columns
        # Learn about spaces here: http://gym.openai.com/docs/#spaces
        self.action_space = spaces.Discrete(self.columns)
        self.observation_space = spaces.Box(low=0, high=2, 
                                            shape=(1,self.rows,self.columns), dtype=int)
        # Tuple corresponding to the min and max possible rewards
        self.reward_range = (-10, 1)
        # StableBaselines throws error if these are not defined
        self.spec = None
        self.metadata = None
    def reset(self):
        self.obs = self.env.reset()
        return np.array(self.obs['board']).reshape(1,self.rows,self.columns)
    def change_reward(self, old_reward, done):
        if old_reward == 1: # The agent won the game
            return 1
        elif done: # The opponent won the game
            return -1
        else: # Reward 1/42
            return 1/(self.rows*self.columns)
    def step(self, action):
        # Check if agent's move is valid
        is_valid = (self.obs['board'][int(action)] == 0)
        if is_valid: # Play the move
            self.obs, old_reward, done, _ = self.env.step(int(action))
            reward = self.change_reward(old_reward, done)
        else: # End the game and penalize agent
            reward, done, _ = -10, True, {}
        return np.array(self.obs['board']).reshape(1,self.rows,self.columns), reward, done, _
```
