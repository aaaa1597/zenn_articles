### 数学的変換（Mathematical Transforms）

数値特徴量同士の関係は、数学的な式で表されることが多く、ドメイン調査の中で頻繁に目にします。Pandas では、列に対して通常の数値と同じように算術演算を適用できます。

自動車データセットには、エンジンに関する特徴量があります。調べてみると、有用そうな新しい特徴量を作るためのさまざまな式が見つかります。例えば「ストローク比（stroke ratio）」は、エンジンの効率と性能のバランスを表す指標です。

```python
autos["stroke_ratio"] = autos.stroke / autos.bore
```

```python
autos["displacement"] = (
```

### カウント（Counts）

「何かが存在するかどうか」を表す特徴量は、しばしばセットで与えられます（例えば病気のリスク因子の集合など）。こうした特徴量は、合計して「数」にすることができます。

### 特徴量を分解・結合する（Building-Up and Breaking-Down Features）
複雑な文字列は、より単純な要素に分解すると有用な場合があります。よくある例としては次のようなものがあります。
- ID番号: 123-45-6789
- 電話番号: (999) 555-0123
- 住所: 8241 Kaggle Ln., Goose City, NV
- URL: http://www.kaggle.com
- 商品コード: 0 36000 29145 2

このような特徴量には、たいてい何らかの構造があります。例えば米国の電話番号では、市外局番（(999) の部分）が発信地を表します。ここでも、事前の調査が役に立ちます。

### グループ変換（Group Transforms）

最後に、カテゴリごとに複数行をまとめて情報を集約する「グループ変換」です。
例えば次のような特徴量を作れます。「その人が住んでいる州の平均所得」、「ジャンルごとの平日公開映画の割合」。もしカテゴリ間の相互作用を見つけていれば、そのカテゴリでグループ化した変換は検討する価値があります。

グループ変換では、グループ分けに使うカテゴリ特徴量、集約したい数値特徴量、集約関数（平均など）を組み合わせます。「州ごとの平均所得」であれば、State をグループ、Income を対象、mean を集約関数として使います。Pandas では groupby と transform を使います。


```python
customer["AverageIncome"] = (
    customer.groupby("State")  # for each state
    ["Income"]                 # select the income
    .transform("mean")         # and compute its mean
)

customer[["State", "Income", "AverageIncome"]].head(10)
```

### 特徴量作成のヒント（Tips on Creating Features）
最後に、モデルの特性を意識した特徴量作成の指針です。
- 線形モデルは和や差を自然に学習できるが、それ以上に複雑な関係は苦手。
- 比率（ratio）は多くのモデルにとって学習しにくいが、作ってあげると性能向上につながることが多い。
- 線形モデルやニューラルネットは正規化された特徴量の方が得意。特にニューラルネットでは、0 から大きく離れないスケールが重要。
- 木系モデル（ランダムフォレストや XGBoost など）は正規化の恩恵は比較的小さいが、場合によっては役立つ。
- 木系モデルは特徴量の組み合わせを自動的に学習できるが、データが少ない場合などは、重要な組み合わせを明示的に作った方が良いこともある。
- カウント特徴量は、複数の情報を一度に集約するのが苦手な木系モデルにとって特に有効。
