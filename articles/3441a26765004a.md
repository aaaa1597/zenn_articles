## 理論編
### はじめに(Introduction)
線形回帰(Linear regression)はトレンドの外挿（将来への延長予測）には優れていますが、相互作用を学習することはできません。(※外挿...既知のデータや観測データを使用し、観測されていない時間、位置、サンプリングされたエリア外にあるその他の変数の値を推量または計算する処理)
一方、XGBoostは相互作用の学習には優れていますが、トレンドを外挿することはできません。このレッスンでは、互いを補完し合う学習アルゴリズムを組み合わせ、一方の強みで他方の弱みを補う「ハイブリッド」予測モデルの作り方を学びます。

### 構成要素と残差
効果的なハイブリッドを設計するためには、時系列がどのように構成されているのかをより深く理解する必要があります。これまでに、トレンド・季節性・循環という3つの依存パターンを学んできました。多くの時系列は、これら3つの要素と本質的に予測不可能な完全にランダムな誤差を加えた加法モデルでうまく表すことができます。

`series = trend + seasons + cycles + error`
系列 ＝ トレンド ＋ 季節性 ＋ 循環 ＋ 誤差

このモデルの各項を、時系列の「構成要素（コンポーネント）」と呼びます。

モデルの**残差**とは、モデルが学習に用いたターゲットとモデルの予測値との差、つまり実際の曲線と当てはめられた曲線との差のことです。残差をある特徴量に対してプロットすると、その特徴量からモデルが学習できなかったターゲットの「残りの部分」が見えてきます。

![](https://storage.googleapis.com/zenn-user-upload/9d2b3d0ec0fa-20260302.png)*ターゲット系列と予測値（青）との差が、残差系列になります。*

上図の左側には、Tunnel Traffic の系列の一部と、レッスン3で作成したトレンド＋季節性の曲線が示されています。当てはめた曲線を差し引くと、右側に示されている残差が得られます。残差には、トレンド＋季節性モデルが Tunnel Traffic から学習できなかったすべての情報が含まれています。

時系列の構成要素を学習することは、反復的なプロセスとして考えることができます。まずトレンドを学習して差し引き、次にトレンドを除いた残差から季節性を学習して差し引き、さらに循環を学習して差し引き、最後には予測不可能な誤差だけが残る、という流れです。

![](https://storage.googleapis.com/zenn-user-upload/d6a89696b53f-20260302.png)*Mauna Loa CO2 の構成要素を段階的に学習していきます。青の当てはめ曲線を元の系列から引くことで、次のステップの系列を得ます。*

Add together all the components we learned and we get the complete model.
学習したすべての構成要素を足し合わせれば、完全なモデルになります。これは基本的に、トレンド、季節、サイクルをモデル化する完全な特徴セットで線形回帰を学習させた場合と同じ結果になります。

![](https://storage.googleapis.com/zenn-user-upload/d80ab77aa6b4-20260302.png)*学習した構成要素を足し合わせて、完全なモデルを得ます。*

### 残差を用いたハイブリッド予測

これまでのレッスンでは、単一のアルゴリズム（線形回帰）を使って、すべての構成要素を同時に学習していました。しかし、ある構成要素には一つのアルゴリズムを、残りには別のアルゴリズムを使うことも可能です。こうすることで、各構成要素に対して最適なアルゴリズムを選ぶことができます。そのために、まず一つ目のアルゴリズムで元の系列を当てはめ、次に二つ目のアルゴリズムで残差系列を当てはめます。

具体的な手順は次の通りです。

```python
# 1. Train and predict with first model
model_1.fit(X_train_1, y_train)
y_pred_1 = model_1.predict(X_train)

# 2. Train and predict with second model on residuals
model_2.fit(X_train_2, y_train - y_pred_1)
y_pred_2 = model_2.predict(X_train_2)

# 3. Add to get overall predictions
y_pred = y_pred_1 + y_pred_2
```

通常、それぞれのモデルに何を学習させたいかに応じて、異なる特徴量セット（上の X_train_1 と X_train_2）を使います。例えば、最初のモデルで傾向(trend)を学習させるなら、二つ目のモデルにはトレンド特徴量は通常必要ありません。

2つ以上のモデルを使うことも可能ですが、実際にはそれほど有効ではないようです。実際、ハイブリッド構築で最も一般的な戦略は今説明した方法です。まず単純な（通常は線形の）アルゴリズムを使い、その後にGBDTや深層ニューラルネットのような複雑で非線形な学習器を続けます。単純なモデルは、後続の強力なアルゴリズムを補助する「ヘルパー」として設計されるのが一般的です。

### ハイブリッドの設計
このレッスンで説明した方法以外にも、機械学習モデルを組み合わせる方法は数多くあります。ただし、うまく組み合わせるためには、それぞれのアルゴリズムがどのように動作しているかをもう少し深く理解する必要があります。

There are generally two ways a regression algorithm can make predictions: either by transforming the features or by transforming the target.
一般に、回帰アルゴリズムが予測を行う方法は2つあります。特徴量を変換する方法か、ターゲットを変換する方法です。特徴量変換型アルゴリズムは、特徴量を入力として受け取り、それらを組み合わせたり変換したりする数学的関数を学習し、訓練データのターゲット値に一致する出力を生成します。線形回帰やニューラルネットはこのタイプです。

ターゲット変換型アルゴリズムは、特徴量を使って訓練データのターゲット値をグループ分けし、そのグループ内の平均値をとることで予測を行います。特徴量は「どのグループを平均するか」を示す役割を果たします。決定木や最近傍法はこのタイプです。

重要なのは次の点です。特徴量変換型は、適切な特徴量があれば訓練データの範囲を超えてターゲットを外挿できますが、ターゲット変換型の予測値は常に訓練データの範囲内に制限されます。時間ダミーが時間ステップを数え続ける限り、線形回帰はトレンド線を延長し続けます。同じ時間ダミーを与えた場合、決定木は訓練データの最後のステップで示されたトレンドを、そのまま将来に向けて延々と予測します。決定木はトレンドを外挿できません。ランダムフォレストや勾配ブースティング決定木（XGBoostなど）は決定木のアンサンブルなので、同様にトレンドを外挿できません。

![](https://storage.googleapis.com/zenn-user-upload/262d07af241d-20260302.png)*決定木は訓練データの範囲を超えてトレンドを外挿することができません。*

This difference is what motivates the hybrid design in this lesson: use linear regression to extrapolate the trend, transform the target to remove the trend, and apply XGBoost to the detrended residuals.
この違いこそが、本レッスンでのハイブリッド設計の動機です。線形回帰でトレンドを外挿し、ターゲットからトレンドを取り除き、トレンド除去後の残差にXGBoostを適用します。ニューラルネット（特徴量変換型）をハイブリッド化する場合は、別のモデルの予測値を特徴量として追加し、それをニューラルネットが自らの予測の一部として利用する方法もあります。残差に当てはめる方法は、勾配ブースティングが用いる手法と同じなので「ブーステッド・ハイブリッド」と呼びます。一方、予測値を特徴量として使う方法は「スタッキング」と呼ばれるため、「スタックド・ハイブリッド」と呼びます。

### 例 ― US Retail Sales
[US Retail Sales](https://www.census.gov/retail/index.html) データセットは、米国国勢調査局が収集した1992年から2019年までの各種小売業の月次売上データを含んでいます。目的は、それ以前の売上データを用いて2016年から2019年の売上を予測することです。線形回帰＋XGBoostのハイブリッドを作るだけでなく、XGBoost用に時系列データセットを整形する方法も見ていきます。

```python
import pandas as pd
```

まず、線形回帰モデルを用いて各系列のトレンドを学習してみましょう。デモンストレーションとして、2次（2次）トレンドを使用します。(ここでのコードは、基本的に前回のレッスンのコードと同じです。) フィット感は完璧ではありませんが、ニーズには十分です。

```python
y = retail.copy()

# Create trend features
dp = DeterministicProcess(
    index=y.index,  # dates from the training data
    constant=True,  # the intercept
    order=2,        # quadratic trend
    drop=True,      # drop terms to avoid collinearity
)
X = dp.in_sample()  # features for the training data

# Test on the years 2016-2019. It will be easier for us later if we
# split the date index instead of the dataframe directly.
idx_train, idx_test = train_test_split(
    y.index, test_size=12 * 4, shuffle=False,
)
X_train, X_test = X.loc[idx_train, :], X.loc[idx_test, :]
y_train, y_test = y.loc[idx_train], y.loc[idx_test]

# Fit trend model
model = LinearRegression(fit_intercept=False)
model.fit(X_train, y_train)

# Make predictions
y_fit = pd.DataFrame(
    model.predict(X_train),
    index=y_train.index,
    columns=y_train.columns,
)
y_pred = pd.DataFrame(
    model.predict(X_test),
    index=y_test.index,
    columns=y_test.columns,
)

# Plot
axs = y_train.plot(color='0.25', subplots=True, sharex=True)
axs = y_test.plot(color='0.25', subplots=True, sharex=True, ax=axs)
axs = y_fit.plot(color='C0', subplots=True, sharex=True, ax=axs)
axs = y_pred.plot(color='C3', subplots=True, sharex=True, ax=axs)
for ax in axs: ax.legend([])
_ = plt.suptitle("Trends")
```

```python
# The `stack` method converts column labels to row labels, pivoting from wide format to long
```

XGBoost が 2 つの時系列を区別できるように、「Industries」の行ラベルをラベル エンコーディングによるカテゴリ特徴に変換します。また、時間インデックスから月番号を取得して、年間の季節性の機能も作成します。

```python
# Turn row labels into categorical feature columns with a label encoding
X = X.reset_index('Industries')
# Label encoding for 'Industries' feature
for colname in X.select_dtypes(["object", "category"]):
    X[colname], _ = X[colname].factorize()

# Label encoding for annual seasonality
X["Month"] = X.index.month  # values are 1, 2, ..., 12

# Create splits
X_train, X_test = X.loc[idx_train, :], X.loc[idx_test, :]
y_train, y_test = y.loc[idx_train], y.loc[idx_test]
```
ここで、先ほど作成したトレンド予測を長形式に変換し、元の系列から減算します。これにより、XGBoost が学習できるトレンド除去済み（残差）系列が得られます。
```python
# Pivot wide to long (stack) and convert DataFrame to Series (squeeze)
y_fit = y_fit.stack().squeeze()    # trend from training set
y_pred = y_pred.stack().squeeze()  # trend from test set

# Create residuals (the collection of detrended series) from the training set
y_resid = y_train - y_fit

# Train XGBoost on the residuals
xgb = XGBRegressor()
xgb.fit(X_train, y_resid)

# Add the predicted residuals onto the predicted trends
y_fit_boosted = xgb.predict(X_train) + y_fit
y_pred_boosted = xgb.predict(X_test) + y_pred
```

当てはまりはかなり良さそうですが、XGBoostが学習するトレンドは線形回帰が学習したトレンドの質に依存していることがわかります。特に、'BuildingMaterials' 系列で線形回帰のトレンド当てはめが不十分だった点について、XGBoostはそれを補正することができませんでした。

### 2）ブースティング・ハイブリッドのpredictメソッドを定義する
次に、BoostedHybridクラスのpredictメソッドを定義してください。必要であれば、チュートリアルの「Hybrid Forecasting with Residuals」セクションのステップ3を参照してください。
```python
def predict(self, X_1, X_2):
    y_pred = pd.DataFrame(
        # YOUR CODE HERE: predict with self.model_1
        self.model_1.predict(X_1),
        index=X_1.index, columns=self.y_columns,
    )
    y_pred = y_pred.stack().squeeze()  # wide to long

    # YOUR CODE HERE: add self.model_2 predictions to y_pred
    y_pred += self.model_2.predict(X_2)
```
