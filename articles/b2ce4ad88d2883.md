## 理論編
### はじめに（Introduction）

新しいデータセットに初めて向き合うとき、圧倒されてしまうことがあります。説明もないまま、何百・何千もの特徴量を渡されることもあるでしょう。いったいどこから手を付ければいいのでしょうか？

最初の良い一歩は、**「特徴量の有用性」**を測る指標を使ってランキングを作ることです。これは、各特徴量とターゲットとの関連性を測る関数です。こうして最も有用な特徴量の小さな集合を選べば、まずそこから開発を始めることができ、「この時間は無駄にならない」という自信も持てます。

ここで使う指標は "相互情報量（mutual information）" と呼ばれるものです。相互情報量は相関係数と少し似ていて、2つの量の関係性を測ります。ただし大きな違いがあります。
相関が「線形な関係」しか捉えられないのに対し、相互情報量は あらゆる種類の関係 を検出できます。

相互情報量は汎用性の高い指標で、特に「どんなモデルを使うかまだ決めていない」特徴量開発の初期段階でとても役に立ちます。理由は次のとおりです。
- 使いやすく、解釈しやすい
- 計算コストが低い
- 理論的な裏付けがある
- 過学習に強い
- どんな種類の関係も検出できる

### 相互情報量とは何を測っているのか（Mutual Information and What it Measures）

相互情報量は「不確実性」という観点から関係性を表します。
2つの量の相互情報量（MI）とは、「一方を知ることで、もう一方に対する不確実性がどれだけ減るか」を表す指標です。

つまり、「ある特徴量の値が分かったとき、ターゲットについてどれだけ自信を持てるようになるか？」ということです。

以下は Ames Housing データの例です。図は住宅の外観品質（ExterQual）と販売価格（SalePrice）の関係を示しています。各点は1軒の家を表します。

![](https://storage.googleapis.com/zenn-user-upload/37839061ff10-20260208.png)*家の外観の品質を知ることで、その販売価格に関する不確実性が軽減されます。*

- ExterQual には Fair / Typical / Good / Excellent の4カテゴリがある
- 各カテゴリごとに SalePrice が散布図として描かれている
住宅の外観品質を知ることで、その販売価格に対する不確実性は小さくなります。

図を見ると、ExterQual の値を知ることで、対応する SalePrice がある程度の範囲に絞られることが分かります。
ExterQual と SalePrice の相互情報量とは、4つの ExterQual の値それぞれについて、「SalePrice の不確実性がどれだけ減るか」を平均したものです。
たとえば Fair は Typical より出現頻度が低いため、MI の計算では Fair の影響は小さくなります。

（技術的補足）: ここで言う「不確実性」は、情報理論における エントロピー（entropy） という量で測られます。ある変数のエントロピーとは、大まかに言えば「その変数の値を特定するために、平均して何回の Yes / No 質問が必要か」という意味です。質問が多いほど、その変数について不確実だと言えます。相互情報量とは、「その特徴量が、ターゲットについて平均して何問分の質問に答えてくれるか」を表しています。

### 相互情報量スコアの解釈（Interpreting Mutual Information Scores）

相互情報量の最小値は 0.0 です。
MI が 0 のとき、2つの量は独立であり、互いに何の情報も与えません。

一方、理論上は MI に上限はありません。ただし実際には 2.0 を超える値はあまり見られません。
（相互情報量は対数的な量なので、増加がとても緩やかです。）

次の図は、MI の値が特徴量とターゲットの関係の「強さ」や「種類」とどう対応するかを示しています。
![](https://storage.googleapis.com/zenn-user-upload/d32bc63d4ec9-20260208.png)*左：特徴量とターゲットの依存関係が強くなるほど、MI は大きくなる
右：相互情報量は、相関係数と違って線形に限らず、どんな関係でも捉えられる*

相互情報量を使うときに覚えておきたいポイントは次の通りです。
- MI は、その特徴量単体 がターゲットを予測する潜在能力を比較するのに役立つ
- 他の特徴量と組み合わさると有用になるが、単体ではあまり情報を持たない特徴量もあり得る。MI は特徴量同士の相互作用を検出できない（単変量指標）
- 特徴量の実際の有用性は、使用するモデルに依存する。MI が高くても、モデルがその関係を学習できなければ意味がない。関係性を明確にするために、特徴量の変換が必要な場合もある。

scikit-learn の MI アルゴリズムは、離散値の特徴量 と 連続値の特徴量 を区別して扱います。そのため、どの特徴量が離散的かを指定する必要があります。

経験則として、float 型でなければならないものは離散ではありません。
カテゴリ変数（object や categorical 型）は、ラベルエンコーディングを行うことで離散値として扱えます。(ラベルのエンコーディングについては、[カテゴリ変数](https://zenn.dev/rg687076/articles/ca86ff28e6a56f)のレッスンで確認できます。)

```python
X = df.copy()
y = X.pop("price")

# カテゴリ変数をラベルエンコードします。
for colname in X.select_dtypes("object"):
    X[colname], _ = X[colname].factorize()

# これで、すべての離散特徴量は整数型になります（MI を使う前に必ず確認しましょう）。
discrete_features = X.dtypes == int
```


scikit-learn の feature_selection モジュールには、相互情報量の関数が2つあります。
- mutual_info_regression：ターゲットが連続値の場合
- mutual_info_classif：ターゲットがカテゴリの場合

今回は価格（price）が連続値なので、mutual_info_regression を使います。

次のセルでは、MI スコアを計算し、見やすいデータフレームにまとめています。
```python
from sklearn.feature_selection import mutual_info_regression

def make_mi_scores(X, y, discrete_features):
    mi_scores = mutual_info_regression(X, y, discrete_features=discrete_features)
    mi_scores = pd.Series(mi_scores, name="MI Scores", index=X.columns)
    mi_scores = mi_scores.sort_values(ascending=False)
    return mi_scores

mi_scores = make_mi_scores(X, y, discrete_features)
mi_scores[::3]  # show a few features with their MI scores
```

### MIスコアを調べる
スコアは妥当でしょうか？高得点の特徴は、住宅において多くの人が重視すると思われる要素を反映しているでしょうか？特徴に何か共通点はありますか？

![](https://storage.googleapis.com/zenn-user-upload/1d8e38ca311b-20260208.png)
これらの特徴量の多くに共通して見られるテーマには、次のようなものがあります。
- 立地：Neighborhood（近隣地区）
- 広さ：Area や SF 系の特徴量すべて、さらに FullBath や GarageCars のような数を表すもの
- 品質：Qual 系の特徴量すべて
- 年：YearBuilt（建築年）や YearRemodAdd（改修年）
- 種類：Foundation（基礎）や GarageType（ガレージの種類）のような、設備やスタイルの説明

これらはすべて、不動産の掲載情報（Zillow など）でよく目にするタイプの特徴量です。そのため、相互情報量の指標でこれらが高く評価されたのは妥当だと言えます。

一方で、ランキングの低い特徴量は、ほとんどが何らかの意味で珍しかったり例外的だったりするものを表しており、平均的な住宅購入者にとってはあまり重要ではないものだと考えられます。

1Fam：戸建て住宅

TwnhsE：タウンハウス（端部の住戸）

BldgType 特徴量は、相互情報量（MI）のスコアがそれほど高くありませんでした。実際、プロットを見ても、BldgType の各カテゴリは SalePrice の値をうまく区別できていないことが確認できます（言い換えると、分布がかなり似通っているということです）。
