## パイプライン
前処理を伴う複雑なモデルを展開 (さらにはテスト) するための重要なスキル。

このチュートリアルでは、パイプラインを使用してモデリング コードをクリーンアップする方法を学習します。

### はじめに
**パイプライン（Pipeline）** は、データのドロップ（前処理）とモデリングのコードを整理するためのシンプルな手法です。具体的には、前処理とモデリングの各ステップを一つにまとめ、全体を「単一のステップ」であるかのように扱うことができます。

多くのデータサイエンティストはパイプラインを使わずにモデルを構築（ハック）しますが、パイプラインには以下のような重要な利点があります。

1. **コードが綺麗になる**： 前処理の各段階でデータを管理するのは煩雑になりがちです。パイプラインを使えば、各ステップで訓練データと検証データを手動で追跡する必要がなくなります。
2. **バグが減る**： 前処理の手順を間違えたり、ステップを忘れたりするミスが少なくなります。
3. **本番環境への移行が容易になる**： モデルをプロトタイプから大規模な運用環境へ移行させるのは意外と難しいものです。詳細は割愛しますが、パイプラインはその助けになります。
4. **モデル検証の選択肢が増える**： 次のチュートリアルで扱う「交差検証（Cross-validation）」などがその例です。

### 例
前回同様、メルボルンの[住宅価格データセット](https://www.kaggle.com/datasets/dansbecker/melbourne-housing-snapshot)を使用します。データの読み込み部分は本質ではないため、すでに訓練データと検証データ（X_train, X_valid, y_train, y_valid）が準備できている状態から始めます。

```python
import pandas as pd
from sklearn.model_selection import train_test_split

# データの読み込み
data = pd.read_csv('../input/melbourne-housing-snapshot/melb_data.csv')

# ターゲット（目的変数）と特徴量を分離
y = data.Price
X = data.drop(['Price'], axis=1)

# 訓練データと検証データに分割
X_train_full, X_valid_full, y_train, y_valid = train_test_split(X, y, train_size=0.8, test_size=0.2, random_state=0)

# 「カーディナリティ」は列内のユニークな値の数
# 低カーディナリティのカテゴリ列を選択（便宜上の設定）
categorical_cols = [cname for cname in X_train_full.columns if X_train_full[cname].nunique() < 10 and 
                        X_train_full[cname].dtype == "object"]

# 数値列を選択
numerical_cols = [cname for cname in X_train_full.columns if X_train_full[cname].dtype in ['int64', 'float64']]

# 選択した列のみを保持
my_cols = categorical_cols + numerical_cols
X_train = X_train_full[my_cols].copy()
X_valid = X_valid_full[my_cols].copy()
```

### パイプラインの構築（3ステップ）
完全なパイプラインは 下記3つのステップで構築されます。

### ステップ1：前処理ステップの定義
ColumnTransformer クラスを使って、異なる前処理を一つにまとめます。以下のコードでは下記2つを実施します：
- 数値データの欠損値を補完します。
- カテゴリデータの欠損値を補完し、ワンホットエンコーディング（One-Hot Encoding）を適用します。

```python
from sklearn.compose import ColumnTransformer
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import OneHotEncoder

# 数値データの処理
numerical_transformer = SimpleImputer(strategy='constant')

# カテゴリデータの処理
categorical_transformer = Pipeline(steps=[
    ('imputer', SimpleImputer(strategy='most_frequent')),
    ('onehot', OneHotEncoder(handle_unknown='ignore'))
])

# 数値とカテゴリの前処理を統合
preprocessor = ColumnTransformer(
    transformers=[
        ('num', numerical_transformer, numerical_cols),
        ('cat', categorical_transformer, categorical_cols)
    ])
```

### ステップ2：モデルの定義
次に、おなじみの RandomForestRegressor クラスを使ってランダムフォレストモデルを定義します。

```python
from sklearn.ensemble import RandomForestRegressor
model = RandomForestRegressor(n_estimators=100, random_state=0)
```

### ステップ3：パイプラインの作成と評価
最後に Pipeline クラスを使って、前処理とモデルを一つにまとめます。注目すべきポイントは以下の通りです。

- **一括処理**： 訓練データの処理とモデルの学習をたった1行で行えます。（パイプラインなしだと、補完、エンコーディング、学習を別々に行う必要があり、変数の管理が非常に面倒になります。）
- **自動予測**： predict() を呼ぶ際、未処理の X_valid を渡すだけで、パイプラインが自動的に前処理を施してから予測値を算出します。

```python
from sklearn.metrics import mean_absolute_error

# 前処理とモデルをパイプラインに統合
my_pipeline = Pipeline(steps=[('preprocessor', preprocessor),
                              ('model', model)])

# 訓練データの処理とモデル学習
my_pipeline.fit(X_train, y_train)

# 検証データの処理と予測の取得
preds = my_pipeline.predict(X_valid)

# モデルの評価
score = mean_absolute_error(y_valid, preds)
print('MAE:', score)
```
