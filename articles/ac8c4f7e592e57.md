この度、勉強会をすることになって開発環境を一斉に構築する必要にせまられ、Dockerで楽できんかな～と考えた。

# 前提
ubuntuマシンにインストールするのが前提です。
※VirtualBoxにUbuntu22.04をインストールするのは[ココ](https://zenn.dev/rg687076/articles/795f4307117d35)を参考に。

# 手順
1. dockerインストール
2. dockerでHelloWorld実行
3. 後片付け

# dockerインストール
```shell:パッケージリストを更新 && パッケージの最新バージョンをインストール
$ sudo apt update
$ sudo apt upgrade -y
```

```shell:dockerに必要なライブラリのインストール
$ sudo apt install -y apt-transport-https ca-certificates curl gnupg-agent software-properties-common
```

```shell:PGPキーの追加
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
```

```shell:Dockerリポジトリを追加
$ sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu focal stable"
```

```shell:リポジトリのパッケージリストを更新
$ sudo apt update
```

```shell:dockerインストール
$ sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

```shell:docker権限を追加
$ sudo usermod -aG docker $USER
```
※↑これは再起動せんと反映されんかった...

```shell:dockerバージョン確認
$ docker --version
```

# hello-world実行
```shell:dockerで、hello-world実行
$ docker run hello-world
```

# 後片付け
```shell:dockerコンテナ削除
$ docker rm b49a6178e819
```
```shell:dockerイメージ削除
$ docker rmi hello-world
```

```xxx:削除を確認
$ docker ps -a
CONTAINER ID   IMAGE     COMMAND   CREATED   STATUS    PORTS     NAMES
$ docker images
```
