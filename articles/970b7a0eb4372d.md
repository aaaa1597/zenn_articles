## Abstract
Kaggle「高度なSQLの[分析関数](https://www.kaggle.com/code/alexisbcook/analytic-functions)」の翻訳と実行方法の解説

## 分析関数
行のグループに対して複雑な計算を実行します。

## 理論編
### はじめに(Introduction)

[Intro to SQL のマイクロコース](https://zenn.dev/rg687076/articles/d7cf749c7ba539)では、**[集約関数（aggregate functions）](https://zenn.dev/rg687076/articles/b84a13c92064b3)** の使い方を学びました。集約関数は、複数の行の集合に基づいて計算を行う関数です。
このチュートリアルでは、同じく複数の行を対象に処理を行う **分析関数（analytic functions）** の定義方法を学びます。ただし、集約関数とは異なり、分析関数は元のテーブルの各行ごとに（場合によっては異なる）値を返します。

分析関数を使うと、比較的シンプルな構文で複雑な計算が可能になります。
たとえば、移動平均や累積合計などを簡単に計算できます。

### Syntax（構文）

分析関数の書き方を理解するために、レースに向けてトレーニングしている2人のデータを含む小さなテーブルを考えます。

* `id` 列：ランナーを識別するID
* `date` 列：トレーニング日
* `time` 列：トレーニングに費やした時間（分）

各ランナーについて、「現在のトレーニング」と「その直前のトレーニング」の平均を常に取る移動平均を計算したいとします。
これは次のクエリで実現できます。

![](https://storage.googleapis.com/zenn-user-upload/ce0940022fc0-20260215.png)

すべての分析関数には **OVER 句** があり、各計算で使用される行の集合を定義します。
OVER 句には（任意の）3つの要素があります。

#### 1. PARTITION BY 句

テーブルの行をグループに分割します。
上のクエリでは `id` で分割しているため、ランナーごとに計算が分かれます。

#### 2. ORDER BY 句

各パーティション内の並び順を定義します。
サンプルクエリでは `date` 列で並べ替えることで、古いトレーニングが先に来るようにしています。

#### 3. ウィンドウフレーム句

（例：`ROWS BETWEEN 1 PRECEDING AND CURRENT ROW`）
これは各計算で使用する行の集合を指定します。
この行の集合を **ウィンドウ（window）** と呼びます。
（実際に、分析関数は「分析ウィンドウ関数（analytic window functions）」や単に「ウィンドウ関数（window functions）」とも呼ばれます。）

![](https://storage.googleapis.com/zenn-user-upload/4b7ee612e65c-20260215.png)

---

### ウィンドウフレーム句（詳細）

ウィンドウフレーム句にはさまざまな書き方があります。

* `ROWS BETWEEN 1 PRECEDING AND CURRENT ROW`
  → 直前の1行と現在の行

* `ROWS BETWEEN 3 PRECEDING AND 1 FOLLOWING`
  → 直前3行、現在の行、次の1行

* `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`
  → パーティション内のすべての行

もちろん、これ以外にも多くの指定方法があります。
後ほどのコードで、これらの例が実際に使われているのを確認します。

### 分析関数の3つの種類

上の例では1種類の分析関数しか使っていませんが、BigQuery はさまざまな分析関数をサポートしています。
完全な一覧は[公式ドキュメント](https://docs.cloud.google.com/bigquery/docs/reference/standard-sql/window-function-calls)で確認できます。

#### 1）分析集約関数（Analytic aggregate functions）

例に出てきた `AVG()` は集約関数でした。
`OVER` 句を付けることで、分析（集約）関数として扱われます。

これらの関数は、**ウィンドウ内の**すべての値を入力として受け取り、単一の値を返します。

* `MIN()`（または `MAX()`）
  → 入力値の最小値（または最大値）を返す

* `AVG()`（または `SUM()`）
  → 入力値の平均（または合計）を返す

* `COUNT()`
  → 入力内の行数を返す

#### 2）分析ナビゲーション関数（Analytic navigation functions）

ナビゲーション関数は、現在の行とは（通常）異なる行の値に基づいて値を割り当てます。

* `FIRST_VALUE()`（または `LAST_VALUE()`）
  → 入力内の最初（または最後）の値を返す

* `LEAD()`（および `LAG()`）
  → 後続（または直前）の行の値を返す

#### 3）分析番号付け関数（Analytic numbering functions）

番号付け関数は、並び順に基づいて各行に整数値を割り当てます。

* `ROW_NUMBER()`
  → 入力内での行の順番を返す（1から始まる）

* `RANK()`
  → 並び順の列で同じ値を持つ行は同じ順位を持つ。
  次の順位は、前の順位に属する行数分だけ増加する。

#### 2015年の各日付ごとの累積利用回数を計算する
表の各行は異なる自転車旅行に対応しており、分析関数を使用して 2015 年の各日付の累計旅行回数を計算できます。

分析関数を使って、2015年の各日付における**累積トリップ数**を計算できます。
```python
# Query to count the (cumulative) number of trips per day
num_trips_query = """
                  WITH trips_by_day AS
                  (
                  SELECT DATE(start_date) AS trip_date,
                      COUNT(*) as num_trips
                  FROM `bigquery-public-data.san_francisco.bikeshare_trips`
                  WHERE EXTRACT(YEAR FROM start_date) = 2015
                  GROUP BY trip_date
                  )
                  SELECT *,
                      SUM(num_trips) 
                          OVER (
                               ORDER BY trip_date
                               ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
                               ) AS cumulative_trips
                      FROM trips_by_day
                  """

# Run the query, and return a pandas DataFrame
num_trips_result = client.query(num_trips_query).result().to_dataframe()
num_trips_result.head()
```

* `PARTITION BY` 句がないため、テーブル全体が1つのパーティションとして扱われます。
* `ORDER BY` 句で日付順に並べ替え、古い日付が先に来るようにしています。
* ウィンドウフレーム句を
  `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`

#### 2015年10月25日に各自転車が開始・終了した駅を追跡する

次のクエリでは、2015年10月25日において、各自転車がどの駅から出発し、どの駅で終了したかを追跡します。

```python
# Query to track beginning and ending stations on October 25, 2015, for each bike
start_end_query = """
                  SELECT bike_number,
                      TIME(start_date) AS trip_time,
                      FIRST_VALUE(start_station_id)
                          OVER (
                               PARTITION BY bike_number
                               ORDER BY start_date
                               ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
                               ) AS first_station_id,
                      LAST_VALUE(end_station_id)
                          OVER (
                               PARTITION BY bike_number
                               ORDER BY start_date
                               ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
                               ) AS last_station_id,
                      start_station_id,
                      end_station_id
                  FROM `bigquery-public-data.san_francisco.bikeshare_trips`
                  WHERE DATE(start_date) = '2015-10-25' 
                  """

# Run the query, and return a pandas DataFrame
start_end_result = client.query(start_end_query).result().to_dataframe()
start_end_result.head()
```

このクエリでは `FIRST_VALUE()` と `LAST_VALUE()` の両方を分析関数として使用しています。

* `PARTITION BY bike_number`
  → 自転車ごとにデータを分割しています。
  `bike_number` は自転車の一意な識別子なので、自転車ごとに独立した計算が行われます。

* `ORDER BY start_date`
  → 各自転車内で時系列順に並べています。

* ウィンドウフレーム句を
  `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`
  とすることで、各行の計算にそのパーティション全体を使用します。
  そのため、同じパーティション内の行では計算結果が同じになります。

これで、分析関数（ウィンドウ関数）の基本的な考え方と使い方が理解できました。

#### 3) 乗車の間にはどれくらいの時間が空いていますか？

以下の（途中まで書かれた）クエリは、選択された期間内の各乗車について、次の列を表示します：

* `taxi_id`
* `trip_start_timestamp`
* `trip_end_timestamp`

このクエリを修正し、**prev_break** という追加の列を含めてください。

この列は、各乗車が始まる前にドライバーが取った休憩時間（分単位）を示します。
