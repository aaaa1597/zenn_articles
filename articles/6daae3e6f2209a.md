# 理論編
## はじめに (Introduction)
pandasのDataFrameやSeriesから特定の値を選択して操作することは、ほぼすべてのデータ処理において不可欠なステップです。そのため、Pythonでデータを扱う際にまず学ぶべきことの一つは、自分に関連のあるデータポイントを迅速かつ効果的に選択する方法です。
```python
import pandas as pd
reviews = pd.read_csv("../input/wine-reviews/winemag-data-130k-v2.csv", index_col=0)
pd.set_option('display.max_rows', 5)
```

```python
reviews['country'][0]
```

### pandasにおけるインデックス（Indexing）
インデックス演算子や属性選択は、Pythonの他のエコシステムと同様に機能するため、初心者には使いやすいでしょう。しかし、pandasには独自のアクセサ演算子である loc と iloc があります。**より高度な操作を行う場合は、これらの使用が推奨されます。**

### インデックスベースの選択 (Index-based selection)
Pandasのインデックスには2つのパラダイムがあります。1つ目は「インデックスベースの選択」で、データ内での数値的な位置に基づいてデータを選択します。iloc はこのパラダイムに従います。
DataFrameの最初の行を選択するには、次のように記述します。
```python
reviews.iloc[0]
```

loc も iloc も、**「行が先、列が後」**（row-first, column-second）です。これは標準的なPython（リストのリストなど）の「列が先、行が後」とは逆であることに注意してください。

これにより、行の取得はわずかに簡単になり、列の取得はわずかに手間がかかります。iloc で列を取得するには、次のように記述します。

### ラベルベースの選択 (Label-based selection)
2つ目のパラダイムは loc 演算子が従う「ラベルベースの選択」です。このパラダイムでは、位置ではなく **データのインデックス値（ラベル）** が重要になります。

例えば、reviews の最初の項目を取得するには次のようにします。
```python
reviews.loc[0, 'country']
```
'Italy'

iloc はデータセットのインデックスを無視して「行列」として扱うため概念的に単純ですが、通常、データセットには意味のあるインデックスが割り当てられているため、loc を使う方が簡単な場合が多いです。例えば、複数の列を選択する操作は loc の方が直感的です。
```python
reviews.loc[:, ['taster_name', 'taster_twitter_handle', 'points']]
```

### loc と iloc の使い分け
loc と iloc を切り替える際、1つ注意すべき「落とし穴」があります。それは、**両者でインデックスの範囲指定のルールがわずかに異なる**ことです。
1. iloc はPythonの標準的な範囲指定と同じで、**「開始点は含み、終了点は含まない」**（0:10なら0〜9）。
2. loc は、**「開始点も終了点も含む」**（0:10なら0〜10）。

なぜ loc はこのような仕様なのでしょうか？ loc は文字列のインデックスも扱えるからです。例えば、インデックスが Apples から Potatoes までの果物リストがあるとき、df.loc['Apples':'Potatoes'] と書ければ、「ApplesからPotatoesまでのすべて」を簡単に選択できます（もし終了点を含まない仕様だと、Potatoesの次の単語を調べる必要があり不便です）。

インデックスが単純な数値リスト（0, 1, 2...）の場合、特に混乱を招きやすいので注意しましょう。

df.iloc[0:1000] → 1,000個の要素を返す。
df.loc[0:1000] → 1,001個の要素を返す。
