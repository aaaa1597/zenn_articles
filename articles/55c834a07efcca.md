## 最終回 SHAP値の高度な使用法
集計SHAP値により、さらに詳細なモデル分析が可能になります。

## 理論編
### まとめ(Recap)
まず最初に、モデルが何を学習したのかを把握するために、特徴量重要度(Permutation Importance)と部分依存プロット(Partial Dependence Plot)を学びました。

次に、個々の予測を分解するためにSHAP値について学びました。

そして今回は、SHAP値をさらに発展させ、多数のSHAP値を集約することで、特徴量重要度(Permutation Importance)や部分依存プロット(Partial Dependence Plot)のより詳細な代替手段が得られることを見ていきます。

### SHAP値の復習
SHAP値は、ある特徴量が基準値(baseline)と比べて、どれだけ予測を変化させたかを示します。

例えば、非常に単純なモデルを考えます：
`y = 4 * x1 + 2 * x2`

もし x1 が基準値0ではなく2を取る場合、x1のSHAP値は8（= 4 × 2）になります。

実際のモデルでは計算はもっと複雑になりますが、アルゴリズム上の工夫により、SHAP値を使うことで1つの予測を、各特徴量の寄与の合計として分解することが可能です。

![](https://storage.googleapis.com/zenn-user-upload/f05983f15956-20260320.png)

さらに、[Shapライブラリ](https://github.com/shap/shap)は個別の予測だけでなく、複数のSHAP値をまとめた可視化も提供しています。ここではその中でも2つの可視化に注目します。これらは特徴量重要度(Permutation Importance)や部分依存プロット(Partial Dependence Plot)と概念的に似ています。

### サマリープロット(Summary Plots)
[特徴量重要度(Permutation Importance)](https://zenn.dev/rg687076/articles/1060577e1e0b24)は、どの特徴量が重要かをシンプルな数値で示せるため便利です。

ただし、「どのように効いているか」は分かりません。例えば、重要度が中くらいの特徴量は次のどちらかかもしれません：
* 一部の予測に対しては大きく影響するが、全体では影響が小さい
* すべての予測に対して中程度の影響を持つ

SHAPのサマリープロットは、特徴量の重要度とその影響の仕方を俯瞰的に示してくれます。
![](https://storage.googleapis.com/zenn-user-upload/76dde4c38f4f-20260320.png)

このプロットは多くの点で構成され、それぞれの点は以下を表します：
* 縦位置：どの特徴量か
* 色：その特徴量の値が高いか低いか
* 横位置：その値が予測をどれだけ上げたか下げたか

例えば、左上の点は得点数の少ないチームを表しており、予測値を0.25下げています。

この図から読み取れること：
* RedやYellow & Redはモデルに無視されている
* Yellow Cardは通常は影響しないが、例外的に大きく影響するケースがある
* Goal Scoredは、値が大きいほど予測値が高くなり、値が少ないほど予測値が低くなる。

じっくり見ると、この図には非常に多くの情報が含まれています。演習では、これらの情報をどのように読み取るかを試すための質問が出題されます。

### サマリープロットのコード
以下は、サッカー／フットボールのデータを読み込むコードです。
```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier

data = pd.read_csv('../input/fifa-2018-match-statistics/FIFA 2018 Statistics.csv')
y = (data['Man of the Match'] == "Yes")  # Convert from string "Yes"/"No" to binary
feature_names = [i for i in data.columns if data[i].dtype in [np.int64, np.int64]]
X = data[feature_names]
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state=1)
my_model = RandomForestClassifier(random_state=0).fit(train_X, train_y)
```

以下のコードを使用すると、すべての検証データに対するSHAP値を取得できます。コードは短いため、コメントで説明します。

```python
import shap  # Shap値を計算するために使用されるパッケージ

# Shap値を計算できるオブジェクトを作成する
explainer = shap.TreeExplainer(my_model)

# Shap値を計算します。これがこれからプロットするものです。
# プロットに使用できるデータを増やすため、単一行ではなくval_Xのすべての行に対してshap_valueを計算します。
shap_values = explainer.shap_values(val_X)

# グラフを作成します。[1]のインデックスについては、以下のテキストで説明します。
shap.summary_plot(shap_values[1], val_X)
```
![](https://storage.googleapis.com/zenn-user-upload/b4d12b0ea9bf-20260320.png)

注意点：
* プロットするときは、shap_values[1] を呼び出します。分類問題の場合、考えられる結果ごとに SHAP 値の配列が別々に用意されています。この場合、インデックスを使用して「True」の予測に対する SHAP 値を取得します。
* SHAPの計算は遅いことがある(特にデータが大きい場合)
* xgboost は例外で、SHAPにはxgboostモデル向けに最適化が施されているため、はるかに高速になります。

これはモデルの概要を把握する上で非常に役立ちますが、個々の特徴量について詳しく調べたい場合は、SHAPの依存関係寄与度プロットが役立ちます。


### SHAP依存プロット(SHAP Dependence Contribution Plots)
部分依存プロット(Partial Dependence Plot)（PDP）は、1つの特徴量の影響を示すのに便利です。これらは洞察に富み、多くの実世界のユースケースに役立つものです。さらに、少し工夫すれば、技術的な知識のない人にも説明できます。

しかし、次のような点は分かりません：
* 影響のばらつき（分布）は？
* 他の特徴量との相互作用は？

SHAP依存プロットは、PDPと似ていますが、より詳細な情報を提供します。
![](https://storage.googleapis.com/zenn-user-upload/e48348496fe3-20260320.png)

まずは形に注目しましょう。色については後ほどまた触れます。各点はデータの1行を表しています。横軸はデータセットからの実際の値を示し、縦軸はその値が予測にどのような影響を与えたかを示しています。このグラフが右肩上がりになっているということは、ボール保持時間が長ければ長いほど、マン・オブ・ザ・マッチを獲得する確率が高くなるということです。

この分布は、他の要素がボール保持率と相互作用していることを示唆しています。例えば、ここではボール保持率が類似している2つのポイントを強調表示しています。この値によって、一方の予測値は上昇し、もう一方の予測値は低下しました。
![](https://storage.googleapis.com/zenn-user-upload/af4319467161-20260320.png)

比較のために述べると、単純な線形回帰では、このようなばらつきのない、完全な直線のグラフが得られます。

これは、相互作用を詳しく調べる必要があることを示唆しており、グラフにはそれを容易にするための色分けが含まれています。主な傾向は上昇傾向ですが、ドットの色によってその傾向が異なるかどうかを視覚的に確認できます。

具体性を示すために、以下の非常に限定的な例を考えてみましょう。
![](https://storage.googleapis.com/zenn-user-upload/558fd3225c75-20260320.png)

これら2つの地点は、上昇傾向から空間的に大きく離れている点で際立っています。両方とも紫色で塗られており、チームが1ゴールを決めたことを示しています。これは、**一般的に、ボールを保持しているほど、チームの選手が賞を受賞する可能性が高まる、と解釈できます。** **しかし、もし彼らが1ゴールしか決めなかった場合、その傾向は逆転し、得点が少ないにもかかわらずボールを保持しすぎたことを理由に、審査員は彼らに減点を与える可能性があります。**

ごく一部の例外を除けば、色によって示される相互作用はここではそれほど劇的ではありません。しかし、時折、それが際立って見えることもあります。

### 依存プロットのコード(Dependence Contribution Plots in Code)
以下のコードで依存関係寄与度プロットを取得できます。summary_plotと異なるのは最後の行のみです。

```python
import shap  # Shap値を計算するためのパッケージ

# shap値を計算できるオブジェクトを作成
explainer = shap.TreeExplainer(my_model)

# shap値を計算。これがこれからプロットするものになります。
shap_values = explainer.shap_values(X)

# プロット作成
shap.dependence_plot('Ball Possession %', shap_values[1], X, interaction_index="Goal Scored")
```
![](https://storage.googleapis.com/zenn-user-upload/3753babcd2a2-20260320.png)

interaction_indexに引数を指定しない場合、Shapleyは独自のロジックを用いて、興味深いと思われるものを選択します。

これには多くのコードを書く必要はありませんでした。しかし、これらの手法のコツは、コードそのものを書くのではなく、結果について批判的に考えることにあります。

### 質問6
