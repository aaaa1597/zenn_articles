## 理論編
### はじめに(Introduction)
前のレッスンでは、畳み込み分類器が 2 つの部分から構成されていることを学びました。

* 畳み込み**ベース**（convolutional base）
* 全結合（Dense）層からなる**ヘッド**

ベースの役割は、画像から視覚的特徴を抽出することです。そしてヘッドは、その特徴を使って画像を分類します。

これから数回のレッスンでは、畳み込み画像分類器のベース部分によく使われる、最も重要な 2 種類の層について学びます。

* ReLU 活性化を持つ畳み込み層
* 最大プーリング層

レッスン5では、これらの層をブロックとして組み合わせ、自分自身の畳み込みネットワーク（convnet）を設計する方法を学びます。

このレッスンでは、ReLU 活性化関数を持つ畳み込み層について扱います。

---

### 特徴抽出(Feature Extraction)

畳み込みの詳細に入る前に、ネットワーク内でこれらの層が果たす役割について考えてみましょう。これから、次の 3 つの操作がどのようにして特徴抽出を実現するかを見ていきます。
特徴抽出は、次の 3 つの基本操作から構成されます：
1. 画像を特定の特徴でフィルタリングする（convolution）
2. フィルタリングされた画像内でその特徴を検出する（ReLU）
3. 特徴を強調するために画像を凝縮する（maximum pooling）

下図では、この 3 つの操作によって元画像の特定の特徴（この例では水平線）がどのように抽出されるかが示されています。
![](https://storage.googleapis.com/zenn-user-upload/62bac7a02917-20260222.png)

通常、ネットワークは 1 枚の画像に対して複数の特徴抽出を並列に行います。
現代の畳み込みネットワークでは、ベースの最終層で 1000 種類以上の視覚的特徴を生成することも珍しくありません。

---

### 畳み込みによるフィルタリング(Filter with Convolution)

畳み込み層は、フィルタリングの処理を行います。
Keras では、例えば次のように定義できます。

```python
from tensorflow import keras
from tensorflow.keras import layers

layers.Conv2D(filters=64, kernel_size=3)  # activation is None
```
これらのパラメータを理解するには、レイヤーの重みと活性化関数との関係を見る必要があります。それでは、実際に見ていきましょう。

#### 重み(Weights)

畳み込みネットワーク(convnet) が訓練中に学習する重みは、主に畳み込み層の含まれています。これらの重みを「**カーネル（kernel）**」と呼びます。カーネルは小さな配列で表されます（例：3×3 行列）。
![](https://storage.googleapis.com/zenn-user-upload/9279f1b131d9-20260222.png)

カーネルは画像上をスキャンしながら、画素値の加重和を計算します。
この働きは、ある種の偏光レンズのようなもので、特定のパターンを強調したり抑えたりします。
![](https://storage.googleapis.com/zenn-user-upload/08fc479c67df-20260222.png)*カーネルは一種のレンズとして機能します。*

カーネルは、ある層と次の層の接続方法を定義します。
たとえば 3×3 カーネルなら、出力の 1 つのニューロンは入力の 9 個のニューロンと接続されます。

`kernel_size` を指定することで、この接続の形を決めます。
多くの場合、(3,3) や (5,5) のような奇数サイズが使われます。中央に 1 つの画素を置けるからです（必須ではありません）。

畳み込み層のカーネルは、どんな特徴を作り出すかを決定します。
学習中、畳み込みネットワーク(convnet) は分類問題を解くのに必要な特徴を見つけるため、最適なカーネルの値を学習します。

---

### 活性(Activations)

ネットワーク内で得られる出力は「**特徴マップ（feature map）**」と呼ばれます。これはフィルタを画像に適用した結果であり、カーネルが抽出した視覚的特徴を含みます。以下に、生成された機能マップとともに表示されたカーネルをいくつか示します。
![](https://storage.googleapis.com/zenn-user-upload/17c6ec67d686-20260222.png)*カーネルと特徴*

カーネル内の数値パターンを見ると、どのような特徴マップが作られるか分かります。一般に、カーネル内の正の値の形状が強調される特徴と対応します。例えば、水平方向に正の値が並んでいれば、水平線を強調します。

`filters` パラメータは、畳み込み層に出力としていくつの特徴マップを作るかを指定します。

---

### ReLU による検出(Detect with ReLU)

フィルタリング後、特徴マップは活性化関数を通ります。**整流関数(rectifier function)** のグラフは次のようになります。
![](https://storage.googleapis.com/zenn-user-upload/4adffacc82cc-20260222.png)*整流関数(ReLU:Rectified Linear Unit)のグラフは、負の部分が 0 に「整流」された線のように見えます。*

ReLU（Rectified Linear Unit）のグラフは、負の部分が 0 に切り取られた直線です。

ReLU を持つニューロンは「正規化線形ユニット(Rectified Linear Unit)」と呼ばれます。そのため、活性化関数は ReLU 活性化、または単に ReLU 関数と呼ばれます。
ReLU アクティベーションは独自のアクティベーション レイヤーで定義できますが、ほとんどの場合、Conv2D のアクティベーション関数として含めるだけです。

```python
model = keras.Sequential([
    layers.Conv2D(filters=64, kernel_size=3, activation='relu')
    # More layers follow
])
```

活性化関数は、画素値の「重要度スコア」を付ける役割と考えられます。ReLU は「負の値は重要でない」と判断し、0 にします。（**「重要でないものは、すべて等しく重要でない」** というわけです。）
下図は上の特徴マップにReLUを適用した例です。特徴の分離に成功していることに注目してください。
![](https://storage.googleapis.com/zenn-user-upload/842559862753-20260222.png)

他の活性化関数と同様に、ReLU は非線形関数です。
本質的には、ネットワーク内のすべての層の合計効果は、単なる足し算では表せません。
この非線形性によって、特徴が層を深く進むにつれて複雑に組み合わさります。（この「特徴の組み合わせ」はレッスン5で詳しく扱います。）

### Example ― 畳み込みと ReLU の適用

この例では、畳み込みネットワーク(convnet) が内部で何をしているのかを理解するため、自分で特徴抽出を行います。
```python
import tensorflow as tf
import matplotlib.pyplot as plt
plt.rc('figure', autolayout=True)
plt.rc('axes', labelweight='bold', labelsize='large',
       titleweight='bold', titlesize=18, titlepad=10)
plt.rc('image', cmap='magma')

image_path = '../input/computer-vision-resources/car_feature.jpg'
image = tf.io.read_file(image_path)
image = tf.io.decode_jpeg(image)

plt.figure(figsize=(6, 6))
plt.imshow(tf.squeeze(image), cmap='gray')
plt.axis('off')
plt.show();
```
この例で使用する画像は次のとおりです。
![](https://storage.googleapis.com/zenn-user-upload/9431b1ad2133-20260222.png)

フィルタリングのステップでは、カーネルを定義し、それを畳み込みに適用します。この場合のカーネルは「エッジ検出」カーネルです。Numpyでnp.arrayを使って配列を定義するのと同じように、tf.constantを使って定義できます。これにより、TensorFlowで使用される種類のテンソルが作成されます。

まず、「エッジ検出」カーネルを定義します。

```python
import tensorflow as tf

kernel = tf.constant([
    [-1, -1, -1],
    [-1,  8, -1],
    [-1, -1, -1],
])

plt.figure(figsize=(3, 3))
show_kernel(kernel)
```
![](https://storage.googleapis.com/zenn-user-upload/3f00c76a9331-20260222.png)

TensorFlow には、ニューラルネットワークでよく使われる演算が `tf.nn` [モジュール](https://www.tensorflow.org/api_docs/python/tf/nn)に含まれています。
ここでは2つ、conv2dとreluを使います。これらはKerasレイヤーの関数バージョンです。

次のpythonコードは、TensorFlow と互換性を持たせるためにフォーマットを変更します。この例では詳細は重要ではありません。
```python
# Reformat for batch compatibility.
image = tf.image.convert_image_dtype(image, dtype=tf.float32)
image = tf.expand_dims(image, axis=0)
kernel = tf.reshape(kernel, [*kernel.shape, 1, 1])
kernel = tf.cast(kernel, dtype=tf.float32)
```

それではカーネルを適用して何が起こるか見てみましょう。
```python
image_filter = tf.nn.conv2d(
    input=image,
    filters=kernel,
    # we'll talk about these two in lesson 4!
    strides=1,
    padding='SAME',
)

plt.figure(figsize=(6, 6))
plt.imshow(tf.squeeze(image_filter))
plt.axis('off')
plt.show();
```

次に ReLU を適用します。この関数は、設定するパラメータがないため、畳み込みよりもはるかにシンプルです。

```python
image_detect = tf.nn.relu(image_filter)

plt.figure(figsize=(6, 6))
plt.imshow(tf.squeeze(image_detect))
plt.axis('off')
plt.show();
```
![](https://storage.googleapis.com/zenn-user-upload/51903be8312d-20260222.png)

こうして特徴マップが作られました！このような画像が、ヘッドが分類問題を解決するために使用するものです。例えば、ある特徴は「Car」に特徴的で、別の特徴は「Truck」に特徴的かもしれません。畳み込みネットワーク(convnet) の学習の目的は、それらの特徴を見つけ出すカーネルを作ることです。
