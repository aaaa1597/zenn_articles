## 理論編
### はじめに(Introduction)
レッスン2では、convnet のベース部分がどのように特徴抽出を行うのかについて学び始めました。その過程の最初の 2 つの操作が、ReLU 活性化を持つ Conv2D 層で行われることを学びました。

このレッスンでは、その一連の処理の 3 つ目（そして最後）の操作を見ていきます。それが「**最大プーリング**による**凝縮（condense）**」です。Keras では **MaxPool2D** 層で実装します。

---

### 最大プーリングによる凝縮(Condense with Maximum Pooling)

先ほどのモデルに「凝縮」ステップを追加すると、次のようになります。

```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.Conv2D(filters=64, kernel_size=3),  # activation is None
    layers.MaxPool2D(pool_size=2),
    # More layers follow
])
```

MaxPool2D 層は Conv2D 層とよく似ていますが、カーネルの代わりに単純な「最大値」関数を使います。`pool_size` は `kernel_size` に対応するパラメータです。ただし、MaxPool2D には Conv2D のような学習可能な重み（カーネル）はありません。

前のレッスンの特徴抽出の図をもう一度見てみましょう。MaxPool2D は **「Condense（凝縮）」** の ステップであることを覚えておいてください。
![](https://storage.googleapis.com/zenn-user-upload/70332e5fb816-20260222.png)

ReLU を適用した後の特徴マップには、多くの「デッドスペース」（0 だけの領域）ができます。これらの 0 の活性をネットワーク全体に渡って保持するのは、モデルを大きくするだけで有用な情報はほとんど増えません。そこで、特徴そのものだけを残すように特徴マップを凝縮したいわけです。

まさに、**最大プーリング**はそれを行います。最大プーリングは、元の特徴マップ内のパッチ(小さな領域)を取得し、そのパッチ内の最大値だけを残します。
![](https://storage.googleapis.com/zenn-user-upload/3a96f66bd4dc-20260222.png)

つまり、各パッチを、その中の最大値で置き換えます。
ReLU の後に適用すると、特徴を「強調」する効果があります。アクティブなピクセルの割合が増え、ゼロとの比率が高まります。

---

### Example ― 最大プーリングの適用
レッスン2で行った特徴抽出に「凝縮」ステップを追加してみます。
準備コードは下記です。
```python
import tensorflow as tf
import matplotlib.pyplot as plt
import warnings

plt.rc('figure', autolayout=True)
plt.rc('axes', labelweight='bold', labelsize='large',
       titleweight='bold', titlesize=18, titlepad=10)
plt.rc('image', cmap='magma')
warnings.filterwarnings("ignore") # to clean up output cells

# Read image
image_path = '../input/computer-vision-resources/car_feature.jpg'
image = tf.io.read_file(image_path)
image = tf.io.decode_jpeg(image)

# Define kernel
kernel = tf.constant([
    [-1, -1, -1],
    [-1,  8, -1],
    [-1, -1, -1],
], dtype=tf.float32)

# Reformat for batch compatibility.
image = tf.image.convert_image_dtype(image, dtype=tf.float32)
image = tf.expand_dims(image, axis=0)
kernel = tf.reshape(kernel, [*kernel.shape, 1, 1])

# Filter step
image_filter = tf.nn.conv2d(
    input=image,
    filters=kernel,
    # we'll talk about these two in the next lesson!
    strides=1,
    padding='SAME'
)

# Detect step
image_detect = tf.nn.relu(image_filter)

# Show what we have so far
plt.figure(figsize=(12, 6))
plt.subplot(131)
plt.imshow(tf.squeeze(image), cmap='gray')
plt.axis('off')
plt.title('Input')
plt.subplot(132)
plt.imshow(tf.squeeze(image_filter))
plt.axis('off')
plt.title('Filter')
plt.subplot(133)
plt.imshow(tf.squeeze(image_detect))
plt.axis('off')
plt.title('Detect')
plt.show();
```

プーリングステップを適用するために、tf.nn の別の関数 tf.nn.pool を使用します。これはモデル構築時に使用する MaxPool2D レイヤーと同じ機能を持つ TensorFlow 関数ですが、シンプルな関数なので直接使用する方が簡単です。

```python
import tensorflow as tf

image_condense = tf.nn.pool(
    input=image_detect,
    window_shape=(2, 2),
    pooling_type='MAX',
    strides=(2, 2),
    padding='SAME',
)

plt.figure(figsize=(6, 6))
plt.imshow(tf.squeeze(image_condense))
plt.axis('off')
plt.show();
```

すごいですね！プーリングによって、活性の強い部分を中心に画像が凝縮され、特徴がよりはっきりしたことがおわかりいただけたかと思います。

### 平行移動不変性(Translation Invariance)

0 のピクセルは「重要でない」と呼びましたが、本当にまったく情報を持っていないのでしょうか？実は、0 のピクセルは位置情報を持っています。空白部分によって、特徴が画像内のどこにあるかが分かります。

MaxPool2D は一部のピクセルを取り除くため、特徴マップ内の位置情報も一部失われます。
これにより、convnet は「平行移動不変性（translation invariance）」という性質を持ちます。(平行移動とは、形や大きさを変えずに、位置だけを変えることです。)

次の特徴マップに対して最大プーリングを繰り返し適用すると何が起こるかを見てみましょう。
![](https://storage.googleapis.com/zenn-user-upload/8c33bc6f2ac9-20260222.png)

最大プーリングを持つ convnet は、特徴の「位置」にあまり敏感でなくなります。
例えば、2 つの点がある画像に最大プーリングを何度も適用すると、最終的にそれらは区別できなくなります。

つまり：
* 位置情報の一部が失われる
* ネットワークはその位置の違いを区別できなくなる
* その違いに対して「不変」になる

ただし、この不変性は小さな距離に対してのみ起こります。最初から遠く離れた特徴は、プーリング後も区別可能です。位置情報の一部のみが失われ、すべてが失われるわけではありません。

![](https://storage.googleapis.com/zenn-user-upload/cf12a4acb2c2-20260222.png)

特徴の位置の小さな違いに対する不変性は、画像分類器にとって優れた特性です。
**なぜ良い性質なのか？**
画像分類において、同じ特徴が、少し上にあったり、少し横にずれていたりすることはよくあります（視点や構図の違いによる）。それでも、それらを「同じ特徴」と認識してほしいわけです。

この不変性がネットワークに組み込まれているため、位置の違いを無視するようにわざわざ大量のデータで学習させる必要が減ります。

これにより、畳み込みネットワークは、Dense 層だけのネットワークに比べて、大きな効率的優位性を得られます。（レッスン6では、**Data Augmentation** によって不変性を得る別の方法も学びます。）

### グローバル平均プーリング(Global Average Pooling)

前の演習で、畳み込みベース部分では平均プーリングは最大プーリングにほぼ取って代わられている、と述べました。しかし、平均プーリングの一種で、今でも広く使われているものがあります。それが**グローバル平均プーリング（Global Average Pooling）** です。

`GlobalAvgPool2D` 層は、ネットワークのヘッド部分にある一部またはすべての隠れ Dense 層の代わりとして使われることがあります。例えば次のようにです：

```python
model = keras.Sequential([
    pretrained_base,
    layers.GlobalAvgPool2D(),
    layers.Dense(1, activation='sigmoid'),
])
```

この層は何をしているのでしょうか？通常、ベースの後には `Flatten` 層を置いて、2次元の特徴データを分類器が必要とする1次元データに変換します。しかしここでは `Flatten` 層がありません。代わりに `GlobalAvgPool2D` 層がその役割を担っています。ただし `Flatten` のように特徴を「展開（unstack）」するのではなく、**各特徴マップ全体をその平均値で置き換えます。** かなり情報を削減する操作ですが、うまく機能することが多く、さらに**モデルのパラメータ数を削減できる**という利点があります。

ランダムに生成した特徴マップに対して `GlobalAvgPool2D` が何をするのか見てみましょう。これは、ベースが出力した特徴マップのスタックをどのように「平坦化」できるのか理解するのに役立ちます。

```python
feature_maps = [visiontools.random_map([5, 5], scale=0.1, decay_power=4) for _ in range(8)]

gs = gridspec.GridSpec(1, 8, wspace=0.01, hspace=0.01)
plt.figure(figsize=(18, 2))
for i, feature_map in enumerate(feature_maps):
    plt.subplot(gs[i])
    plt.imshow(feature_map, vmin=0, vmax=1)
    plt.axis('off')
plt.suptitle('Feature Maps', size=18, weight='bold', y=1.1)
plt.show()

# reformat for TensorFlow
feature_maps_tf = [tf.reshape(feature_map, [1, *feature_map.shape, 1])
                   for feature_map in feature_maps]

global_avg_pool = tf.keras.layers.GlobalAvgPool2D()
pooled_maps = [global_avg_pool(feature_map) for feature_map in feature_maps_tf]
img = np.array(pooled_maps)[:,:,0].T

plt.imshow(img, vmin=0, vmax=1)
plt.axis('off')
plt.title('Pooled Feature Maps')
plt.show();
```

### 3) プールされた特徴を理解する(Understand the Pooled Features)

VGG16 ベースは 512 個の特徴マップを生成します。各特徴マップは、ホイールやウィンドウなど、元の画像内の高レベルの視覚的特徴を表すものと考えることができます。マップをプールすると、その特徴のスコアとして考えることができる単一の数値が得られます。特徴が存在する場合は大きく、存在しない場合は小さくなります。車はある特徴セットで高得点を獲得する傾向があり、トラックは別の機能セットで高得点を獲得する傾向があります。これで、生の特徴をクラスにマッピングしようとする代わりに、ヘッドは GlobalAvgPool2D が生成したこれらのスコアのみで作業すればよくなり、解決がはるかに容易になります。

グローバル平均プーリングは、現代の畳み込みネットワークでよく使われています。大きな利点の1つは、**モデルのパラメータ数を大幅に削減できること**です。しかも、ある特徴が画像に存在したかどうかを判断するには十分な情報を残します。分類問題では、通常それで十分なのです。
