## はじめての SQL と BigQuery
BigQuery と SQL を使用して大規模なデータセットを処理するワークフローを学びます。

## 理論編
### はじめに(Introduction)
構造化問い合わせ言語（Structured Query Language）、すなわち SQL は、データベースを扱うためのプログラミング言語であり、データサイエンティストにとって重要なスキルです。本コースでは、巨大なデータセットに対してSQLを適用できるウェブサービス BigQuery を使いながら、SQLのスキルを身につけていきます。

このレッスンでは、BigQueryのデータセットにアクセスし、内容を確認するための基本を学びます。これらの基礎を理解した後で、あらためてSQLのスキル構築に戻ってきます。

### 最初のBigQueryコマンド
BigQueryを使うために、まず以下のPythonパッケージをインポートします。

```python
from google.cloud import bigquery
```

ワークフローの最初のステップは、Clientオブジェクトを作成することです。これからわかるように、このClientオブジェクトはBigQueryのデータセットから情報を取得するうえで中心的な役割を果たします。

```python
# "Client" オブジェクトを作成
client = bigquery.Client()
```
Kaggleの公開BigQuery統合を使用する

今回は、コンピュータサイエンスやサイバーセキュリティのニュースを扱うウェブサイト [Hacker News](https://news.ycombinator.com/) の投稿データセットを使います。

BigQueryでは、各データセットは対応する プロジェクト に含まれています。この場合、hacker_news データセットは bigquery-public-data プロジェクト内にあります。

データセットにアクセスするには：
- dataset() メソッドを使ってデータセットへの参照を作成します。
- その参照を get_dataset() メソッドに渡して、データセットを取得します。
```python
# "hacker_news" データセットへの参照を作成
dataset_ref = client.dataset("hacker_news", project="bigquery-public-data")

# APIリクエスト - データセットを取得
dataset = client.get_dataset(dataset_ref)
```

すべてのデータセットは、単なる テーブルの集合 です。データセットは、複数のテーブル（行と列で構成）を含むスプレッドシートファイルのようなものと考えることができます。

list_tables() メソッドを使うと、データセット内のテーブル一覧を取得できます。
```python
# "hacker_news" データセット内のすべてのテーブルを一覧表示
tables = list(client.list_tables(dataset))

# データセット内のすべてのテーブル名を表示（全部で4つあります！）
for table in tables:  
    print(table.table_id)
```

**出力：**
comments  
full  
full_201510  
stories  

:::message
client.dataset()には、ローカルファイルを設定することはできない!
そもそも、BigQueryのDatasetReference の仕組みは、クラウド上の BigQuery リソースを使う想定らしく、ローカルファイルをソースにしたい場合は一旦クラウドに上げる必要があるらしい。

- ローカルデータ("local_data.csv")を使用するときのサンプルコード
```python
from google.cloud import bigquery

client = bigquery.Client()
table_id = "my-project.my_dataset.my_table"
job_config = bigquery.LoadJobConfig(
    source_format=bigquery.SourceFormat.CSV,
    autodetect=True,
)

with open("local_data.csv", "rb") as f:
    load_job = client.load_table_from_file(f, table_id, job_config=job_config)

load_job.result()
```
ちょっとすっきりした!
:::

```python
table_ref = dataset_ref.table("full")

table = client.get_table(table_ref)
```

### テーブルスキーマ

テーブルの構造は スキーマ（schema） と呼ばれます。欲しいデータを効果的に取り出すには、スキーマを理解する必要があります。

ここでは、先ほど取得した full テーブルを調べてみます。
```python
# "hacker_news" データセットの "full" テーブルのすべての列情報を表示
table.schema
```

各 SchemaField は、特定の列（フィールド）についての情報を示しています。順番に、次の情報が含まれています：
- 列名
- 列のデータ型（datatype）
- 列のモード（NULLABLE は NULL値を許可することを意味し、デフォルト）
- その列に含まれるデータの説明

たとえば、次の SchemaField：
SchemaField('by', 'string', 'NULLABLE', "The username of the item's author.",())

これは次のことを意味します：
- 列名は by
- データ型は文字列（string）
- NULL値を許可している
- 各アイテムの投稿者のユーザー名が含まれている

list_rows() メソッドを使えば、full テーブルの最初の5行を確認できます。（データベースの説明が古い場合もあるため、実際に確認するのは良い習慣です。） これは BigQuery の RowIterator オブジェクトを返し、to_dataframe() メソッドで簡単に pandas の DataFrame に変換できます。
```python
# "full" テーブルの最初の5行をプレビュー
client.list_rows(table, max_results=5).to_dataframe()
```

### 注意事項
演習に入る前に、すでにSQLを知っている方向けの注意です。
**Kaggleユーザーは、30日ごとに5TBまで無料でスキャンできます。この上限に達すると、リセットされるまで待つ必要があります。**

ここまでに紹介したコマンドでは、その上限のごく一部しか使用しません。しかし、BigQueryのデータセットの中には非常に巨大なものもあります。すでにSQLを知っている場合は、割り当て量を効率的に使う方法を学ぶまで、SELECT クエリを実行するのは待ちましょう。もしあなたがこの記事を読んでいる多くの人と同様に、まだSQLクエリの書き方を知らないのであれば、この注意事項について心配する必要はありません。
