### データの読み込み
このノートブックでは、アルゼンチン(ARG)、ブラジル(BRA)、スペイン(ESP)、フランス(FRA)、ドイツ(GER)、イタリア(ITA)の6カ国の「FIFAランキング履歴」のデータセットを扱います。データは CSVファイル（カンマ区切りファイル）として保存されています。このファイルを Excel で開くと、行に「日付」、列に「国名」が並んでいるのが確認できます。
![](https://storage.googleapis.com/kaggle-media/learn/images/W0E7GjV.png)

ノートブックにデータを読み込むには、以下の2ステップを行います。
- データセットがある場所（ファイルパス）を指定する。
- そのパスを使って、データの中身をノートブックに読み込む。

```python
# 読み込むファイルのパス
fifa_filepath = "../input/fifa.csv"

# 変数 fifa_data にファイルを読み込む
fifa_data = pd.read_csv(fifa_filepath, index_col="Date", parse_dates=True)
```

![](https://storage.googleapis.com/kaggle-media/learn/images/I6UEDSK.png)

上記のコードセルには4つの行があります。

#### コメント（Comments）
2つの行はシャープ記号（#）で始まり、文字が薄く斜体になっています。これらはコンピュータには完全に無視され、人間がコードを読んで理解するためだけに存在します。これを「コメント」と呼び、コードを分かりやすくするために書くのが良い習慣です。

#### 実行コード（Executable code）
残りの2行は、コンピュータによって実行されるコードです。1行目：fifa_filepath の値を、データの場所に設定します。2行目：pd.read_csv を使って、fifa_data に全ての情報を格納します。括弧内の指示（引数）で読み込み方をカスタマイズしています。

- **fifa_filepath**：最初にファイルパスを伝えます。
- **index_col="Date"：** 1列目（Date）を行のラベルとして使うよう指定します。
- **parse_dates=True：** 各行のラベルを「日付」として認識させます。

これらの詳細は、後ほど実際に自分のデータセットを読み込んで操作する演習を行うと、より理解しやすくなるはずです。

今の時点で重要なのは、2行のコードを実行した最終的な結果として、ノートブック内で fifa_data を使ってデータセットにアクセスできるようになった、という点です。

ちなみに、これらのコード行は何も出力を返していないことに気づいたかもしれません（先ほど実行したコードでは「Setup Complete」と表示されていましたよね）。これは正常な動作です。すべてのコードが出力を返すわけではなく、今回のコードはまさにその代表例です。

次に、コメントと実行コードの違いを思い出してください：

コメントは、ハッシュ記号（#）が前に付いており、文字が薄く斜体で表示されています。これらはコードが実行される際、コンピュータによって完全に無視されます。 実行コードは、コンピュータによって実行されるコードのことです。

下のコードセルでは、すべての行がコメントになっています：

```python
# 下の行のコメントを解除するとヒントを受け取れます
#step_2.hint()
#step_2.solution()
```

もし下のコードセルを何も変更せずに実行した場合、何の出力も返されません。今すぐこれを試してみてください！

次に、step_2.hint() の前にあるハッシュ記号（#）を削除して、上のコードセルが次のように表示されるようにしてください。

```python
# 下の行のコメントを解除するとヒントを受け取れます
step_2.hint()
#step_2.solution()
```
コードの行の前にあるハッシュ記号を取り除くことを、その行を「アンコメント（uncomment）」すると言います。これにより、コメントがコンピュータによって実行される「実行コード」の行へと変わります。今すぐこのコードセルを実行してください。出力として「Hint（ヒント）」が返されるはずです。

最後に、解決策（ソリューション）を見るためにその行をアンコメントし、コードセルが次のように表示されるようにします。

```python
# 下の行のコメントを解除するとヒントを受け取れます
step_2.hint()
step_2.solution()
```
その後、コードセルを実行してください。これで「Hint（ヒント）」と「Solution（解決策）」の両方を受け取ることができます。

もし、問題に対する正解を考える際に行き詰まってしまった場合は、いつでもセルを完成させる前にヒントや解決策を取得して構いません。（つまり、ヒントや解決策を表示させるコードを実行するために、あらかじめ「Correct（正解）」の結果を得ている必要はないということです。）
