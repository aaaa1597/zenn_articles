## 理論編
### はじめに(Introduction)
これまでに、convnet が特徴抽出のために使う各レイヤーを見てきました。いよいよ、それらを組み合わせて自分自身のネットワークを構築する番です！

### 単純から洗練へ(Simple to Refined)

直近の3つのレッスンでは、畳み込みネットワークが **filter（フィルタリング）・detect（検出）・condense（集約）** という3つの操作を通じて特徴抽出を行うことを見ました。

1回の特徴抽出だけでは、画像から取り出せるのは単純な線やコントラストのような、比較的シンプルな特徴に限られます。しかし、これらは多くの分類問題を解くには単純すぎます。そこで、convnet はこの抽出処理を何度も繰り返します。そうすることで、ネットワークの深い層へ進むにつれて、特徴はより複雑で洗練されたものへと変化していきます。

![](https://storage.googleapis.com/zenn-user-upload/fb341813c029-20260223.png)

### 畳み込みブロック(Convolutional Blocks)

この処理は、長い畳み込みブロックの連鎖を通して行われます。
![](https://storage.googleapis.com/zenn-user-upload/5969bdd08ec2-20260223.png)

これらの畳み込みブロックは、`Conv2D` レイヤーと `MaxPool2D` レイヤーを積み重ねた構造になっています。これらのレイヤーが特徴抽出に果たす役割については、ここ数レッスンで学びました。
![](https://storage.googleapis.com/zenn-user-upload/2477ebc2a836-20260223.png)

各ブロックは1回分の特徴抽出を表しています。そして、これらのブロックを組み合わせることで、convnet は生成された特徴を組み合わせたり再構成したりしながら、問題により適した形へと発展させていきます。現代の convnet が持つこの「深い構造」こそが、高度な特徴量エンジニアリングを可能にし、その優れた性能の大きな要因となっています。

### 例：Convnet を設計する(Design a Convnet)

複雑な特徴を設計できる深い畳み込みネットワークをどのように定義するか見ていきましょう。この例では、Keras の Sequential モデルを作成し、それを Cars データセットで訓練します。

### Step 1 データの読み込み(Load Data)
下記コードでデータを読み込みます。

```python
# Imports
import os, warnings
import matplotlib.pyplot as plt
from matplotlib import gridspec

import numpy as np
import tensorflow as tf
from tensorflow.keras.preprocessing import image_dataset_from_directory

# Reproducability
def set_seed(seed=31415):
    np.random.seed(seed)
    tf.random.set_seed(seed)
    os.environ['PYTHONHASHSEED'] = str(seed)
    os.environ['TF_DETERMINISTIC_OPS'] = '1'
set_seed()

# Set Matplotlib defaults
plt.rc('figure', autolayout=True)
plt.rc('axes', labelweight='bold', labelsize='large',
       titleweight='bold', titlesize=18, titlepad=10)
plt.rc('image', cmap='magma')
warnings.filterwarnings("ignore") # to clean up output cells


# Load training and validation sets
ds_train_ = image_dataset_from_directory(
    '../input/car-or-truck/train',
    labels='inferred',
    label_mode='binary',
    image_size=[128, 128],
    interpolation='nearest',
    batch_size=64,
    shuffle=True,
)
ds_valid_ = image_dataset_from_directory(
    '../input/car-or-truck/valid',
    labels='inferred',
    label_mode='binary',
    image_size=[128, 128],
    interpolation='nearest',
    batch_size=64,
    shuffle=False,
)

# Data Pipeline
def convert_to_float(image, label):
    image = tf.image.convert_image_dtype(image, dtype=tf.float32)
    return image, label

AUTOTUNE = tf.data.experimental.AUTOTUNE
ds_train = (
    ds_train_
    .map(convert_to_float)
    .cache()
    .prefetch(buffer_size=AUTOTUNE)
)
ds_valid = (
    ds_valid_
    .map(convert_to_float)
    .cache()
    .prefetch(buffer_size=AUTOTUNE)
)
```

### Step 2 - Define Model（モデルの定義）

使用するモデルの図は次の通りです。
![](https://storage.googleapis.com/zenn-user-upload/2e07ea082ab0-20260223.png)

ではモデルを定義します。このモデルは、3つの `Conv2D` + `MaxPool2D` ブロック（ベース部分）と、その後に続く Dense レイヤーのヘッド部分で構成されています。図を、適切なパラメータを埋めることで、ほぼそのまま Keras の Sequential モデルに変換できます。

```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([

    # First Convolutional Block
    layers.Conv2D(filters=32, kernel_size=5, activation="relu", padding='same',
                  # give the input dimensions in the first layer
                  # [height, width, color channels(RGB)]
                  input_shape=[128, 128, 3]),
    layers.MaxPool2D(),

    # Second Convolutional Block
    layers.Conv2D(filters=64, kernel_size=3, activation="relu", padding='same'),
    layers.MaxPool2D(),

    # Third Convolutional Block
    layers.Conv2D(filters=128, kernel_size=3, activation="relu", padding='same'),
    layers.MaxPool2D(),

    # Classifier Head
    layers.Flatten(),
    layers.Dense(units=6, activation="relu"),
    layers.Dense(units=1, activation="sigmoid"),
])
model.summary()
```

### Convnet を設計する

チュートリアルで見たような、ブロック構造を持つ畳み込みネットワークを設計してみましょう。
例として示されたモデルは、各ブロックに 1 つの畳み込み層を持つ 3 ブロック構成でした。このモデルの「Car or Truck」問題に対する性能はまずまずでしたが、事前学習済みの **VGG16** が達成した性能には遠く及びませんでした。単純なネットワークでは、十分に複雑な特徴を抽出する能力が不足している可能性があります。
改善方法としては、ブロックの数を増やすか、既存の各ブロック内の畳み込み層を増やすことが考えられます。

今回は後者の方法を採用します。3 ブロック構造は維持しつつ、第2ブロックの Conv2D 層を **2 層**に、第3ブロックの Conv2D 層を **3 層** に増やします。

![](https://storage.googleapis.com/zenn-user-upload/a2676e6e12e5-20260224.png)

#### 1) モデルを定義する

上の図を参考に、第3ブロックの層を定義してモデルを完成させてください。
```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    # Block One
    layers.Conv2D(filters=32, kernel_size=3, activation='relu', padding='same',
                  input_shape=[128, 128, 3]),
    layers.MaxPool2D(),

    # Block Two
    layers.Conv2D(filters=64, kernel_size=3, activation='relu', padding='same'),
    layers.MaxPool2D(),

    # Block Three
    # YOUR CODE HERE
    layers.Conv2D(filters=128, kernel_size=3, activation='relu', padding='same'),
    layers.Conv2D(filters=128, kernel_size=3, activation='relu', padding='same'),
    layers.MaxPool2D(),

    # Head
    layers.Flatten(),
    layers.Dense(6, activation='relu'),
    layers.Dropout(0.2),
    layers.Dense(1, activation='sigmoid'),
])
```
