## 理論編
### はじめに(Introduction)
前回のチュートリアルでは、1手先読み（one-step lookahead）を使ったエージェントの作り方を学びました。このエージェントはそれなりに良い性能を発揮しますが、まだ改善の余地があります。例えば、下の図にあるような手を考えてみましょう。（列は0から始まる番号で表しており、一番左の列がcol=0、その次がcol=1…となります。）
![](https://storage.googleapis.com/zenn-user-upload/899b6b3a506f-20260324.png)

1手先読みでは、赤プレイヤーは列5か6のどちらかを50%の確率で選びます。しかし、列5は明らかに悪手で、相手に次の1手で勝たれてしまいます。残念ながら、このエージェントは1手先までしか見られないため、それに気づけません。

このチュートリアルでは、**minimaxアルゴリズム**を使って、エージェントがより先の未来まで見通し、より良い意思決定ができるようにします。

### Minimax
ゲーム木のより深い情報を活用したいと考えます。ここでは深さ3を考えます。つまり、エージェントは自分の手を決める際に、次のすべての可能な盤面を考慮します：
1. エージェントの手
2. 相手の手
3. エージェントの次の手

視覚的な例を用いて説明しましょう。簡単のため、各ターンでエージェントと相手がそれぞれ2通りの手しか持たないと仮定します。図の青い四角は、それぞれ異なる盤面を表しています。
![](https://storage.googleapis.com/zenn-user-upload/dcb0ddba85a2-20260324.png)

木の一番下にある「葉ノード」には、Heuristics(評価関数)によるスコアが付けられています（図では仮のスコアを使用しています。コードでは、前回のチュートリアルと同じHeuristics(評価関数)を使用します。）。これまでと同様、現在の盤面は一番上にあり、エージェントの目標はスコアをできるだけ高くすることです。

ただし重要なのは、エージェントはもはやスコアを完全にコントロールできないという点です。エージェントが手を打った後、相手も手を選び、その選択がエージェントにとって不利になる可能性があります。

具体的には：

* 左の分岐を選ぶと、相手はスコアを-1にすることができる
* 右の分岐を選ぶと、相手はスコアを+10にすることができる

ここは一度図を見て、自分で確認してみてください。

この状況では、右の分岐の方がより良い選択だと考えられます。なぜなら、左では+40という高得点の可能性はあるものの、最悪-1になってしまいます。一方で右は少なくとも+10を保証します。

これが**minimaxアルゴリズム**の基本的な考え方です：

* エージェントはスコアを最大化しようとする
* 相手はそれに対抗してスコアを最小化しようとする

つまり、両者は対立した目的を持ち、相手も最善手を打つと仮定するのがポイントです。

では、実際にはエージェントはこの仮定をどのように利用して行動を選択するのでしょうか？エージェントの思考プロセスを以下の図に示します。
![](https://storage.googleapis.com/zenn-user-upload/67416fe62802-20260324.png)

この例では、ミニマックス法は左側の行動に-1点、右側の行動に+10点のスコアを割り当てます。したがって、エージェントは右側の行動を選択します。

### Code(コード)
前回のチュートリアルの関数をいくつか再利用します。これらの関数は、以下のコードセルに定義されています。
```python
import random
import numpy as np

# Gets board at next step if agent drops piece in selected column
def drop_piece(grid, col, mark, config):
    next_grid = grid.copy()
    for row in range(config.rows-1, -1, -1):
        if next_grid[row][col] == 0:
            break
    next_grid[row][col] = mark
    return next_grid

# Helper function for get_heuristic: checks if window satisfies heuristic conditions
def check_window(window, num_discs, piece, config):
    return (window.count(piece) == num_discs and window.count(0) == config.inarow-num_discs)
    
# Helper function for get_heuristic: counts number of windows satisfying specified heuristic conditions
def count_windows(grid, num_discs, piece, config):
    num_windows = 0
    # horizontal
    for row in range(config.rows):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[row, col:col+config.inarow])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    # vertical
    for row in range(config.rows-(config.inarow-1)):
        for col in range(config.columns):
            window = list(grid[row:row+config.inarow, col])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    # positive diagonal
    for row in range(config.rows-(config.inarow-1)):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[range(row, row+config.inarow), range(col, col+config.inarow)])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    # negative diagonal
    for row in range(config.inarow-1, config.rows):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[range(row, row-config.inarow, -1), range(col, col+config.inarow)])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    return num_windows
```

また、今回は相手も盤面に影響を与えるため、Heuristics(評価関数)を少し修正します。
![](https://storage.googleapis.com/zenn-user-upload/846b5b0a2e27-20260324.png)
特に、相手がディスクをプレイしてゲームに勝利したかどうかを確認する必要があります。新しいHeuristics(評価関数)では、4つ並びの各パターンについて以下のスコアを与えます：

* エージェントが4つ並べた → +1000000（勝利）
* エージェントが3つ並べて1マス空き → +1
* 相手が3つ並べて1マス空き → -100
* 相手が4つ並べた → -10000（敗北）

以下のpythonコードで定義されます。

```python
def get_heuristic(grid, mark, config):
    num_threes = count_windows(grid, 3, mark, config)
    num_fours = count_windows(grid, 4, mark, config)
    num_threes_opp = count_windows(grid, 3, mark%2+1, config)
    num_fours_opp = count_windows(grid, 4, mark%2+1, config)
    score = num_threes - 1e2*num_threes_opp - 1e4*num_fours_opp + 1e6*num_fours
    return score
```
次のコードセルでは、Minimaxエージェントに必要な追加関数をいくつか定義します。
```python
# Uses minimax to calculate value of dropping piece in selected column
def score_move(grid, col, mark, config, nsteps):
    next_grid = drop_piece(grid, col, mark, config)
    score = minimax(next_grid, nsteps-1, False, mark, config)
    return score

# Helper function for minimax: checks if agent or opponent has four in a row in the window
def is_terminal_window(window, config):
    return window.count(1) == config.inarow or window.count(2) == config.inarow

# Helper function for minimax: checks if game has ended
def is_terminal_node(grid, config):
    # Check for draw 
    if list(grid[0, :]).count(0) == 0:
        return True
    # Check for win: horizontal, vertical, or diagonal
    # horizontal 
    for row in range(config.rows):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[row, col:col+config.inarow])
            if is_terminal_window(window, config):
                return True
    # vertical
    for row in range(config.rows-(config.inarow-1)):
        for col in range(config.columns):
            window = list(grid[row:row+config.inarow, col])
            if is_terminal_window(window, config):
                return True
    # positive diagonal
    for row in range(config.rows-(config.inarow-1)):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[range(row, row+config.inarow), range(col, col+config.inarow)])
            if is_terminal_window(window, config):
                return True
    # negative diagonal
    for row in range(config.inarow-1, config.rows):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[range(row, row-config.inarow, -1), range(col, col+config.inarow)])
            if is_terminal_window(window, config):
                return True
    return False

# Minimax implementation
def minimax(node, depth, maximizingPlayer, mark, config):
    is_terminal = is_terminal_node(node, config)
    valid_moves = [c for c in range(config.columns) if node[0][c] == 0]
    if depth == 0 or is_terminal:
        return get_heuristic(node, mark, config)
    if maximizingPlayer:
        value = -np.Inf
        for col in valid_moves:
            child = drop_piece(node, col, mark, config)
            value = max(value, minimax(child, depth-1, False, mark, config))
        return value
    else:
        value = np.Inf
        for col in valid_moves:
            child = drop_piece(node, col, mark%2+1, config)
            value = min(value, minimax(child, depth-1, True, mark, config))
        return value
```

最後に、Minimaxエージェントを競技形式で実装します。N_STEPS変数は、ツリーの深さを設定するために使用されます。
```python
# How deep to make the game tree: higher values take longer to run!
N_STEPS = 3

def agent(obs, config):
    # Get list of valid moves
    valid_moves = [c for c in range(config.columns) if obs.board[c] == 0]
    # Convert the board to a 2D grid
    grid = np.asarray(obs.board).reshape(config.rows, config.columns)
    # Use the heuristic to assign a score to each possible board in the next step
    scores = dict(zip(valid_moves, [score_move(grid, col, obs.mark, config, N_STEPS) for col in valid_moves]))
    # Get a list of columns (moves) that maximize the heuristic
    max_cols = [key for key in scores.keys() if scores[key] == max(scores.values())]
    # Select at random from the maximizing columns
    return random.choice(max_cols)
```

def get_win_percentages(agent1, agent2, n_rounds=100):
    # Use default Connect Four setup
    config = {'rows': 6, 'columns': 7, 'inarow': 4}
    # Agent 1 goes first (roughly) half the time          
    outcomes = evaluate("connectx", [agent1, agent2], config, [], n_rounds//2)
    # Agent 2 goes first (roughly) half the time      
    outcomes += [[b,a] for [a,b] in evaluate("connectx", [agent2, agent1], config, [], n_rounds-n_rounds//2)]
    print("Agent 1 Win Percentage:", np.round(outcomes.count([1,-1])/len(outcomes), 2))
    print("Agent 2 Win Percentage:", np.round(outcomes.count([-1,1])/len(outcomes), 2))
    print("Number of Invalid Plays by Agent 1:", outcomes.count([None, 0]))
    print("Number of Invalid Plays by Agent 2:", outcomes.count([0, None]))

### 4. 仮定を検証する
minimaxエージェントは、相手が最適に行動すると仮定しています（Heuristics(評価関数)に基づき、かつ制限された深さのゲーム木を使用）。しかし実際には、これはほとんどの場合成り立ちません。現実には、最適ではない（つまり最適よりも劣る）相手に遭遇することの方がはるかに多いです。

では、minimaxエージェントがそのような非最適な相手に遭遇した場合でも、うまくプレイできると期待できるでしょうか？ もしそうなら、それはなぜでしょうか？

ヒント: チュートリアルでminimaxエージェントがランダムな対戦相手と対戦したとき、何が起こりましたか？
答え: minimaxエージェントは依然として優れた性能を発揮すると期待できます。大まかに言えば、最適な対戦相手を想定しても、単に相手を過大評価するだけで、アルゴリズムが破綻するわけではないです。相手を過大評価することによる影響は、minimaxエージェントが相手をより正確に理解していた場合よりも、勝利するまでに時間がかかるという点に過ぎません。例えば、minimaxエージェントは最初の3手で同じ列を3回選ぶことはほとんどありません（なぜなら、次の手で確実に勝ちを阻止してくる最適な相手を想定しているためです）。しかし、列を完全にランダムに選ぶエージェントと対戦する場合には、これは決して悪い初期戦略ではありません。
