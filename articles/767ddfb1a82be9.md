## 理論編
### はじめに(Introduction)
コネクトフォーが初めてでも、おそらくいくつかの戦略はすでに考えたことがあるでしょう。このチュートリアルでは、**Heuristics（評価関数）** を使って、その知識をエージェントに共有する方法を学びます。

### ゲーム木(Game trees)
人間はゲームをプレイするとき、どのように考えているでしょうか？複数の手をどう比較していますか？

おそらく、先読みをしています。各候補の手について、相手がどう応じるかを予測し、その後に自分がどう対応するか、さらに相手がどうするか…という流れを考えます。そして、最も勝ちにつながりそうな手を選びます。

この考え方は形式化でき、すべての可能な結果を **(完全な)ゲーム木** として表現できます。
![](https://storage.googleapis.com/zenn-user-upload/7caf8aeb2c0a-20260322.png)

ゲーム木は、空の盤面から始まり、エージェントと相手のすべての可能な手を表します。最初の行には、エージェント（赤）が打てるすべての手が並びます。次に、それぞれに対して相手（黄色）が打てる手を記録し…というように、各分岐がゲーム終了まで続きます。（コネクトフォーのゲーム木は非常に大きいため、図では一部のみを示しています。）

ゲームの終わり方をすべて把握できれば、最も勝ちやすい手を選ぶのに役立ちます。

### Heuristics(ヒューリスティック)(評価関数)

コネクトフォーの完全なゲーム木には、[4兆](https://oeis.org/A212693)以上の盤面が存在します！そのため実際には、エージェントはその一部しか扱いません。

この不完全な木でも有効にするために、Heuristics(評価関数)を使います。Heuristics(評価関数)は盤面にスコアを割り当て、高いスコアほど勝ちやすいと見なします。この設計はゲームに関する知識をもとに行います。

例えば、以下のようなHeuristics(評価関数)があります：

* エージェントが4つ並べた場合：1000000（1e6）点（勝利）
* エージェントが3つ並べて残り1マスが空：1点（次で勝てる）
* 相手が3つ並べて残り1マスが空：-100点（相手が次で勝つ）

これは下図でも示されています。
![](https://storage.googleapis.com/zenn-user-upload/60b0bea786e0-20260322.png)

では、エージェントはこれをどう使うのでしょうか？エージェントの手番で、図のような盤面を考えます。7通りの手（各列）があります。それぞれの手に対して次の盤面を作ります。
![](https://storage.googleapis.com/zenn-user-upload/529e070653b0-20260322.png)

その後、Heuristics(評価関数)で各盤面にスコアをつけます。これは[単語探し](https://en.wikipedia.org/wiki/Word_search)のように盤面を走査し、条件に一致するパターンを探します。見つかるたびにスコアを加減します。

例えば：

* 1つ目の盤面（列0に置く）はスコア2
  → 1点を加えるパターンが2つあるため
* 2つ目はスコア1
* 3つ目（列2）はスコア0
  → パターンが存在しないため

最もスコアが高い1つ目が選ばれます。これは実際にも最善手で、次の一手で勝利が確定しています。図で確認してみてください。

このHeuristics(評価関数)はこの例ではうまく機能しますが、他にも多くの方法があります。より良いHeuristics(評価関数)を設計できるかもしれません。

一般的には、Heuristics(評価関数)の設計方法（つまり、異なるゲーム状態をどのように採点するか、または異なる条件にどのスコアを割り当てるか）がわからない場合は、多くの場合、まず最初に推測を立てて、その後エージェントと対戦してみることです。これにより、エージェントが誤った行動をとった具体的なケースを特定でき、Heuristics(評価関数)を修正することで問題を解決できます。

### ソースコード(Code)
この「1手先読みエージェント」は：
* Heuristics(評価関数)で各手を評価し
* 最もスコアの高い手を選びます（同点ならランダム）

「1手先読み」とは、1手先しか見ないという意味です。

エージェントは下記の関数で実装されています。これらの関数は、エージェントを指定する際に使用することで、より理解しやすくなります。
```python
import random
import numpy as np

# Calculates score if agent drops piece in selected column
def score_move(grid, col, mark, config):
    next_grid = drop_piece(grid, col, mark, config)
    score = get_heuristic(next_grid, mark, config)
    return score

# Helper function for score_move: gets board at next step if agent drops piece in selected column
def drop_piece(grid, col, mark, config):
    next_grid = grid.copy()
    for row in range(config.rows-1, -1, -1):
        if next_grid[row][col] == 0:
            break
    next_grid[row][col] = mark
    return next_grid

# Helper function for score_move: calculates value of heuristic for grid
def get_heuristic(grid, mark, config):
    num_threes = count_windows(grid, 3, mark, config)
    num_fours = count_windows(grid, 4, mark, config)
    num_threes_opp = count_windows(grid, 3, mark%2+1, config)
    score = num_threes - 1e2*num_threes_opp + 1e6*num_fours
    return score

# Helper function for get_heuristic: checks if window satisfies heuristic conditions
def check_window(window, num_discs, piece, config):
    return (window.count(piece) == num_discs and window.count(0) == config.inarow-num_discs)
    
# Helper function for get_heuristic: counts number of windows satisfying specified heuristic conditions
def count_windows(grid, num_discs, piece, config):
    num_windows = 0
    # horizontal
    for row in range(config.rows):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[row, col:col+config.inarow])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    # vertical
    for row in range(config.rows-(config.inarow-1)):
        for col in range(config.columns):
            window = list(grid[row:row+config.inarow, col])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    # positive diagonal
    for row in range(config.rows-(config.inarow-1)):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[range(row, row+config.inarow), range(col, col+config.inarow)])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    # negative diagonal
    for row in range(config.inarow-1, config.rows):
        for col in range(config.columns-(config.inarow-1)):
            window = list(grid[range(row, row-config.inarow, -1), range(col, col+config.inarow)])
            if check_window(window, num_discs, piece, config):
                num_windows += 1
    return num_windows
```
一歩先読みエージェントは、次のコードセルで定義されます。
```python
# The agent is always implemented as a Python function that accepts two arguments: obs and config
def agent(obs, config):
    # Get list of valid moves
    valid_moves = [c for c in range(config.columns) if obs.board[c] == 0]
    # Convert the board to a 2D grid
    grid = np.asarray(obs.board).reshape(config.rows, config.columns)
    # Use the heuristic to assign a score to each possible board in the next turn
    scores = dict(zip(valid_moves, [score_move(grid, col, obs.mark, config) for col in valid_moves]))
    # Get a list of columns (moves) that maximize the heuristic
    max_cols = [key for key in scores.keys() if scores[key] == max(scores.values())]
    # Select at random from the maximizing columns
    return random.choice(max_cols)
```

エージェントのコードでは、まず有効な移動のリストを取得することから始めます。これは前回のチュートリアルで使用したコードと同じものです！

次に、盤面を2次元numpy配列に変換します（6行×7列）。

その後、`score_move()`で各手のスコアを計算します。内部では下記のヘルパー関数を使用します：
* `drop_piece()`：指定列に駒を置いた後の盤面を生成
* `get_heuristic()`：盤面のスコアを計算
* `count_windows()`：条件に一致する「4マスの並び」を数えます。ここで mark はエージェントのマークです。この関数は count_windows() 関数を使用しており、これはHeuristics(評価関数)の条件を満たす「ウィンドウ」（横・縦・斜めに連続する4マス）の数を数えます。具体的には、count_windows(grid, num_discs, piece, config) は、盤面（grid）の中で、指定されたプレイヤー（エージェントまたは相手）を表す piece のマークが num_discs 個含まれ、残りのマスが空であるようなウィンドウの数を返します。例えば、
    - num_discs=4 かつ piece=obs.mark とすると、エージェントが4つ連続で並べた回数を数えます。
    - num_discs=3 かつ piece=obs.mark%2+1 とすると、相手が3つ並べていて残り1マスが空いているウィンドウの数（＝相手が次の一手で勝てる状況）を数えます。

最後に、最大スコアの手をランダムに選びます。

**補足:** このコードは理解しやすさ重視で少し遅めです。`count_windows()`が何度もループしている点に注目すると高速化のヒントになります。

```python
def get_win_percentages(agent1, agent2, n_rounds=100):
    # Use default Connect Four setup
    config = {'rows': 6, 'columns': 7, 'inarow': 4}
    # Agent 1 goes first (roughly) half the time          
    outcomes = evaluate("connectx", [agent1, agent2], config, [], n_rounds//2)
    # Agent 2 goes first (roughly) half the time      
    outcomes += [[b,a] for [a,b] in evaluate("connectx", [agent2, agent1], config, [], n_rounds-n_rounds//2)]
    print("Agent 1 Win Percentage:", np.round(outcomes.count([1,-1])/len(outcomes), 2))
    print("Agent 2 Win Percentage:", np.round(outcomes.count([-1,1])/len(outcomes), 2))
    print("Number of Invalid Plays by Agent 1:", outcomes.count([None, 0]))
    print("Number of Invalid Plays by Agent 2:", outcomes.count([0, None]))
```

### 2. エージェントは勝つか？

次の盤面を考えてください。
![](https://storage.googleapis.com/zenn-user-upload/00d14a902a28-20260323.png)

エージェントが赤で、エージェントの手番だとします。

* チュートリアルのHeuristics(評価関数)を使った場合、このエージェントは勝ちますか？それとも負けますか？
* 今あなたが実装したHeuristics(評価関数)を使った場合はどうなりますか？

ヒント:
![](https://storage.googleapis.com/zenn-user-upload/8aa5d12778ad-20260323.png)
エージェントには2つの選択肢しかありません。ゲームに確実に勝つためには、エージェントはどちらの列を選択すべきでしょうか？

答え:
![](https://storage.googleapis.com/zenn-user-upload/fc89aaa59c32-20260323.png)
エージェントには2つの選択肢があります。0列目（一番左の列）か、6列目（一番右の列）のどちらかです。エージェントが0列目に駒を置いた場合、次の手で必ずゲームに勝利します。そして、もしそれが6列目にプレイされた場合、おそらくゲームに負けるでしょう（なぜなら、相手が同じ列にプレイすることで応戦した場合、相手がゲームに勝つからです）。

もし、エージェントがチュートリアルのHeuristics(評価関数)を使用する場合、両方の列は等しく評価されるため、エージェントはそれらから（均一に）ランダムに選択します。その場合、エージェントがゲームに勝つ確率は約50/50です。

あなたが実装したHeuristics(評価関数)については、その内容に依存します。そのため、ここではこちらが用意した解答用Heuristics(評価関数)に基づいた説明を行います。この場合、エージェントはほぼ確実に最後の列を選んでしまうため、おそらく負けます。

これは興味深い状況です。というのも、平均的には新しいHeuristics(評価関数)を使ったエージェントの方がチュートリアルのエージェントよりも良い性能を示しますが、この盤面に限っては確実に誤った判断をしてしまうからです。
