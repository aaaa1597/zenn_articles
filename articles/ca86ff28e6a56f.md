## Abstract

## カテゴリ変数
世の中には数値以外のデータがたくさんあります。それを機械学習に活用する方法をご紹介します。

## 理論編
このチュートリアルでは、カテゴリ変数とは何か、そしてこのタイプのデータを扱うための3つの手法について学びます。

### はじめに
カテゴリ変数は、限られた数の値のみを取る変数です。

例えば、朝食を食べる頻度を「全くない」「たまに」「大抵食べる」「毎日」の4択で尋ねるアンケートを考えてみましょう。この場合、回答は決められたセット（カテゴリ）のいずれかに当てはまるため、データはカテゴリ型となります。

所有している車のブランド（「ホンダ」「トヨタ」「フォード」など）を尋ねるアンケートも同様にカテゴリ型です。

Pythonのほとんどの機械学習モデルでは、これらの変数をそのまま投入するとエラーになります。事前に前処理が必要です。ここでは、カテゴリデータを準備するための3つのアプローチを比較します。

### 3つのアプローチ
#### 1. カテゴリ変数の削除 (Drop Categorical Variables)
最も簡単な方法は、データセットからカテゴリ変数の列を単に削除してしまうことです。この手法がうまくいくのは、その列に有用な情報が含まれていない場合に限られます。

#### 2. 順序エンコーディング (Ordinal Encoding)
各固有の値を異なる整数に割り当てる手法です。
![](https://storage.googleapis.com/kaggle-media/learn/images/tEogUAr.png)
この手法は、カテゴリに順序があることを前提としています。 
例：「全くない」(0) < 「たまに」(1) < 「大抵食べる」(2) < 「毎日」(3)

この例ではカテゴリ間に明確な順位があるため、この前提は理にかなっています。すべてのカテゴリ変数に明確な順序があるわけではありませんが、順序があるものを**順序変数（Ordinal Variables）** と呼びます。決定木やランダムフォレストなどのツリーベースのモデルでは、順序変数に対して順序エンコーディングがうまく機能することが期待できます。

#### 3. ワンホットエンコーディング (One-Hot Encoding)
元のデータに存在する各値に対して、その値の有無（1か0）を示す新しい列を作成します。
![](https://storage.googleapis.com/kaggle-media/learn/images/TW5m0aJ.png)

例えば「色」という列に「赤」「黄」「緑」の3つのカテゴリがある場合、対応するワンホットエンコーディングでは、それぞれの色に対して1つずつ列が作られます。元の値が「赤」であれば「赤」の列に1を立て、それ以外は0にします。

順序エンコーディングとは対照的に、ワンホットエンコーディングはカテゴリ間の順序を想定しません。したがって、データに明確な順序がない場合（例：「赤」は「黄」より大きいわけでも小さいわけでもない）に特に有効です。このように固有の順位を持たないカテゴリ変数を**名義変数（Nominal Variables）** と呼びます。 ※ただし、カテゴリの種類（値の数）が非常に多い場合（目安として15種類以上）、この手法はあまり適していません。

#### 実装例
前回のチュートリアル同様、[メルボルンの住宅価格データセット](https://www.kaggle.com/datasets/dansbecker/melbourne-housing-snapshot)を使用します。データの読み込み部分は省略し、すでに学習用と検証用のデータ（X_train, X_valid, y_train, y_valid）が準備できている状態からスタートします。

```python
import pandas as pd
from sklearn.model_selection import train_test_split

# Read the data
data = pd.read_csv('../input/melbourne-housing-snapshot/melb_data.csv')

# Separate target from predictors
y = data.Price
X = data.drop(['Price'], axis=1)

# Divide data into training and validation subsets
X_train_full, X_valid_full, y_train, y_valid = train_test_split(X, y, train_size=0.8, test_size=0.2,
                                                                random_state=0)

# Drop columns with missing values (simplest approach)
cols_with_missing = [col for col in X_train_full.columns if X_train_full[col].isnull().any()] 
X_train_full.drop(cols_with_missing, axis=1, inplace=True)
X_valid_full.drop(cols_with_missing, axis=1, inplace=True)

# "Cardinality" means the number of unique values in a column
# Select categorical columns with relatively low cardinality (convenient but arbitrary)
low_cardinality_cols = [cname for cname in X_train_full.columns if X_train_full[cname].nunique() < 10 and 
                        X_train_full[cname].dtype == "object"]

# Select numerical columns
numerical_cols = [cname for cname in X_train_full.columns if X_train_full[cname].dtype in ['int64', 'float64']]

# Keep selected columns only
my_cols = low_cardinality_cols + numerical_cols
X_train = X_train_full[my_cols].copy()
X_valid = X_valid_full[my_cols].copy()
```

各手法の精度を測定する関数の定義
3つの手法を比較するために score_dataset() という関数を定義します。この関数はランダムフォレストモデルを用いて、[平均絶対誤差 (MAE)](https://en.wikipedia.org/wiki/Mean_absolute_error) を算出します。MAEの値は低ければ低いほど優秀です。

##### アプローチ1（削除）のスコア
select_dtypes() を使って object 型の列を除外します。

MAE: 175,703.48

##### アプローチ2（順序エンコーディング）のスコア
Scikit-learnの OrdinalEncoder を使用します。

MAE: 165,936.40

※このコードでは各値をランダムに整数に割り当てていますが、順序変数に対してより適切なラベルを手動で割り当てれば、さらに精度が向上する可能性があります。

##### アプローチ3（ワンホットエンコーディング）のスコア
Scikit-learnの OneHotEncoder を使用します。

MAE: 166,089.48

どの手法がベストか？
今回のケースでは、カテゴリ列を削除した「アプローチ1」が最も高いMAEを記録し、最悪の結果となりました。残りの2つの手法については、MAEの差が非常に小さいため、どちらかが決定的に優れているとは言えません。

一般的には、ワンホットエンコーディング（手法3）が最も良いパフォーマンスを示すことが多く、削除（手法1）が最も悪くなる傾向にありますが、結果はケースバイケースです。

訓練データの列に対して順序エンコーダー（ordinal encoder）を学習（fit）させると、訓練データに現れる固有の値（ユニークな値）ごとに、対応する整数値のラベルが作成されます。

これは実世界のデータを扱う際に頻繁に遭遇する問題であり、解決策も数多く存在します。例えば、新しいカテゴリ（未知の値）に対処するためのカスタム・順序エンコーダーを自作することも可能です。しかし、最もシンプルな方法は、問題のあるカテゴリ列を削除してしまうことです。

さらにスコアを伸ばしたい場合は、画面右上の 「Edit」 ボタンを選択してください。コードを変更して、同じプロセスを繰り返すことができます。改善の余地はたくさんあります。取り組めば取り組むほど、リーダーボードの順位を駆け上がることができるはずです。
