## 傾向(Trend)
移動平均と時間ダミーを使用して長期的な変化をモデル化します。

## 理論編
### 傾向(Trend)とは?
時系列のトレンド成分とは、その系列の平均における持続的かつ長期的な変化を表します。トレンドは系列の中で最もゆっくりと変化する部分であり、最も大きな時間スケールを表す重要な要素です。商品売上の時系列では、年を追うごとに製品の認知度が高まり、市場が拡大していくことによって、増加トレンドが生じることがあります。
![](https://storage.googleapis.com/zenn-user-upload/07e50dfaaaa6-20260228.png)*4つの時系列に見られるトレンドパターン。*

この講座では、平均のトレンドに注目します。ただし一般には、系列における持続的でゆっくりとした変化であれば、それはトレンドとみなすことができます。例えば、時系列には分散（ばらつき）にトレンドが見られることもよくあります。

### 移動平均プロット
時系列にどのようなトレンドがあるかを確認するために、移動平均プロットを利用できます。移動平均を計算するには、ある幅を持つスライディングウィンドウ内の値の平均を求めます。グラフ上の各点は、その点を中心として左右のウィンドウ内に含まれるすべての値の平均を表します。目的は、短期的な変動をなめらかにし、長期的な変化だけを残すことです。
![](https://storage.googleapis.com/kaggle-media/learn/images/EZOXiPs.gif)*線形トレンドを示す移動平均プロット。曲線（青）の各点は、サイズ12のウィンドウ内に含まれる点（赤）の平均です。*

上のマウナロア系列では、毎年繰り返される上下の動き――つまり短期的な季節変動――が見られます。ある変化がトレンドの一部とみなされるには、それが季節変動よりも長い期間にわたって起こる必要があります。そのため、トレンドを可視化するには、系列中のどの季節周期よりも長い期間で平均を取ります。マウナロア系列では、各年の季節変動をならすために、サイズ12のウィンドウを選びました。

### トレンドの特徴量エンジニアリング
トレンドの形状を特定したら、タイムステップ特徴量を使ってそれをモデル化できます。時間ダミーそのものを使うことで、線形トレンドを表現できることはすでに見ました。
`target = a × time + b`

時間ダミーを変換することで、他のさまざまなトレンドも当てはめることができます。トレンドが二次関数（放物線）のように見える場合は、時間ダミーの2乗を特徴量に追加すればよく、次のようになります。
`target = a × time² + b × time + c`

線形回帰は係数 a、b、c を学習します。
以下の図にあるトレンド曲線は、これらの特徴量と scikit-learn の LinearRegression を使って当てはめたものです。
![](https://storage.googleapis.com/zenn-user-upload/5b56ad002b67-20260228.png)*上：線形トレンドを持つ系列。下：二次トレンドを持つ系列。*

この方法を初めて見る場合、線形回帰が直線以外の曲線も当てはめられることに気づかなかったかもしれません。考え方は、適切な形の曲線を特徴量として与えれば、線形回帰がそれらを組み合わせてターゲットに最もよく適合する形を学習できる、というものです。

### 例 ― トンネル交通量
この例では、Tunnel Traffic データセットのトレンドモデルを作成します。
```python
from pathlib import Path
from warnings import simplefilter

import matplotlib.pyplot as plt
import numpy as np
import pandas as pd

simplefilter("ignore")  # ignore warnings to clean up output cells

# Set Matplotlib defaults
plt.style.use("seaborn-whitegrid")
plt.rc("figure", autolayout=True, figsize=(11, 5))
plt.rc(
    "axes",
    labelweight="bold",
    labelsize="large",
    titleweight="bold",
    titlesize=14,
    titlepad=10,
)
plot_params = dict(
    color="0.75",
    style=".-",
    markeredgecolor="0.25",
    markerfacecolor="0.25",
    legend=False,
)
%config InlineBackend.figure_format = 'retina'


# Load Tunnel Traffic dataset
data_dir = Path("../input/ts-course-data")
tunnel = pd.read_csv(data_dir / "tunnel.csv", parse_dates=["Day"])
tunnel = tunnel.set_index("Day").to_period()
```
この系列にどのようなトレンドがあるかを見るために、移動平均プロットを作成しましょう。この系列は日次データなので、1年内の短期変動をならすために365日のウィンドウを選びます。

移動平均を作るには、まず rolling メソッドでウィンドウ計算を開始し、その後 mean メソッドでウィンドウ内の平均を計算します。確認すると、Tunnel Traffic のトレンドはおおよそ線形に見えます。
```python
moving_average = tunnel.rolling(
    window=365,       # 365-day window
    center=True,      # puts the average at the center of the window
    min_periods=183,  # choose about half the window size
).mean()              # compute the mean (could also do median, std, min, max, ...)

ax = tunnel.plot(style=".", color="0.5")
moving_average.plot(
    ax=ax, linewidth=3, title="Tunnel Traffic - 365-Day Moving Average", legend=False,
);
```
![](https://storage.googleapis.com/zenn-user-upload/b094569d5504-20260228.png)

レッスン1では、Pandas を使って時間ダミーを直接作成しました。しかしこれからは、statsmodels ライブラリの DeterministicProcess という関数を使います。この関数を使うことで、時系列と線形回帰で起こりがちな厄介な失敗ケースを避けることができます。order 引数は多項式の次数を表し、1は線形、2は二次、3は三次、という意味です。

```python
from statsmodels.tsa.deterministic import DeterministicProcess

dp = DeterministicProcess(
    index=tunnel.index,  # dates from the training data
    constant=True,       # dummy feature for the bias (y_intercept)
    order=1,             # the time dummy (trend)
    drop=True,           # drop terms if necessary to avoid collinearity
)
# `in_sample` creates features for the dates given in the `index` argument
X = dp.in_sample()

X.head()
```
| Day | const | trend |
| :--- | :--- | :--- |
| 2003-11-01 | 1.0 | 1.0 |
| 2003-11-02 | 1.0 | 2.0 |
| 2003-11-03 | 1.0 | 3.0 |
| 2003-11-04 | 1.0 | 4.0 |
| 2003-11-05 | 1.0 | 5.0 |

（ちなみに deterministic process とは、const や trend 系列のように、ランダムではなく完全に決定される時系列を指す専門用語です。時間インデックスから作られる特徴量は一般に決定論的になります。）

トレンドモデルは基本的にこれまでと同様に作成しますが、fit_intercept=False 引数を追加している点に注意してください。

```python
from sklearn.linear_model import LinearRegression

y = tunnel["NumVehicles"]  # the target

# The intercept is the same as the `const` feature from
# DeterministicProcess. LinearRegression behaves badly with duplicated
# features, so we need to be sure to exclude it here.
# 切片は DeterministicProcess の const 特徴量と同じです。LinearRegression は重複した
# 特徴量があるとうまく動作しないため、ここでは切片を除外する必要があります。
model = LinearRegression(fit_intercept=False)
model.fit(X, y)

y_pred = pd.Series(model.predict(X), index=X.index)
```

LinearRegression モデルによって見つかったトレンドは、移動平均プロットとほぼ一致しています。これは、この場合に線形トレンドを選んだのが適切だったことを示唆しています。

```python
ax = tunnel.plot(style=".", color="0.5", title="Tunnel Traffic - Linear Trend")
_ = y_pred.plot(ax=ax, linewidth=3, label="Trend")
```
![](https://storage.googleapis.com/zenn-user-upload/5912216317ae-20260228.png)

予測を行うには、モデルを「out of sample」の特徴量に適用します。「out of sample」とは、学習データの観測期間外の時点を指します。以下は30日先を予測する方法です。
```python
X = dp.out_of_sample(steps=30)

y_fore = pd.Series(model.predict(X), index=X.index)
```

より複雑な傾向を当てはめる方法の一つは、使用する多項式の次数を上げることです。店舗売上高のやや複雑な傾向をより正確に当てはめるには、11次の多項式を使ってみましょう。
```python
from statsmodels.tsa.deterministic import DeterministicProcess

dp = DeterministicProcess(index=y.index, order=11)
X = dp.in_sample()

model = LinearRegression()
model.fit(X, y)

y_pred = pd.Series(model.predict(X), index=X.index)

ax = y.plot(**plot_params, alpha=0.5, title="Average Sales", ylabel="items sold")
ax = y_pred.plot(ax=ax, linewidth=3, label="Trend", color='C0')
ax.legend();
```

### 4）高次多項式で予測するリスクを理解する
しかし、高次多項式は一般的に予測にはあまり適していません。
なぜだと思いますか？
![](https://storage.googleapis.com/zenn-user-upload/e5aabbfb1c5a-20260228.png)
