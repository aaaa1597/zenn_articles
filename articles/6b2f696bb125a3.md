## 理論編
### はじめに(Introduction)
このレッスンでは、ディープニューラルネットワークが得意とする**複雑な関係性**を学習できるネットワークを、どのように構築するかを見ていきます。

ここでの重要な考え方は **モジュール性（modularity）** です。
つまり、より単純な機能ユニットを組み合わせて、複雑なネットワークを作るという考え方です。

これまでに、線形ユニットが**線形関数**を計算することを学びました。
次は、それらの単一ユニットを組み合わせたり変形したりして、より複雑な関係をモデル化する方法を見ていきます。

---

## レイヤー（Layers）

ニューラルネットワークでは、通常ニューロンを**レイヤー（層）** にまとめます。

同じ入力を共有する線形ユニットをまとめたものを **Dense層（全結合層）** と呼びます。

![](https://storage.googleapis.com/zenn-user-upload/763a4004ae05-20260217.png)*2 つの入力とバイアスを受け取る 2 つの線形ユニットの全結合レイヤー。*

ニューラルネットワークの各レイヤーは、比較的単純な変換を行っていると考えることができます。
レイヤーを深く積み重ねることで、入力データをより複雑な形へと変換していきます。

よく訓練されたネットワークでは、各レイヤーが「解答に一歩ずつ近づくための変換」を担っています。

:::message
## さまざまな種類のレイヤー

Kerasにおける「レイヤー」は非常に広い概念です。
レイヤーとは本質的に、**任意のデータ変換**と考えることができます。

* 畳み込み層（[convolutional](https://www.tensorflow.org/api_docs/python/tf/keras/layers/Conv2D) layer）
* 再帰層（[recurrent](https://www.tensorflow.org/api_docs/python/tf/keras/layers/RNN) layer）

のような層は、ニューロンを用いてデータを変換しますが、主な違いは接続パターンにあります。

一方で、

* 特徴量エンジニアリングを行う層
* 単純な算術演算を行う層

なども存在します。
:::
レイヤーの世界は非常に広いので、ぜひいろいろ調べてみてください。

## 活性化関数（Activation Function）

ここで重要なポイントがあります。

**Dense層を2つ重ねただけでは、1つのDense層と本質的に変わりません。**

Dense層だけでは、モデルは線や平面の世界から抜け出すことができません。
つまり、**線形な関係しか学習できない**のです。

そこで必要になるのが **非線形性（nonlinearity）**、つまり**活性化関数**です。
![](https://storage.googleapis.com/zenn-user-upload/cb710cd93fcc-20260217.png)*活性化関数がなければ、ニューラルネットワークは線形関係しか学習できません。
曲線を近似するには、活性化関数が必要です。*

---

### 活性化関数とは？

活性化関数とは、**各レイヤーの出力（活性値）に対して適用する関数** のことです。最も一般的なのは **ReLU関数（Rectifier function）** です：
![](https://storage.googleapis.com/zenn-user-upload/74261fca87c4-20260217.png)

これは、
[max(0, x)]
* ( x > 0 ) のとき ( y = x )
* ( x < 0 ) のとき ( y = 0 )

となる関数で、折れ曲がった「ヒンジ（_/）」のような形をしています。

負の部分をゼロに「整流（rectify）」しているため、この名前がついています。
![](https://storage.googleapis.com/zenn-user-upload/693811ca7b84-20260217.png)*修正された線形ユニット。*

### ReLU（Rectified Linear Unit）

線形ユニットにReLUを適用すると、**ReLUユニット**になります。
出力は次のようになります：
[max(0, w * x + b)]

そのため、ReLU関数はしばしば「ReLU活性化関数」とも呼ばれます。

ReLUを使うことで、単なる直線ではなく、**折れ曲がった形の表現**が可能になります。

### Dense層を積み重ねる
非線形性が加わったことで、レイヤーを重ねる意味が生まれます。
![](https://storage.googleapis.com/zenn-user-upload/f2d62ea2499b-20260217.png)*高密度のレイヤーを積み重ねることで、「完全に接続された」ネットワークが形成されます。*

入力層 → 隠れ層 → 出力層
という構造を作ることで、複雑なデータ変換が可能になります。

出力層より前の層は、**隠れ層（hidden layers）** と呼ばれます。
これは、私たちがそれらの出力を直接見ることがないからです。

### 出力層について

最後の層が**線形ユニット（活性化なし）** であることに注目してください。

このネットワークは、任意の数値を予測しようとする**回帰問題（regression）** に適しています。
任意の数値を予測する場合には、出力に活性化関数をつけないのが一般的です。

一方で、分類問題などでは出力層に活性化関数が必要になります。

## Sequentialモデルの構築

これまで使ってきた `Sequential` モデルは、レイヤーを**順番に**接続します。

* 最初のレイヤーが入力を受け取り
* 最後のレイヤーが出力を生成します

例：

```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    # hidden ReLU layers
    layers.Dense(units=4, activation='relu', input_shape=[2]),
    layers.Dense(units=3, activation='relu'),
    # linear output layer
    layers.Dense(units=1),
])
```

ポイント：

* レイヤーは **リスト `[layer, layer, ...]`** として渡す
* 活性化関数は `activation` 引数で指定する
