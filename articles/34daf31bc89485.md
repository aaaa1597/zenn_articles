### ジオコーディング
ジオコーディングとは、地名や住所を地図上の位置に変換するプロセスです。たとえば、Google マップ、Bing マップ、Baidu マップなどのランドマークの説明に基づいて地理的な場所を検索したことがあるなら、ジオコーダーを使用したことになります。
![](https://storage.googleapis.com/kaggle-media/learn/images/1IrgZQq.png)

#### すべてのジオコーディングには geopy を使用します。

```python
from geopy.geocoders import Nominatim
```
上のコードセルにおいて、Nominatim は位置情報を生成するために使用されるジオコーディング・ソフトウェアを指します。

まず、ジオコーダーをインスタンス化（初期化）することから始めます。その後は、場所の名称や住所を Python の文字列として渡すだけです。（この例では、ギザの大ピラミッドとして知られる "Pyramid of Khufu" を指定しています。）

#### ジオコーディングが成功すると、2つの重要な属性を持つ geopy.location.Location オブジェクトが返されます。

- "point" 属性: 緯度（latitude）と経度（longitude）の情報を含みます。
- "address" 属性: 完全な住所（フルアドレス）を含みます。

```python
geolocator = Nominatim(user_agent="kaggle_learn")
location = geolocator.geocode("Pyramid of Khufu")

print(location.point)
print(location.address)
```
**出力**
29 58m 44.976s N, 31 8m 3.17625s E
هرم خوفو, شارع ابو الهول السياحي, نزلة البطران, الجيزة, 12125, مصر

"point" 属性の値は geopy.point.Point オブジェクトであり、そこから latitude 属性で緯度を、longitude 属性で経度をそれぞれ取得できます。

```python
point = location.point
print("Latitude:", point.latitude)
print("Longitude:", point.longitude)
```
**出力**
Latitude: 29.97916
Longitude: 31.134215625236113

この場合、**ラムダ関数（lambda function）** を使用して、DataFrame のすべての行にジオコーダーを適用できます。（ジオコーディングが失敗した場合に備えて、try/except 文を使用しています。）

```python
def my_geocoder(row):
    try:
        point = geolocator.geocode(row).point
        return pd.Series({'Latitude': point.latitude, 'Longitude': point.longitude})
    except:
        return None

universities[['Latitude', 'Longitude']] = universities.apply(lambda x: my_geocoder(x['Name']), axis=1)

print("{}% of addresses were geocoded!".format(
    (1 - sum(np.isnan(universities["Latitude"])) / len(universities)) * 100))

# Drop universities that were not successfully geocoded
universities = universities.loc[~np.isnan(universities["Latitude"])]
universities = gpd.GeoDataFrame(
    universities, geometry=gpd.points_from_xy(universities.Longitude, universities.Latitude))
universities.crs = {'init': 'epsg:4326'}
universities.head()
```

### テーブル結合(Table Joins)
ここからはトピックを変えて、異なるソースからのデータを組み合わせる方法について考えていきましょう。

- 属性結合(Attribute join)
- 空間結合(Spatial join)

#### 属性結合(Attribute join)
共通のインデックスを持つ複数の DataFrame を結合するために pd.DataFrame.join() を使う方法はすでにご存知かと思います。このように、単にインデックスの値を一致させてデータを結合する方法を **属性結合(Attribute join)** と呼びます。

GeoDataFrame で属性結合を行う場合は、gpd.GeoDataFrame.merge() を使用するのが最適です。具体例として、ヨーロッパの全国家の境界線データを含む GeoDataFrame europe_boundaries を使用します。

```python
world = gpd.read_file(gpd.datasets.get_path('naturalearth_lowres'))

europe_stats = europe[["name", "pop_est", "gdp_md_est"]]
europe_boundaries = europe[["name", "geometry"]]
```

#### 空間結合(Spatial join)
もう一つの結合タイプは **空間結合(Spatial join)** です。空間結合では、「geometry」列にあるオブジェクト同士の空間的な位置関係に基づいて GeoDataFrame を組み合わせます。

例えば、先ほどジオコーディングした大学の住所データ universities があります。これに空間結合を用いることで、各大学をその所在国に一致させることができます。これには gpd.sjoin() を使用します。

```python
# Use spatial join to match universities to countries in Europe
european_universities = gpd.sjoin(universities, europe)

# Investigate the result
print("We located {} universities.".format(len(universities)))
print("Only {} of the universities were located in Europe (in {} different countries).".format(
    len(european_universities), len(european_universities.name.unique())))

european_universities.head()
```
We located 91 universities.
Only 87 of the universities were located in Europe (in 14 different countries).

**出力**
||Name|Latitude|Longitude|geometry|index_right|name|pop_est|gdp_md_est|
|---|---|---|---|---|---|---|---|---|
|0|University of Oxford|51.758879|-1.259603|POINT (-1.25960 51.75888)|28|United Kingdom|64769452|2788000.0|
|1|University of Cambridge|52.200623|0.110474|POINT (0.11047 52.20062)|28|United Kingdom|64769452|2788000.0|
|2|Imperial College London|51.498959|-0.175641|POINT (-0.17564 51.49896)|28|United Kingdom|64769452|2788000.0|
|4|UCL|51.521785|-0.135151|POINT (-0.13515 51.52179)|28|United Kingdom|64769452|2788000.0|
|5|London School of Economics and Political Science|51.514211|-0.116808|POINT (-0.11681 51.51421)|28|United Kingdom|64769452|2788000.0|

上記の空間結合では、両方の GeoDataFrame の 「geometry」列を確認します。もし universities の Point（点） オブジェクトが europe の Polygon（多角形/境界線） オブジェクトと重なっている（交差している）場合、それらの行が結合され、european_universities の1つの行として追加されます。

一致する大学がない国（または一致する国がない大学）は、結果から除外されます。

gpd.sjoin() メソッドは、how や op 引数を通じて、さまざまな結合タイプにカスタマイズ可能です。例えば、SQL の左結合（または右結合）と同じことをしたい場合は、how='left'（または how='right'）と設定します。このコースでは詳細には触れませんが、[公式ドキュメント](https://geopandas.org/en/stable/docs/reference/api/geopandas.sjoin.html)でさらに詳しく学ぶことができます。
