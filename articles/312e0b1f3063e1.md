## Abstract
Kaggle「高度なSQLの[ネストされたデータと繰り返し](https://www.kaggle.com/code/alexisbcook/nested-and-repeated-data)」の翻訳と実行方法の解説

## ネストされたデータと繰り返し
BigQuery で複雑なデータ型をクエリする方法を学びます。

## 理論編
### はじめに(Introduction)
これまでに、整数や浮動小数点数などの数値型、文字列型、そして DATETIME 型など、さまざまなデータ型を扱ってきました。
このチュートリアルでは、**ネストされたデータ（nested data）** と **繰り返しデータ（repeated data）** をどのようにクエリするかを学びます。これらは、BigQuery データセットで見られる中でも最も複雑なデータ型です。

### Nested data（ネストされたデータ）

ペットとそのおもちゃに関する情報を含む架空のデータセットを考えてみましょう。
この情報は、2つのテーブル（`pets` テーブルと `toys` テーブル）に分けて整理することができます。`toys` テーブルには「Pet_ID」列を持たせ、それぞれのおもちゃを対応するペットと紐づけることができます。

BigQuery では、これとは別の方法として、すべての情報を1つのテーブルにまとめることも可能です。以下の `pets_and_toys` テーブルのような形です。

![](https://storage.googleapis.com/zenn-user-upload/83fe13460b61-20260215.png)

この場合、`toys` テーブルの情報はすべて1つの列（`Toy` 列）にまとめられています。
`pets_and_toys` テーブルの `Toy` 列は **ネストされた列（nested column）** と呼ばれ、`Name` と `Type` フィールドがその中にネストされています。

ネストされた列の型は **STRUCT（または RECORD）型** です。これは、テーブルのスキーマにも反映されます。

> テーブルの構造は **スキーマ（schema）** と呼ばれます。スキーマの読み方を復習したい場合は、[Intro to SQL マイクロコース](https://zenn.dev/rg687076/articles/d7cf749c7ba539)の該当レッスンを参照してください。

![](https://storage.googleapis.com/zenn-user-upload/df6a9edc7424-20260215.png)

ネストされたデータを含む列をクエリする場合は、その列の文脈の中で各フィールドを指定します。

* `Toy.Name` は「Toy」列の中の「Name」フィールド
* `Toy.Type` は「Toy」列の中の「Type」フィールド

![](https://storage.googleapis.com/zenn-user-upload/5bad31233dc4-20260215.png)

それ以外のクエリの書き方は通常と変わりません。

### 繰り返しデータ(Repeated data)

より現実的なケースとして、1匹のペットが複数のおもちゃを持っている場合を考えましょう。この場合、1つのテーブルにまとめるためには、別のデータ型を利用する必要があります。

![](https://storage.googleapis.com/zenn-user-upload/231af866c3a2-20260215.png)

「Toys」列は **繰り返しデータ（repeated data）** を含むと言います。
これは、1行に対して複数の値を持つことができるからです。

スキーマを見ると、「Toys」列の mode が **`'REPEATED'`** になっています。

![](https://storage.googleapis.com/zenn-user-upload/5efbdad9084c-20260215.png)

繰り返しフィールドの各エントリは **ARRAY（配列）** です。
ARRAY は、同じデータ型を持つ値を順序付きで（0個以上）格納したものです。
例えば、Moon という犬の「Toys」列の値が

```
[Frisbee, Bone, Rope]
```

であれば、これは3つの値を持つ ARRAY です。

繰り返しデータをクエリする場合は、対象の列を **UNNEST() 関数** の中に入れます。

![](https://storage.googleapis.com/zenn-user-upload/9f6fa02b36d5-20260215.png)

これにより、繰り返しデータがフラット化（展開）され、各要素が1行ずつ表示されるようになります。

![](https://storage.googleapis.com/zenn-user-upload/0eb0da013188-20260215.png)

### Nested と Repeated の両方を持つデータ

では、ペットが複数のおもちゃを持ち、さらに各おもちゃについて「名前」と「種類」の両方を保持したい場合はどうでしょうか？

![](https://storage.googleapis.com/zenn-user-upload/d44d48b29dac-20260215.png)

この場合、「Toys」列を **ネストかつ繰り返し（nested and repeated）** にします。

`more_pets_and_toys` テーブルでは：

* `Name` と `Type` は「Toys」STRUCT の中のフィールド
* そして「Toys」列自体が REPEATED
* `Toys.Name` と `Toys.Type` の各エントリは ARRAY

![](https://storage.googleapis.com/zenn-user-upload/0f7f68bb0de5-20260215.png)

### サンプルクエリ

![](https://storage.googleapis.com/zenn-user-upload/d713806a63e1-20260215.png)

「Toys」列は繰り返しなので、**`UNNEST()`** で展開します。また、展開後の列に `t` というエイリアスを付けているため、

* `t.Name`
* `t.Type`

のように参照できます。

### hits 列（ネストかつ繰り返しデータ）
ここでは、ネストされかつ繰り返しのあるデータの例として、「ヒット」列を取り上げます。

`hits` 列は、
* STRUCT 型（ネスト）
* かつ REPEATED（繰り返し）

です。

さらに：
* `hitNumber`
* `page`
* `type`

が `hits` の中にネストされており、

* `pagePath` は `page` の中にネストされています。

次の構文を使用してこれらのフィールドをクエリできます。

### 最も人気のあるランディングページを調べるクエリ

```python
# Query to determine most popular landing point on the website
query = """
        SELECT hits.page.pagePath as path,
            COUNT(hits.page.pagePath) as counts
        FROM `bigquery-public-data.google_analytics_sample.ga_sessions_20170801`, 
            UNNEST(hits) as hits
        WHERE hits.type="PAGE" and hits.hitNumber=1
        GROUP BY path
        ORDER BY counts DESC
        """

# Run the query, and return a pandas DataFrame
result = client.query(query).result().to_dataframe()
result.head()
```

|| path | counts |
| :--- | :--- | :--- |
| 0 | /home | 1257 |

| 2 | /google+redesign/apparel/mens/mens+t+shirts | 117 |
| 3 | /signin.html | 78 |
| 4 | /basket.html | 35 |

このクエリでは：

* `UNNEST(hits)` によって `hits` を展開
* `hits.type="PAGE"` かつ `hits.hitNumber=1` で最初のページビューを抽出

結果から、最も多くのユーザーが `/home` ページからサイトに訪れていることがわかります。

ここまでが、ネストされたデータと繰り返しデータの基本です。
