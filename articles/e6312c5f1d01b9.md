## 理論編
### K平均法によるクラスタリング
クラスター ラベルを使用して複雑な空間関係を解明します。

### はじめに

このレッスンと次のレッスンでは、「教師なし学習アルゴリズム（unsupervised learning algorithms）」と呼ばれる手法を使います。教師なしアルゴリズムは目的変数（target）を使いません。その代わりに、データのある性質を学習し、特徴量の構造を何らかの形で表現することを目的とします。
予測のための特徴量エンジニアリングという文脈では、教師なしアルゴリズムは「特徴量の発見（feature discovery）」の手法だと考えることができます。

**クラスタリング**とは、データ点同士の類似度に基づいて、それらをグループに割り当てることを意味します。いわば「類は友を呼ぶ」という考え方です。

特徴量エンジニアリングとしてクラスタリングを使う場合、たとえば市場セグメントを表す顧客グループや、似た気象パターンを共有する地理的エリアを発見しようとすることができます。クラスタラベルを特徴量として追加すると、空間的な関係や近接性といった複雑な関係を、機械学習モデルがうまく整理できるようになります。

### 特徴量としてのクラスタラベル

クラスタリングを単一の実数値特徴量に適用すると、従来の「ビニング」や「[離散化（discretization）](https://scikit-learn.org/stable/auto_examples/preprocessing/plot_discretization_classification.html)」と似た働きをします。
複数の特徴量に対して行う場合は、「多次元ビニング」（ベクトル量子化とも呼ばれます）に相当します。

![](https://storage.googleapis.com/zenn-user-upload/756974e7c7b5-20260210.png)
**左:** 単一特徴量に対するクラスタリング **右:** 2つの特徴量にまたがるクラスタリング

データフレームにクラスタラベルという特徴量を追加すると、次のようになります。

| Longitude | Latitude | Cluster |
| --------- | -------- | ------- |
| -93.619   | 42.054   | 3       |
| -93.619   | 42.053   | 3       |
| -93.638   | 42.060   | 1       |
| -93.602   | 41.988   | 0       |

ここで重要なのは、この `Cluster` 特徴量はカテゴリ変数だという点です。ここでは、一般的なクラスタリングアルゴリズムが出力するように、整数の並び（ラベルエンコーディング）として示されていますが、モデルによってはワンホットエンコーディングの方が適切な場合もあります。

クラスタラベルを追加する動機は、複数の特徴量にまたがる複雑な関係を、クラスタによってより単純なかたまりに分割できることにあります。モデルは、複雑な全体を一度に学習する代わりに、単純になった部分を一つずつ学習すればよくなります。これはいわば「分割統治（divide and conquer）」の戦略です。

![](https://storage.googleapis.com/zenn-user-upload/82878b397230-20260210.png)*`YearBuilt` 特徴量をクラスタリングすると、この線形モデルは `SalePrice` との関係をよりうまく学習できるようになります。*

図は、クラスタリングが単純な線形モデルをどのように改善するかを示しています。`YearBuilt` と `SalePrice` の関係は曲線的で、この種のモデルには複雑すぎるため、アンダーフィットしてしまいます。しかし、小さな区間（クラスタ）に分けると、その関係はほぼ線形になり、モデルが容易に学習できるようになります。

### k-means クラスタリング

クラスタリングアルゴリズムには非常に多くの種類があります。それらの違いは主に、「類似性」や「近さ」をどう測るか、そしてどのような特徴量に対応できるかです。
ここで使うアルゴリズムである **k-means** は直感的で、特徴量エンジニアリングの文脈でも使いやすい手法です。ただし、用途によっては別のアルゴリズムの方が適している場合もあります。

**k-means クラスタリング**は、通常の直線距離（つまりユークリッド距離）を使って類似性を測ります。このアルゴリズムは、特徴量空間の中に「重心（centroid）」と呼ばれる点をいくつか配置することでクラスタを作ります。
データセット内の各点は、最も近い重心を持つクラスタに割り当てられます。「k」は作成する重心（＝クラスタ）の数を表しており、これはユーザーが指定します。

各重心が同心円状に点を取り込んでいく様子を想像するとよいでしょう。複数の重心から広がる円が重なり合うと、その境界線ができます。この結果として得られるのが「**ボロノイ分割（Voronoi tessellation）**」です。
この分割は、新しいデータがどのクラスタに割り当てられるかを示しており、本質的には k-means が学習した内容そのものです。

上に示した [Ames](https://www.kaggle.com/c/house-prices-advanced-regression-techniques/data) データセットのクラスタリングは k-means によるものです。次の図は、分割領域と重心を表示した同じ結果です。

![](https://storage.googleapis.com/zenn-user-upload/7734e4ac6d3a-20260210.png)*k-means クラスタリングは、特徴量空間をボロノイ分割します。*

### k-means の学習過程

ここでは、k-means アルゴリズムがどのようにクラスタを学習し、それが特徴量エンジニアリングにとって何を意味するのかを確認します。scikit-learn の実装における、次の3つのパラメータに注目します。

* `n_clusters`
* `max_iter`
* `n_init`

アルゴリズムは、次のようなシンプルな2段階の処理を繰り返します。

1. 各データ点を、最も近い重心のクラスタに割り当てる
2. 各重心を、そのクラスタ内の点との距離が最小になる位置へ移動させる

重心が動かなくなるか、最大反復回数（`max_iter`）に達するまで、この2ステップを繰り返します。

初期の重心位置がランダムに決まるため、良くないクラスタリングに収束してしまうこともよくあります。そのため、アルゴリズムはこれを複数回（`n_init` 回）繰り返し、各点と重心との総距離が最も小さい結果、つまり最適なクラスタリングを返します。

下のアニメーションは、このアルゴリズムの動作を示しています。初期重心への依存性や、収束まで反復することの重要性が分かります。

![](https://storage.googleapis.com/kaggle-media/learn/images/tBkCqXJ.gif)*NYC の Airbnb 物件に対する k-means クラスタリング*

クラスタ数が多い場合には `max_iter` を増やす必要があったり、データが複雑な場合には `n_init` を増やした方がよいこともあります。ただし、通常は自分で調整する必要があるのは `n_clusters`（つまり k）だけです。
どの分割が最適かは、使うモデルや予測したい対象によって異なるため、他のハイパーパラメータと同様に、交差検証などでチューニングするのがよいでしょう。

### 例：California Housing

[California Housing](https://www.kaggle.com/datasets/camnugent/california-housing-prices) データセットでは、空間的特徴量である `Latitude`（緯度）と `Longitude`（経度）が、k-means クラスタリングの自然な候補になります。この例では、これらに `MedInc`（中央値の所得）を加えて、カリフォルニア各地の経済セグメントを作成します。

```python
import matplotlib.pyplot as plt
import pandas as pd
import seaborn as sns
from sklearn.cluster import KMeans

plt.style.use("seaborn-whitegrid")
plt.rc("figure", autolayout=True)
plt.rc(
    "axes",
    labelweight="bold",
    labelsize="large",
    titleweight="bold",
    titlesize=14,
    titlepad=10,
)

df = pd.read_csv("../input/fe-course-data/housing.csv")
X = df.loc[:, ["MedInc", "Latitude", "Longitude"]]
X.head()
```

|   | MedInc | Latitude | Longitude |
| - | ------ | -------- | --------- |
| 0 | 8.3252 | 37.88    | -122.23   |
| 1 | 8.3014 | 37.86    | -122.22   |
| 2 | 7.2574 | 37.85    | -122.24   |
| 3 | 5.6431 | 37.85    | -122.25   |
| 4 | 3.8462 | 37.85    | -122.25   |

k-means クラスタリングはスケールに敏感なので、極端な値を持つデータではリスケーリングや正規化を行うのがよい場合があります。今回の特徴量はすでにおおよそ同じスケールにあるため、そのまま使います。

```python
# Create cluster feature
kmeans = KMeans(n_clusters=6)
X["Cluster"] = kmeans.fit_predict(X)
X["Cluster"] = X["Cluster"].astype("category")

X.head()
```

結果は次のようになります。

| - | ------ | -------- | --------- | ------- |
| 0 | 8.3252 | 37.88    | -122.23   | 5       |

| 2 | 7.2574 | 37.85    | -122.24   | 5       |
| 3 | 5.6431 | 37.85    | -122.25   | 5       |
| 4 | 3.8462 | 37.85    | -122.25   | 2       |

次に、このクラスタリングがどれだけ有効かを確認するために、いくつかのプロットを見てみます。まず、クラスタの地理的分布を示す散布図です。沿岸部の高所得地域が、別のセグメントとして分かれているように見えます。

```
sns.relplot(

```

k-means アルゴリズムは スケール（尺度）に敏感 です。つまり、特徴量を どのように、あるいは本当にスケーリングすべきか を慎重に考える必要があります。選択次第で、まったく異なる結果が得られる可能性があるからです。経験則として、**すでに直接比較できる特徴量**（たとえば、異なる時点でのテスト結果など）の場合は、スケーリングしない方がよいでしょう。一方で、**同じ尺度で比較できない特徴量**（たとえば、身長と体重のようなもの）は、通常スケーリングすることで恩恵を受けます。ただし、どちらが正しいかがはっきりしない場合もあります。そのようなときは、**値が大きい特徴量ほど重く扱われる** という点を意識しつつ、常識的に判断するようにしましょう。

**正解：**
