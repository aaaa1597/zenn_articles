## 理論編
### はじめに(Introduction)
レッスン2と3では、予測を単純な回帰問題として扱い、すべての特徴量を単一の入力である時間インデックスから作成しました。そのため、必要なトレンド特徴量や季節性特徴量を生成するだけで、将来のどの時点についても簡単に予測を作ることができました。

しかし、レッスン4でラグ特徴量を追加したことで、問題の性質が変わりました。ラグ特徴量は、予測を行う時点で、そのラグされた目的変数の値が既知であることを前提とします。lag1の特徴量は時系列を1ステップ前方にずらすため、1ステップ先までは予測できますが、2ステップ先までは予測できません。

レッスン4では、予測したい期間まで常にラグを生成できると仮定していました（つまり、すべての予測は1ステップ先だけを対象にしていました）。しかし実務の予測では通常それ以上が求められるため、このレッスンではさまざまな状況に対応した予測方法を学びます。

### 予測タスクの定義
予測モデルを設計する前に、次の2つを明確にする必要があります。
- 予測を行う時点で利用可能な情報は何か（特徴量）、
- そして、どの期間の予測値が必要か（目的変数）です。

**forecast origin**とは、予測を行う時点のことです。実務的には、予測対象期間に対して利用可能な学習データの最後の時点を forecast origin と考えることができます。オリジン以前のすべてのデータを特徴量作成に利用できます。

**forecast horizon**とは、予測を行う対象期間のことです。予測はしばしば horizon のステップ数で表現され、「1ステップ予測」や「5ステップ予測」と呼ばれます。forecast horizonは目的変数を定義します。
![](https://storage.googleapis.com/zenn-user-upload/47bd3e5ebb22-20260303.png)*4つのラグ特徴量を用いた、リードタイム2ステップ・horizon 3ステップの予測の例です。この図は、トレーニングデータの1行分、つまり1つの予測のためのデータを表しています。*

オリジンから horizon までの間の時間を、予測のリードタイム（またはレイテンシ）と呼びます。リードタイムは、オリジンから horizon までのステップ数で表され、「1ステップ先予測」や「3ステップ先予測」と呼ばれます。実務では、データ取得や処理の遅延により、オリジンから数ステップ先から予測を開始しなければならない場合もあります。

### 予測のためのデータ準備
機械学習アルゴリズムで時系列を予測するには、その時系列をアルゴリズムで扱えるデータフレーム形式に変換する必要があります。（もちろん、トレンドや季節性のような決定論的特徴量のみを使う場合は別です。）

この処理の前半は、レッスン4でラグから特徴量セットを作成したときに見ました。後半は目的変数の準備です。その方法は予測タスクによって異なります。

データフレームの各行は1つの予測に対応します。その行の時間インデックスは horizon の最初の時点を表し、 horizon 全体の値を同じ行に並べます。
マルチステップ予測では、各ステップごとに1つずつ、複数の出力をモデルに求めることになります。
```python
import numpy as np
import pandas as pd

N = 20
ts = pd.Series(
    np.arange(N),
    index=pd.period_range(start='2010', freq='A', periods=N, name='Year'),
    dtype=pd.Int8Dtype,
)

# Lag features
X = pd.DataFrame({
    'y_lag_2': ts.shift(2),
    'y_lag_3': ts.shift(3),
    'y_lag_4': ts.shift(4),
    'y_lag_5': ts.shift(5),
    'y_lag_6': ts.shift(6),    
})

# Multistep targets
y = pd.DataFrame({
    'y_step_3': ts.shift(-2),
    'y_step_2': ts.shift(-1),
    'y_step_1': ts,
})

data = pd.concat({'Targets': y, 'Features': X}, axis=1)

data.head(10).style.set_properties(['Targets'], **{'background-color': 'LavenderBlush'}) \
                   .set_properties(['Features'], **{'background-color': 'Lavender'})
```
- y_step_3  ～ y_step_1は目的変数
- y_lag_2 ～ y_lag_6はラグ変数

### マルチステップ予測の戦略
マルチステップ予測で必要な複数のターゲットを生成するための戦略はいくつかあります。ここでは、長所と短所を持つ代表的な4つの戦略を紹介します。

#### マルチアウトプットモデル
もともと複数出力を生成できるモデルを使用します。線形回帰やニューラルネットワークは複数出力に対応できます。この戦略はシンプルで効率的ですが、すべてのアルゴリズムで使えるわけではありません。たとえば XGBoost では直接は対応できません。
![](https://storage.googleapis.com/zenn-user-upload/253d46018ce9-20260303.png)

#### ダイレクト戦略

horizon の各ステップごとに別々のモデルを学習します。1ステップ先用、2ステップ先用、といった具合です。1ステップ先予測と2ステップ先予測は異なる問題なので、ステップごとに別モデルを使うことが有効な場合があります。欠点は、多数のモデルを学習させるため計算コストが高くなることです。
![](https://storage.googleapis.com/zenn-user-upload/bb1cef7ffdad-20260303.png)

#### リカーシブ戦略
1ステップ予測モデルを1つだけ学習し、その予測値を次のステップのラグ特徴量として更新しながら使います。リカーシブ法では、モデルの1ステップ予測値を同じモデルに入力し直し、次の予測ステップのラグ特徴量として利用します。1つのモデルだけで済みますが、誤差がステップごとに伝播するため、長い horizon では予測精度が低下する可能性があります。
![](https://storage.googleapis.com/zenn-user-upload/f1724f10234c-20260303.png)

#### DirRec戦略
ダイレクト法とリカーシブ法を組み合わせた方法で、各ステップごとにモデルを学習しつつ、前ステップの予測値を新たなラグ特徴量として利用します。ステップが進むごとに、各モデルは追加のラグ入力を受け取ります。各モデルは常に最新のラグ特徴量を持つため、DirRecはダイレクト法よりも系列依存を捉えやすいですが、リカーシブ法と同様に誤差伝播の影響を受ける可能性があります。
![](https://storage.googleapis.com/zenn-user-upload/80da476bd2f0-20260303.png)

### 例 ― Flu Trends(インフルエンザの傾向)

この例では、レッスン4のFlu Trendsデータに対してマルチアウトプット戦略とダイレクト戦略を適用し、今回は学習期間を超えた複数週先の実際の予測を行います。

予測タスクは、リードタイム1週間、horizon 8週間と定義します。つまり、翌週から始まる8週間分のインフルエンザ患者数を予測します。

非表示セルでは、例のセットアップと補助関数 plot_multistep の定義を行っています。
```python
from pathlib import Path
from warnings import simplefilter

import matplotlib.pyplot as plt
import numpy as np
import pandas as pd
import seaborn as sns
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_squared_error
from sklearn.model_selection import train_test_split
from xgboost import XGBRegressor

simplefilter("ignore")

# Set Matplotlib defaults
plt.style.use("seaborn-whitegrid")
plt.rc("figure", autolayout=True, figsize=(11, 4))
plt.rc(
    "axes",
    labelweight="bold",
    labelsize="large",
    titleweight="bold",
    titlesize=16,
    titlepad=10,
)
plot_params = dict(
    color="0.75",
    style=".-",
    markeredgecolor="0.25",
    markerfacecolor="0.25",
)
%config InlineBackend.figure_format = 'retina'


def plot_multistep(y, every=1, ax=None, palette_kwargs=None):
    palette_kwargs_ = dict(palette='husl', n_colors=16, desat=None)
    if palette_kwargs is not None:
        palette_kwargs_.update(palette_kwargs)
    palette = sns.color_palette(**palette_kwargs_)
    if ax is None:
        fig, ax = plt.subplots()
    ax.set_prop_cycle(plt.cycler('color', palette))
    for date, preds in y[::every].iterrows():
        preds.index = pd.period_range(start=date, periods=len(preds))
        preds.plot(ax=ax)
    return ax


data_dir = Path("../input/ts-course-data")
flu_trends = pd.read_csv(data_dir / "flu-trends.csv")
flu_trends.set_index(
    pd.PeriodIndex(flu_trends.Week, freq="W"),
    inplace=True,
)
flu_trends.drop("Week", axis=1, inplace=True)
```

まず、目的系列（週ごとのインフルエンザ外来受診数）をマルチステップ予測用に準備します。これができれば、学習と予測はとても簡単です。
```python
def make_lags(ts, lags, lead_time=1):
    return pd.concat(
        {
            f'y_lag_{i}': ts.shift(i)
            for i in range(lead_time, lags + lead_time)
        },
        axis=1)


# Four weeks of lag features
y = flu_trends.FluVisits.copy()
X = make_lags(y, lags=4).fillna(0.0)


def make_multistep_target(ts, steps):
    return pd.concat(
        {f'y_step_{i + 1}': ts.shift(-i)
         for i in range(steps)},
        axis=1)


# Eight-week forecast
y = make_multistep_target(y, steps=8).dropna()

# Shifting has created indexes that don't match. Only keep times for
# which we have both targets and features.
y, X = y.align(X, join='inner', axis=0)
```

### マルチアウトプットモデル
マルチアウトプット戦略として線形回帰を使用します。複数出力用にデータを準備してしまえば、学習と予測の手順はこれまでと同じです。
```python
# Create splits
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.25, shuffle=False)

model = LinearRegression()
model.fit(X_train, y_train)

y_fit = pd.DataFrame(model.predict(X_train), index=X_train.index, columns=y.columns)
y_pred = pd.DataFrame(model.predict(X_test), index=X_test.index, columns=y.columns)
```

マルチステップモデルは、各入力インスタンスに対して horizon 全体の予測を出力することを覚えておきましょう。学習セットには269週、テストセットには90週あり、それぞれの週に対して8ステップ分の予測が得られます。
![](https://storage.googleapis.com/zenn-user-upload/e154c4782194-20260303.png)

### ダイレクト戦略
XGBoostは回帰タスクで複数出力を直接生成することはできません。しかし、ダイレクト戦略を適用することで、マルチステップ予測に利用できます。これは scikit-learn の MultiOutputRegressor でラップするだけで実現できます。
```python
from sklearn.multioutput import MultiOutputRegressor

model = MultiOutputRegressor(XGBRegressor())
model.fit(X_train, y_train)

y_fit = pd.DataFrame(model.predict(X_train), index=X_train.index, columns=y.columns)
y_pred = pd.DataFrame(model.predict(X_test), index=X_test.index, columns=y.columns)
```

ここではXGBoostは学習データに対して明らかに過学習しています。しかしテストデータでは、線形回帰モデルよりもインフルエンザ流行の動きをうまく捉えているように見えます。ハイパーパラメータを調整すれば、さらに良い結果が期待できるでしょう。
```python
train_rmse = mean_squared_error(y_train, y_fit, squared=False)
test_rmse = mean_squared_error(y_test, y_pred, squared=False)
print((f"Train RMSE: {train_rmse:.2f}\n" f"Test RMSE: {test_rmse:.2f}"))

palette = dict(palette='husl', n_colors=64)
fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(11, 6))
ax1 = flu_trends.FluVisits[y_fit.index].plot(**plot_params, ax=ax1)
ax1 = plot_multistep(y_fit, ax=ax1, palette_kwargs=palette)
_ = ax1.legend(['FluVisits (train)', 'Forecast'])
ax2 = flu_trends.FluVisits[y_pred.index].plot(**plot_params, ax=ax2)
ax2 = plot_multistep(y_pred, ax=ax2, palette_kwargs=palette)
_ = ax2.legend(['FluVisits (test)', 'Forecast'])
```
