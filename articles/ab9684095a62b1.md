## SHAP値
個々の予測を理解します

## 理論編
### はじめに(Introduction)
これまでに、機械学習モデルから一般的な洞察を引き出す手法を見てきました（そして実際に使ってきました）。しかし、個々の予測についてモデルがどのように動作しているのかを分解して理解したい場合はどうでしょうか？

SHAP Values（SHapley Additive exPlanations の略）は、予測を分解して各特徴量の影響を示します。これはどのような場面で使えるでしょうか？

* モデルが「銀行はある人に融資すべきではない」と判断した場合、銀行は各融資拒否の理由を法的に説明する必要がある
* 医療提供者が、各患者の特定の病気リスクを高めている要因を特定し、それらのリスク要因に対して直接的な介入を行いたい場合

このレッスンでは、SHAP Values を使って個々の予測を説明します。次のレッスンでは、それらを集約してモデル全体の強力な洞察を得る方法を学びます。

### How They Work
SHAP値は、ある特徴量が特定の値をとることによる影響を、その特徴量が基準値（baseline）をとった場合の予測と比較することで解釈します。

例を見ると分かりやすいので、これまでの [順列の重要度(permutation importance)](https://zenn.dev/rg687076/articles/1060577e1e0b24) や [部分的なプロット(partial dependence plots)](https://zenn.dev/rg687076/articles/79a9edbf1fedfe) のレッスンで扱ったサッカーの例を続けて使います。

これらのチュートリアルでは、「チームの選手が Man of the Match を受賞するかどうか」を予測していました。

例えば、次のように問いかけることができます：

* チームが3ゴール決めたことは、どの程度予測に影響したのか？

しかし、これをより具体的で数値的に答えやすくするには、次のように言い換えます：

* チームが「**ある基準のゴール数ではなく**、3ゴール決めたこと」は、どの程度予測に影響したのか？

もちろん、各チームには多くの特徴量があります。したがって、ゴール数についてこの問いに答えた後、他のすべての特徴量についても同様に繰り返すことができます。

SHAP値は、優れた特性を保証する方法でこれを実現します。具体的には、次の式を使用して予測を分解します。

sum(すべての特徴量の SHAP値) = チームの予測値 − 基準値での予測

つまり、すべての特徴量のSHAP値を合計すると、「なぜ予測が基準値からズレたのか」を説明できます。これにより、次のようなグラフで予測を分解できます。

![](https://storage.googleapis.com/zenn-user-upload/bd1163c7fc8f-20260319.png)

このグラフはどのように解釈するのでしょうか？

私たちの予測は0.7で、基準値は0.4979です。予測を増加させる特徴量はピンクで示され、その長さは影響の大きさを表します。予測を減少させる特徴量は青で示されます。最も大きな影響は「得点（Goal Scored）が2」であることです。一方で、ボール支配率は予測を下げる方向に意味のある影響を持っています。

青いバーの長さをピンクのバーの長さから引くと、基準値から最終的な予測値までの差に一致します。

この手法にはいくつかの複雑さがあります。特に「基準値＋各特徴量の効果の合計＝予測値」となるようにするための仕組みは、一見すると単純ではありません。ただし、この詳細は手法の利用に必須ではないため、ここでは扱いません。この[ブログ記事](https://towardsdatascience.com/the-shapley-value-for-ml-models-f1100bff78d1/)には、より詳しい理論的説明が掲載されています

### SHAP値を計算するコード
SHAP値は [Shap](https://github.com/shap/shap) ライブラリを使って計算します。

この例では、これまで見てきたサッカーデータのモデルを再利用します。
```python
import numpy as np
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier

data = pd.read_csv('../input/fifa-2018-match-statistics/FIFA 2018 Statistics.csv')
y = (data['Man of the Match'] == "Yes")  # Convert from string "Yes"/"No" to binary
feature_names = [i for i in data.columns if data[i].dtype in [np.int64, np.int64]]
X = data[feature_names]
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state=1)
my_model = RandomForestClassifier(random_state=0).fit(train_X, train_y)
```

データセットの1行（ここでは5行目）についてSHAP値を確認します。その前に、まずは通常の予測結果を見ておきます。

```python
row_to_show = 5
data_for_prediction = val_X.iloc[row_to_show]  # use 1 row of data here. Could use multiple rows if desired
data_for_prediction_array = data_for_prediction.values.reshape(1, -1)

my_model.predict_proba(data_for_prediction_array)
```
出力:
/opt/conda/lib/python3.7/site-packages/sklearn/base.py:451: UserWarning: X does not have valid feature names, but RandomForestClassifier was fitted with feature names
  "X does not have valid feature names, but"
array([[0.29, 0.71]])


チームが受賞する確率は70%です。
それでは、その単一の予測に対するSHAP値を取得するためのコードに移りましょう。

```python
import shap  # package used to calculate Shap values

# Create object that can calculate shap values
explainer = shap.TreeExplainer(my_model)

# Calculate Shap values
shap_values = explainer.shap_values(data_for_prediction)
```

上記の shap_values は2つの配列を持つリストです。
1つ目は「負の結果（受賞しない）」に対するSHAP値、2つ目は「正の結果（受賞する）」に対するSHAP値です。通常は「正の結果」に対する予測に注目するため、shap_values[1] を取り出します。

生の配列は扱いにくいですが、shap パッケージには結果を可視化する便利な機能があります。

```python
shap.initjs()
shap.force_plot(explainer.expected_value[1], shap_values[1], data_for_prediction)
```
![](https://storage.googleapis.com/zenn-user-upload/733411372c26-20260320.png)

SHAP値を作成したコードを見ると、shap.TreeExplainer(my_model) のように Tree を参照していることに気づくかもしれません。しかし、SHAPパッケージはあらゆるモデルタイプに対応した explainer を提供しています。

* shap.DeepExplainer はディープラーニングモデルに対応
* shap.KernelExplainer はすべてのモデルに対応（ただし他より遅く、近似値を返す）

以下は KernelExplainer を使った例です。結果は完全には一致しませんが、同じ傾向を示します。

```python
# use Kernel SHAP to explain test set predictions
k_explainer = shap.KernelExplainer(my_model.predict_proba, train_X)
k_shap_values = k_explainer.shap_values(data_for_prediction)
shap.force_plot(k_explainer.expected_value[1], k_shap_values[1], data_for_prediction)
```
