# Abstract
Stable Diffusionの連続使用で黒画面になって以降、再起動せんと使えんくなるんだけど、原因がGPUの熱暴走だったという話。

# 経緯
Stable Diffusionって、ローカルPCに構築すると無料で出来るらしい。
というのが分かったので、動かしてみたんだけど(手順は[ココ](https://zenn.dev/rg687076/articles/217204da53d67a)参照)、"Generate forever"で連続実行させると、ものの10分で黒画面に。。。
なんでだ。。。

# 判明
いろいろ調べると、GPUのメモリ不足ってキーワードが出てきた。メモリならタスクマネージャで、見ながら動かせばホントかどうかが分かる。
![](https://storage.googleapis.com/zenn-user-upload/39bd30ff7413-20240406.png)
さっそくタスクマネージャ起動して見てみたら、一目瞭然、GPU温度がもの凄い高温になってた。なるほど～。これが原因か～。
GPUのメモリ不足は関係なかった。

# 解決へ
GPUだけ水冷化するのも検討したけど、水冷化キットってCPU向けしかなくてけっこうお高い。
改造してGPUにつけるには、初心者にはハードル高い。(失敗がね。)
どうしようかと調べてたら、Paperspace Gradientってサービスを見つけた。
解決！！よかった。
Paperspace Gradientの使い方は[解決編](https://zenn.dev/rg687076/articles/0ad4da437d4782)を参照してね。

※結局ローカルPCで動かす時は、送風機でグラボに送風するのが一番コストが安かった。\2,000ぐらいの安いやつ!
