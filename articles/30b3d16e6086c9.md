## 6. Random Forests
### 理論編
#### はじめに（Introduction）

決定木は、扱いが難しい選択を伴います。
葉の数が多い深い木は、各予測がその葉に属する少数の住宅データという過去情報だけに基づくため、過学習（オーバーフィッティング） しやすくなります。
一方で、葉の数が少ない浅い木は、生データに含まれる多くの違いを捉えきれず、性能が悪くなってしまいます。

この 過学習と未学習（アンダーフィッティング） の緊張関係は、現代の最も高度なモデリング手法であっても避けられません。
しかし、多くのモデルには、より良い性能を引き出すための巧妙なアイデアがあります。ここでは、その一例として ランダムフォレスト（Random Forest） を見ていきます。

#### ランダムフォレストとは

ランダムフォレストは 多数の決定木 を使用し、それぞれの木が出した予測結果を 平均 することで最終的な予測を行います。
一般的に、単一の決定木よりも はるかに高い予測精度 を持ち、しかも デフォルトのパラメータ設定でも十分にうまく動作 します。

モデリングを続けていけば、さらに高性能なモデルも学ぶことができますが、それらの多くは 適切なパラメータ調整に非常に敏感 です。

#### 例（Example）

データを読み込むコードは、すでに何度か見てきました。
データ読み込みの最後には、次の変数が用意されています。
- train_X
- val_X
- train_y
- val_y
```python
import pandas as pd
    
# Load data
melbourne_file_path = '../input/melbourne-housing-snapshot/melb_data.csv'
melbourne_data = pd.read_csv(melbourne_file_path) 
# Filter rows with missing values
melbourne_data = melbourne_data.dropna(axis=0)
# Choose target and features
y = melbourne_data.Price
melbourne_features = ['Rooms', 'Bathroom', 'Landsize', 'BuildingArea', 
                        'YearBuilt', 'Lattitude', 'Longtitude']
X = melbourne_data[melbourne_features]

from sklearn.model_selection import train_test_split

# split data into training and validation data, for both features and target
# The split is based on a random number generator. Supplying a numeric value to
# the random_state argument guarantees we get the same split every time we
# run this script.
train_X, val_X, train_y, val_y = train_test_split(X, y,random_state = 0)
```

ここでは、scikit-learn を使って決定木を作成したときとほぼ同じ手順で、
DecisionTreeRegressor の代わりに RandomForestRegressor クラス を使ってランダムフォレストモデルを構築します。
```python
from sklearn.ensemble import RandomForestRegressor
from sklearn.metrics import mean_absolute_error

forest_model = RandomForestRegressor(random_state=1)
forest_model.fit(train_X, train_y)
melb_preds = forest_model.predict(val_X)
print(mean_absolute_error(val_y, melb_preds))
```

出力結果： 191669.7536453626

#### まとめ（Conclusion）

さらに改善の余地はあるかもしれませんが、これは 最良の決定木モデルの誤差（250,000） と比べると、大きな改善です。

ランダムフォレストにも、単一の決定木で最大深さを調整したように、性能を変更できるパラメータが存在します。
しかし、ランダムフォレストの大きな利点の一つは、こうしたチューニングを行わなくても、概ね良好に動作する点 にあります。

### 実践編
#### ここまでのまとめ
ここまでに書いたコードは次のとおりです。
```python
# Code you have previously used to load data
import pandas as pd
from sklearn.metrics import mean_absolute_error
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeRegressor


# Path of the file to read
iowa_file_path = '../input/home-data-for-ml-course/train.csv'

home_data = pd.read_csv(iowa_file_path)
# Create target object and call it y
y = home_data.SalePrice
# Create X
features = ['LotArea', 'YearBuilt', '1stFlrSF', '2ndFlrSF', 'FullBath', 'BedroomAbvGr', 'TotRmsAbvGrd']
X = home_data[features]

# Split into validation and training data
train_X, val_X, train_y, val_y = train_test_split(X, y, random_state=1)

# Specify Model
iowa_model = DecisionTreeRegressor(random_state=1)
# Fit Model
iowa_model.fit(train_X, train_y)

# Make validation predictions and calculate mean absolute error
val_predictions = iowa_model.predict(val_X)
val_mae = mean_absolute_error(val_predictions, val_y)
print("Validation MAE when not specifying max_leaf_nodes: {:,.0f}".format(val_mae))

# Using best value for max_leaf_nodes
iowa_model = DecisionTreeRegressor(max_leaf_nodes=100, random_state=1)
iowa_model.fit(train_X, train_y)
val_predictions = iowa_model.predict(val_X)
val_mae = mean_absolute_error(val_predictions, val_y)
print("Validation MAE for best value of max_leaf_nodes: {:,.0f}".format(val_mae))


# Set up code checking
from learntools.core import binder
binder.bind(globals())
from learntools.machine_learning.ex6 import *
print("\nSetup complete")
```
